# 高质量信息：定义与量化标准

> 2026-06-01 重构版 | 基于"你能用它做什么"而非"它看起来对不对"

---

## 0、上一版的问题

上一版犯了两个错：

1. **质量判断与决策场景脱钩** —— 同一条信息对你投资是噪声、对AI能力是依据。绝对评分没有意义。
2. **五维评分太复杂** —— 真到每日过滤信息时，没人会逐条打100分。需要的是"30秒过滤法"。

重构后的框架只有一个核心追问：

---

## 一、核心定义

**高质量信息 = 让你在三个角色中至少一个角色改变行动的信息。**

你的三个角色：

| 角色 | 决策类型 | 信息需求节奏 | 典型信息源 |
|:----|:---------|:-----------:|:----------|
| 🏦 **银行家** | 科技金融框架迭代、客户方案、政策建言 | 周/月 | 政策文件、行内数据、同业动态 |
| 📈 **投资者** | 仓位调整、买卖时机、行业判断 | 日/周 | 财报、行业数据、市场信号 |
| 🤖 **AI实践者** | 方法论文档更新、工具选型、知识沉淀 | 日 | AI前沿简报、模型发布、工具更新 |

**一条信息如果不能让上述任一角色的决策发生改变（哪怕只是"我决定本周关注X"），它就是噪声。**

---

## 二、两步过滤法（30秒）

### 第一步：信源分级

| 级别 | 定义 | 来源类型 | 颜色标记 |
|:----:|:-----|:---------|:--------:|
| **S级** | 一手官方数据 | 财报PDF原文、交易所公告、央行/金融监管总局正式文件 | 🔵 |
| **A级** | 可信第三方数据 | Wind/同花顺行业数据、多源交叉验证的券商研报、官方媒体报道 | 🟢 |
| **B级** | 单来源但有价值 | 单份深度研报、行业KOL的分析、未经交叉验证的新闻 | 🟡 |
| **C级** | 无法追溯的传闻 | 微信群截图、社交媒体爆料、"据传"类消息 | 🔴 |

**规则**：C级默认跳过。B级标注来源后决定是否L2存储。A/S级直接进入第二步。

### 第二步：决策映射

用一个问题判断：**"这条信息能改变我哪个角色的哪个决策？"**

```
3×3 决策映射矩阵:

        改变认知   改变关注   改变行动
银行家   ┌──────┬──────┬──────┐
        │ L2参考│ L2参考│ L3依据│
        ├──────┼──────┼──────┤
投资者  │ L2参考│ L2-L3 │ L3依据│
        ├──────┼──────┼──────┤
AI实践者│ L1跳过│ L2参考│ L3依据│
        └──────┴──────┴──────┘
```

- **改变认知**：以前不知道的事，但暂时不需要行动 → L2 参考（存储备查）
- **改变关注**：以前没注意的方向，决定本周关注 → L2-L3（加入跟踪列表）
- **改变行动**：直接影响买卖/写作/建系统 → L3 依据（立即处理）

## 三、三级信息流处理

```
每日信息洪流（AI简报/财经新闻/行业动态）
           │
           ▼
    第一步：信源分级
    ┌───┬───┬───┬───┐
    │ S │ A │ B │ C │ ← C级直接丢弃
    └─┬─┴─┬─┴─┬─┘
      │   │   └→ 标注来源后可选L2存储
      │   │
      ▼   ▼
    第二步：决策映射
    ┌────────┬────────┬────────┐
    │改变认知 │改变关注 │改变行动 │
    └───┬────┴───┬────┴───┬────┘
        │        │        │
        ▼        ▼        ▼
      L2参考   L2-L3    L3依据
     ├──────────┼──────────┤
     │ 存入      │ 加入      │ 立即      │
     │ 信息池    │ 跟踪列表  │ 处理      │
     └──────────┴──────────┴──────────┘
```

## 四、量化选项（可选，非必须）

如果需要对信息质量做定量追踪（如：每周统计信息质量趋势），用单一指标：

**信息质量指数 = 信源等级分 × 决策影响深度**

```
信源等级分：
  S=100  A=70  B=40  C=10

决策影响深度：
  改变行动=3  改变关注=2  改变认知=1  无影响=0

示例：
  央行正式发文（S=100）× 影响投资仓位决策（3）= 300 → L3
  微信群传闻（C=10）× 改变AI关注（2）= 20 → L1 噪声
```

| 指数区间 | 等级 | 处理 |
|:--------:|:----:|:----|
| ≥200 | L3依据 | 立即处理或输入Agent |
| 80-199 | L2参考 | 存储，标记来源 |
| <80 | L1噪声 | 跳过 |

## 五、具体场景应用

### 场景1：每日AI简报搜索
搜索到5条AI新闻，每条的过滤路径：

```
新闻："英伟达发布N1X AI芯片，200 TOPS端侧算力"
→ 信源级别：A（多源科技媒体报道一致）
→ 决策映射：改变AI实践者的关注（新工具影响工具选型）
→ 等级：L2参考 → 加入AI简报+标注"对陈颖芳的意义"
```

```
新闻："据传某AI公司将裁员"
→ 信源级别：C（无明确来源）
→ 决策映射：无（不持仓、不影响行业判断）
→ 等级：L1噪声 → 跳过
```

### 场景2：投资决策
```
新闻："昆仑万维Q1营收同比增长X%"
→ 信源级别：S（公司财报PDF原文，假设已获取）
→ 决策映射：改变投资者的行动（清仓/减仓/持有判断）
→ 等级：L3依据 → 输入Agent A组
```

### 场景3：微信群信息
```
群消息："新能源又有政策利好"
→ 信源级别：C（无出处、无具体政策文号）
→ 决策映射：投资方面可能改变认知，但无法验证
→ 等级：L1噪声 → 跳过
```

## 六、与现有系统的对接

| 现有系统 | 对接方式 |
|:---------|:---------|
| **每日AI简报** | 搜索结果的过滤标准：至少B级以上信源+至少改变认知的决策影响 |
| **Agent A-E流程** | 只有L3依据级别的信息可以输入Agent A（财报分析） |
| **记忆文件更新** | 只有改变"认知"以上的信息才值得写入 |
| **飞轮周扫描** | 飞轮报告中的"新发现"板块仅收录至少L2参考级别的信息 |
| **定时任务推送** | 推送到微信的信息必须至少L2参考，避免产生"推送疲劳" |

---

## 七、一句话原则

**如果这条信息不能让你改变一个具体决策，它就不是高质量信息。**

不需要打分、不需要五维评估、不需要纠结。两步判断：信源可信吗？→ 能改变什么决策？→ 通过就处理，不通过就跳过。
