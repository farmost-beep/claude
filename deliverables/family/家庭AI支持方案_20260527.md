# 家庭AI支持方案 — 陈颖芳

> 制定日期：2026-05-27
> 适用对象：陈于东（20岁，考研备考）+ 两位母亲（74岁，异地赡养）
> 目标：用AI把重复性、分析性工作自动化，让陈颖芳只做"决策"和"情感连接"这两件不可替代的事

---

## 一、设计原则

### 1.1 三个"不增加"

| 原则 | 含义 | 具体体现 |
|------|------|---------|
| **不增加输入负担** | 数据采集自动化或半自动化，不需要她每天手动填表 | 拍照即录入、语音即记录 |
| **不增加认知负担** | AI做分析，她只看结论和异常提醒 | 所有数据由Claude分析后输出一句话摘要 |
| **不增加决策负担** | AI提供选项和建议，但不替她做决定 | 每个建议都附带"如果不做会怎样"的后果说明 |

### 1.2 三个核心角色

| 角色 | 赋予谁 | 做什么 | 不做什么 |
|------|--------|--------|---------|
| **数据管家** | Claude | 接收数据、归档、绘制趋势图、标记异常 | 不替孩子做职业选择 |
| **规划助手** | Claude | 生成周计划、提醒节点、提供选项 | 不替主人做执行决策 |
| **情感锚点** | 陈颖芳自己 | 与孩子散步聊天、给母亲打电话 | 这是AI无法替代的，也是最重要的部分 |

### 1.3 AI的边界

AI能做的：整理信息、发现模式、生成方案、定时提醒
AI不能做的：替孩子做职业选择、代替母子之间的情感交流、替代母亲陪伴

## 二、AI辅助考研备考系统

> 目标用户：陈于东为主，陈颖芳获取摘要即可
> 核心思路：AI帮他做"整理和分析"，他只需要"学"和"问"

### 2.1 个性化学习计划生成器

**工作流**：每周日晚上，陈于东花5分钟回答3个问题发给Claude

```
输入（陈于东，每周日晚，3分钟语音或文字）：
1. 本周完成了什么？（英语单词打卡X天，数学学到哪一章，精读了几篇阅读）
2. 最难的地方在哪里？（哪个知识点击败了你？）
3. 下周有什么特殊安排？（实习/出行/聚会占用几天？）

Claude输出（2分钟后）：
1. 下周每日学习计划表（精确到时段，自动匹配每日时间模板）
2. 本周薄弱知识点强化建议（推荐3-5道针对性的练习题编号）
3. 一句话鼓励（根据本周表现生成，不鸡汤，说事实）
```

**为什么这样设计**：
- 3个问题30秒语音就能回答，不增加负担
- 计划由AI生成，他只负责执行，降低心理启动成本
- 每周日固定节奏，形成仪式感

**实际产出示例**：
```
【陈于东 8月第2周学习计划】

数学（重点：微分中值定理--上周卡住的地方）
- 周一三五：08:30-10:00 看完李永乐P85-102（中值定理+泰勒公式）
- 周二四：08:30-10:00 做习题集第3章选择题1-20题
- 周六：09:00-11:00 本周中值定理错题重做 + 整理

英语（保持节奏）
- 每日：30个新词（墨墨自动安排）+ 复习旧词
- 周四：精读2010年Text 2（上周Text 1你正确率80%，继续保持）
- 周日：精读2010年Text 3

院校调研（本周收尾）
- 周六下午：把你做的对比表发给妈妈征求意见

鼓励：上周英语连续打卡7天，数学高数上册前两章笔记完成了。
       中值定理卡住很正常--这是高数上册最难的一章，1950年代的数学系学生也为它熬夜。
```

### 2.2 错题分析系统

**工作流**：拍照即分析

```
输入（陈于东，随时）：
用手机拍下做错的数学题（一道也可，多道也可），发给Claude

Claude输出：
1. 错题归类（知识点标签：如"拉格朗日中值定理·构造函数技巧"）
2. 错误类型分析（概念不清/计算粗心/思路错误/题型不熟）
3. 推荐3道同类题（从真题/习题库中匹配）
4. 一句话解题技巧提示
5. 自动归档到个人错题库（按章节+错误类型索引）
```

**为什么这样设计**：
- 拍照是最低门槛的输入方式
- 自动归类节省了"打开错题本、抄题、写分析"的时间
- 同类题推荐是AI的核心优势--匹配比人类快

**错题库积累到20题以上时**：
- Claude自动生成"薄弱章节热力图"（哪个章节错题最多，一目了然）
- 考前生成"个人易错点清单"（考前15分钟看一眼的那种）

**实现方式**：
- 陈于东在微信中拍照发给专属群聊（母子+Claude bot）
- 或者直接用Claude App拍照上传
- 错题库由Claude维护为Markdown文件，定期更新

### 2.4 院校选择AI参谋

**已在现有方案中部分实现**。AI可进一步增强：

```
输入（陈于东，大三上学期）：确定了3个候选专业方向，请帮我分析就业前景

Claude输出：
1. 每个方向近3年就业去向数据（来源：公开就业报告、招聘平台数据）
2. 每个方向在上海的就业岗位数量趋势
3. 结合陈颖芳的金融行业人脉，哪个方向资源利用率最高
4. 跨考难度评估（针对非本专业方向）
5. 一句话建议："根据你当前的数学水平和兴趣，XX方向匹配度最高，理由是..."
```

### 2.5 复试模拟面试教练

**2027年12月初试后启动，2028年1-3月密集使用**：

- AI生成面试题库（根据目标院校和专业的历史面试经验帖）
- 模拟面试对话（AI扮演面试教授，陈于东语音回答，AI给反馈）
- 英文自我介绍打磨（AI逐句优化语法和表达，不是翻译腔）
- 常见陷阱问题准备（"你为什么选择我们学校""你的研究兴趣是什么""你有什么缺点"）

## 三、AI辅助异地母亲赡养

> 此部分轻量化设计 -- 赡养的核心是情感连接而非技术方案

### 3.1 通话提醒 + 通话后记录

```
每周日Claude提醒："本周和两位母亲通过电话了吗？"
（如已通话，陈颖芳只需要回复"通了"）

如果通话时提到健康相关的事（如"妈说最近膝盖疼"），
陈颖芳语音告诉Claude一句话，Claude做三件事：
1. 记录到母亲的健康档案
2. 分析是否需要关注（如"膝盖疼"在冬季需关注防滑倒，提示适老化检查）
3. 下次通话前提醒："上次妈说膝盖疼，这次问问有没有好转"
```

### 3.2 赡养月历自动提醒

Claude在每个月的第一天推送：

```
【6月赡养提醒】
- 主题：建立基础档案
- 本月任务：端午节回乡探望，当面整理健康档案
- 财务：6月赡养费按时转账
- 待完成：确认两位母亲医保类型和报销范围
- 预算提醒：医疗应急备用金10-20万需独立于投资仓位
```

### 3.3 应急准备清单

Claude维护的应急信息卡片，包含：
- 两位母亲的家庭地址（精确到门牌号）
- 紧急联系人姓名+电话
- 最近的120急救中心电话
- 从上海出发的最快回乡方式
- 陈颖芳办公室和家中应急背包的位置

在Claude对话中说"紧急预案"即可调出全部信息。

## 四、工具栈与每周工作流

### 4.1 工具选择

| 工具 | 用途 | 费用 | 谁用 |
|------|------|------|------|
| **Claude** | 核心分析引擎：学习计划生成、错题分析、进度追踪 | 已有 | 陈颖芳+陈于东 |
| **微信** | 消息推送、语音输入、拍照发送 | 0元 | 全体 |
| **手机日历** | 关键日期提醒（与Claude输出的日期同步） | 0元 | 陈颖芳 |
| **墨墨背单词/百词斩** | 考研英语单词打卡 | 0元（基础功能） | 陈于东 |

### 4.2 每周工作流

```
【每周日晚 20:00-20:30】-- 学习/赡养信息汇总

陈于东（5分钟）：
  → 发语音/文字："本周英语打卡7天，数学学到定积分，做了2篇精读"
  → 陈颖芳转发给Claude，或陈于东直接发给Claude

Claude（5分钟，自动生成）：
  → 输出给陈于东：下周学习计划 + 错题推荐
  → 输出给陈颖芳：一句话学习摘要 + 赡养提醒

陈颖芳（2分钟）：
  → 看一眼摘要，转发给陈于东（如果他没直接收到）
  → 确认是否和母亲通话
  → Done.
```

### 4.3 数据流架构

```
采集层                分析层              输出层
───────              ───────            ───────
微信语音 ──→ 学习进度 ──┐                   ┌→ 陈于东周报（完整版）
微信拍照 ──→ 错题    ──┤                   ├→ 陈颖芳摘要（一句话）
语音输入 ──→ 通话记录 ─┼──→ Claude分析 ──→ ├→ 学习预警提醒
日历提醒 ──→ 关键节点 ─┘                   ├→ 赡养月历推送
                                           └→ 应急信息调取
```

## 六、FAQ

**Q：我儿子不愿意把错题发给AI分析怎么办？**
A：那是他的选择。这个系统的核心原则是"不替代人的意愿"。如果他不愿意，AI仍然可以帮他做学习计划生成和进度追踪。错题分析是他的工具，用不用他说了算。

**Q：我要额外花钱吗？**
A：不需要。Claude已是你日常使用的AI工具，单词APP免费，微信和日历已有。0额外费用。

**Q：这个系统会不会让我变成被数据控制的机器人？**
A：不会。AI多做分析，你只做两件事：接收摘要、做情感连接。你不是在给AI打工，AI在给你打工。
