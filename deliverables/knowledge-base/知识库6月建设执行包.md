## 四、知识卡片模板

> 以下模板已整合操作手册附录A（阅读卡片）和附录C（框架索引卡片）的通用结构。所有卡片存于对应领域的文件夹下。

### 4.1 阅读卡片（Zettelkasten格式）

文件名格式：`CARD-YYYYMMDD-NNN.md`（如 `CARD-20260608-001.md`）

```
---
卡片ID: CARD-YYYYMMDD-NNN
标题: [一句话核心观点]
日期: YYYY-MM-DD
来源: 《书名》，作者，第X章
领域: [INV|TF|BIZ|TECH|FAM|HEALTH|PHIL|LIFE]
标签: [#标签1 #标签2 #标签3]
状态: [初稿|已连接|已迭代]
---

# [标题]

## 观点摘录
> 原文关键段落（≤100字）

## 个人思考
用自己的话重构观点。≤300字。

## 连接提示
- 关联卡片：[[CARD-XXXXX]]
- 关联框架：[[框架名称]]
- 行动启示：[可以如何指导实践？]

## 版本记录
- V1 | YYYY-MM-DD | 初稿
```

### 4.2 框架索引卡片

文件名格式：`[框架全称].md`（如 `Long Now 长现.md`）

```
---
框架ID: FRAMEWORK-NN
名称: [框架全称]
版本: V1.0
创建日期: YYYY-MM-DD
领域: [PHIL|INV|TF|LIFE]
状态: [活跃|待迭代|归档]
---

# [框架名称]

## 一句话定义
[50字以内]

## 核心观点
1. [观点1]
2. [观点2]
3. [观点3]

## 适用范围
| 场景 | 说明 |
|------|------|
| 适用 | [...] |
| 不适用 | [...] |

## 存储位置
- 脚本：`scripts/build_xxx_pdf.py`
- 文档：`deliverables/xxx/xxx.pdf`

## 关联框架
- [[框架名称]] — [关联关系]

## 版本历史
- V1.0 | YYYY-MM-DD | 初版
```

### 4.3 卡片质量标准（适用所有类型）

| 维度 | 标准 | 检查方法 |
|------|------|---------|
| **原子性** | 一张卡片只讲一个观点 | 能否用一句话总结？ |
| **可连接** | ≥1条wikilink到已有卡片或框架 | 打开Obsidian图谱视图检查 |
| **有观点** | 不只是摘录，必须有"个人思考"或"一句话定义" | 删除摘录后，剩余内容是否有独立价值？ |
| **守篇幅** | 阅读卡片≤500字，框架卡片≤300字 | wc -m 计数 |

## 六、自动化管道

### 6.1 管道架构

```
┌──────────────┐     ┌──────────────────┐     ┌──────────────┐
│  md编辑       │ ──→ │ build_goal_pdfs  │ ──→ │  PDF生成      │
│ (deliverables)│     │ .py (批量构建)    │     │ (deliverables)│
└──────────────┘     └──────────────────┘     └──────┬───────┘
                                                     │
                                                     ▼
┌──────────────┐     ┌──────────────────┐     ┌──────────────┐
│  Obsidian     │ ←── │  手动/半自动同步  │ ←── │  PDF验证      │
│  Vault摘要    │     │  (当前阶段)       │     │ (文件存在+大小)│
└──────────────┘     └──────────────────┘     └──────────────┘
```

### 6.2 当前管道能力

| 环节 | 工具 | 状态 | 触发方式 |
|------|------|------|---------|
| md→PDF | `scripts/build_goal_pdfs.py` | 已就绪 | 手动运行 `python3 scripts/build_goal_pdfs.py` |
| PDF验证 | 检查文件存在 + 文件大小 > 0 | 已就绪 | 手动检查 |
| Vault同步 | 手动复制摘要段落到Obsidian | 当前方式 | 按SOP手动执行 |
| 微信提醒 | 未建立 | 待建设 | 7月评估是否加入定时提醒 |

### 6.3 标准操作流程（SOP）：更新deliverable后同步到Vault

```
步骤1：编辑 deliverable md文件
  → 例如：更新 deliverables/investment/投资自由路线图.md

步骤2：运行PDF批量构建
  → cd /Users/cyingfang/claude
  → python3 scripts/build_goal_pdfs.py
  → 确认返回 "OK" 且PDF文件大小 > 0

步骤3：提取核心摘要（≤300字）
  → 从md文件中提取：核心结论 + 关键数据 + 下一步行动
  → 写入对应Vault位置（如 06-知识库/LIFE-人生规划/）

步骤4：更新Vault中的MOC链接
  → 在知识库MOC中添加新摘要的wikilink
  → 如涉及跨域连接，在对应跨域笔记中补充链接

步骤5：验证
  → 在Obsidian中打开MOC，点击新链接确认跳转正常
  → git add + git commit（记录变更）
```
