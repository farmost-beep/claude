# 科技企业风险序列结构

**一句话**：传统信贷风险是"并行加总"的（经营+财务+管理+行业四因子加权求和），科技企业风险是"序列化条件概率"的——技术必须先跑通，市场风险才会暴露，两者有严格的时间先后。

**核心要点**：
- 传统四因子模型的两个隐含假设对科技企业都不成立：(1)风险并行——流片还没做，市场风险就不存在（没有产品可卖）；(2)风险渐进——不会一夜突变，但芯片流片成功估值一夜翻倍，算法被开源替代技术壁垒一夜归零
- 六因子序列模型（按触发顺序）：技术可行性→技术替代→产品化→市场接受→执行→财务。前5个是"能不能成"，第6个是"撑不撑得到成"。传统银行只评估第6个——只关心病人没钱付医药费，不关心得什么病
- 贷后管理应从"月度报表监控"变为"技术里程碑追踪"——每个项目列3-5个关键技术节点，到达→更新风险判断，错过→触发重评估。流片进度比资产负债表有用100倍

**与我相关**：在信贷审批中引入"当前处于哪个序列阶段"的判断，可以让风险描述从"这家企业风险很大"变成"这家企业当前最大单一风险是产品化（良率未知），技术风险已解除"。后者是银行可以定价的，前者只是拒绝的理由。
