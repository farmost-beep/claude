# RAG (检索增强生成)

**一句话**：大模型在生成回答前，先从外部知识库检索相关文档，再基于检索结果生成——既答得准又有据可查。

**核心要点**：
- RAG = Retrieval（检索）+ Augmented（增强）+ Generation（生成）——解决LLM的三大缺陷：知识截止日期（2024年前的停训）、幻觉（编造事实）、不可溯源（不知道答案从哪来）
- 工作流程：用户提问 → 在知识库中检索最相关的文档片段 → 将检索结果作为"参考资料"与问题一起送给LLM → LLM基于参考资料生成答案（附带引用来源）
- RAG vs 微调(Fine-tuning)：RAG适合让模型"会查资料"，微调适合让模型"学会风格和技能"——企业级应用通常两者结合

**与我相关**：RAG是个人知识库建设的下一步方向。当前的知识卡片、投资分析报告、行业研究都是"原材料"，未来可以搭建一个RAG系统：把个人知识库作为检索源，用Claude等LLM做生成层——问"邮储银行科技金融有哪些护城河企业案例"，系统自动检索过往授信记录+研究笔记，生成结构化答案并引用具体文件。

**来源**：Lewis et al., "Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks" (Meta AI, 2020)；LangChain/LlamaIndex框架文档
