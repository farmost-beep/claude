# 管理模式30天启动清单

> 陈颖芳 | 2026年5月
>
> 本文约3500字，阅读时间约9分钟。本文仅代表作者个人观点。
>
> 这是AI方法论系列第十篇。前九篇建立了理论、认知、反模式到行业落地的完整体系。这一篇是操作手册——给你一个30天的行动清单，每天不超过30分钟。

---

## 为什么需要一份启动清单

前九篇文章加起来约4万字。如果你看完了，可能会产生一个合理的问题："所以，我明天该做什么？"

**知道方法和执行方法之间，差一份"最小行动路径"。**

这份清单的设计原则：

1. **每天不超过30分钟**——不给你的日常增加负担
2. **每步有验证标准**——做完和没做完之间没有模糊地带
3. **先跑通再优化**——第一轮不求完美，求完成
4. **四个阶段递进**——从"选择一个场景"到"系统自运转"

---

## 第1周：选场景 + 建记忆

> 核心目标：选定一个你每天都在做的任务，建一份AI记忆文件。

### 第1天：选一个场景（10分钟）

从你每天/每周重复做的任务中，挑一个：
- 阅读研报/财报（投资）
- 写工作汇报/方案（事业）
- 做健康数据分析（健康）
- 整理学习笔记（学习）

**选的标准只有一个：你做这件事每周至少2小时。** 越重复、越结构化，管理模式的ROI越高。

**验证标准：** 写下这个任务的名称和你每周花在上面的时间。

### 第2天：拆解任务（15分钟）

把你选的任务拆成3-5个子步骤。以"财报分析"为例：
1. 下载财报+研报
2. 手动提取核心指标
3. 和上季度/行业对比
4. 写分析总结
5. 标记异常

**验证标准：** 每个子步骤能用一个动词+一个名词描述（如"提取指标"、"对比行业"）。

### 第3天：写一份分析规范（20分钟）

你"怎么判断做得好不好"？写下来：

```
# [任务名]分析规范

## 我需要关注的核心指标/维度（列出5-8个）

## 我判断"好/不好"的标准
- 好：...
- 需要关注：...
- 有问题：...

## 我的输出格式要求（一句话/一张表/一段话？）

## 我必须回答的问题
- ...
- ...
```

**验证标准：** 一个从来没做过这个任务的人，看了这份规范后"知道要做什么"。

### 第4天：补风险信号（15分钟）

给你的规范加一个"预警"部分。问自己：

"过去因为什么出错？如果发生什么，我需要立即注意？"

写3-5个触发条件。每个格式："当[条件]发生时 → [行动]"

**验证标准：** 至少有3个"如果...就..."的触发规则。

### 第5天：存为记忆文件（5分钟）

把前4天写的规范存为AI记忆文件。命名清晰：`[任务名]_分析规范_日期`

**验证标准：** 文件名包含任务名，内容包含核心指标+标准+风险信号。

### 第6天：用记忆跑一次（20分钟）

把你的下一个真实任务交给AI，附带你刚写的规范。观察AI的产出——哪里符合预期？哪里需要调整规范？

**验证标准：** 完成一次"规范驱动的AI协作"，记录3个"规范需要改的地方"。

### 第7天：优化规范（10分钟）

根据昨天的3个改进点，更新规范。**你的记忆文件从"初稿"变成了"经过实战检验的v1.0"。**

**验证标准：** 规范更新了至少1处。

---

## 第2周：拆任务 + 跑并行

> 核心目标：把一个任务拆成多个并行子任务，感受效率差。

### 第8天：把一个任务拆成4个子任务（15分钟）

不按串行步骤拆，按"独立信息源/维度"拆：

不要这样拆：
```
步骤1 → 步骤2 → 步骤3 → 步骤4
```

要这样拆：
```
并行任务A：从维度1分析
并行任务B：从维度2分析
并行任务C：从维度3分析
并行任务D：从维度4分析
↓
汇总Agent：整合+检查矛盾+列关键问题
```

**验证标准：** 至少有3个子任务可以独立执行，不需要彼此的结果。

### 第9天：第一次并行跑（20分钟）

把你的任务同时发给3-4个Agent。每个Agent做不同维度。观察——跑完要多长时间？以前逐个做要多长时间？

**验证标准：** 记录了"新模式耗时 vs 旧模式耗时"。

### 第10天：加一个汇总Agent（15分钟）

在并行Agent跑完后，加一个汇总Agent：

```
你是一个汇总审查员。以下是[X]个Agent的产出。
你的任务：
1. 检查数据一致性——不同Agent对同一事实有没有矛盾表述？
2. 找最重要的3个问题——这些产出揭示了什么我需要判断的问题？
3. 告诉我——哪部分我可以不用看（无异常），哪部分我需要深度看？
```

**验证标准：** 汇总Agent生成了"需要我判断的Top 3问题"。

### 第11天：人只判断关键问题（15分钟）

不看Agent的完整产出。只看汇总Agent列出的3个关键问题。每个做判断。

**这是管理模式的精髓：** 你把注意力从"信息处理"彻底转到"判断决策"。

**验证标准：** 你的决策时间 < 15分钟，且你有信心判断是充分的。

### 第12天：回顾并行流程（15分钟）

- 4个子任务的划分合理吗？有没有两个任务在做重复的事？
- 汇总Agent列的问题是否真的是最重要的？
- 有没有你看了汇总后，觉得"我需要看原始产出"的地方？

**验证标准：** 记录了至少2个流程改进点。

### 第13天：重跑一次优化后的流程（20分钟）

用改进后的分拆方式+汇总指令，重跑同一个任务。

**验证标准：** 第二次明显比第一次流畅——少问了AI问题，多做了判断。

### 第14天：写一份流程总结（10分钟）

把这两周的流程总结成200字以内的"我怎么做[任务]"的SOP。

**验证标准：** 200字以内，包含"拆→跑→汇→判"四个环节。

## 第3周：交叉审查 + 异常聚焦

> 核心目标：从"做一个任务"升级到"批量做+异常聚焦"。

### 第15天：选一个可以批量的场景（10分钟）

从你的日常中找一个"重复做同一件事"的场景：
- 每周复盘多只股票
- 每月写多份同类型报告
- 每周看多篇文章/研报

**验证标准：** 这个任务你一个月做至少3次。

### 第16天：并行处理多个对象（20分钟）

不要一个接一个做。一次把3-5个对象同时交给并行Agent组。

**验证标准：** 跑完3-5个对象的并行分析，总耗时 < 20分钟。

### 第17天：加交叉审查层（20分钟）

两个独立Agent从不同角度分析同一对象，然后对比：

```
Agent A：正向分析（按规范分析）
Agent B：批评者角色（"找出这个分析中最薄弱的3个地方"）

对比：A和B共同发现的问题 = 高置信度
     B指出了但A没注意的 = 盲区
```

**验证标准：** 交叉审查层至少发现了1个"A没注意但B注意到了"的问题。

### 第18天：建立异常排序（15分钟）

不要按"每家公司/每篇文章一页"排列产出。按异常程度排序。

**验证标准：** 排在最前面的3项，确实是你最需要关注的。

### 第19天：只深度看异常项（30分钟）

只看排在前面的异常项。其余的——扫一眼，标记"无异常"，过。

**验证标准：** 你在30分钟内处理完了所有项，且对"为什么这几项排在前面"有自己的判断。

### 第20天：回顾交叉审查效果（15分钟）

- 异常排序是否合理？有没有排错的情况？
- 交叉审查漏掉了什么？
- 有没有低异常项其实很重要但你跳过了？

**验证标准：** 记录了至少2个"异常排序规则"的改进。

### 第21天：建立周度节奏（10分钟）

固定一个时间："每周[X]做批量分析+异常聚焦"。

**验证标准：** 在日历上建了一个重复事件。

---

## 第4周：回顾优化 + 固化习惯

> 核心目标：把前三周跑通的流程固化为"不用想就能执行"的习惯。

### 第22天：全流程总回顾（20分钟）

用一张纸回答：
- 我改了什么？（从第1周到现在的变化）
- 省了多少时间？（用数字，不是感觉）
- 质量变好了还是变差了？（用具体例子）

**验证标准：** 纸上有至少3个具体数字（时间、数量、频次）。

### 第23天：更新所有记忆文件（20分钟）

把前三周学到的改进写回记忆文件：
- 规范需要更新的地方
- 分拆方式需要调整的地方
- 风险信号需要增减的地方

**验证标准：** 至少更新了2个记忆文件。

### 第24天：选第二个场景（10分钟）

管理模式已经在一个场景跑通了。从第1天列出的候选任务中，选第二个。

**验证标准：** 第二个场景选定了，执行第1天的"建规范"流程。

### 第25天：给第二个场景写规范（20分钟）

按照第1周的方法，给第二个任务写分析规范。

**验证标准：** 第二个场景的规范写好并存为记忆。

### 第26天：并行跑第二个场景（20分钟）

用第2周的方法，拆解+并行+汇总第二个场景。

**验证标准：** 第二个场景的并行流程跑通了一次。

### 第27天：建立"每周回顾"习惯（10分钟）

固定时间："每周[X]花10分钟回顾——哪些记忆文件需要更新？"

这是管理模式的"系统维护"——不让记忆腐烂。

**验证标准：** 在日历上建了一个每周10分钟的回顾提醒。

### 第28天：写一份"我的管理模式"总结（15分钟）

用150字以内，回答：我现在的AI协作方式和工作流是什么样的？

**验证标准：** 写完了。不需要完美，但需要写完。

### 第29天：给一个朋友讲一遍（15分钟）

找一个人，用3分钟讲清楚"我怎么用AI改变了我做[任务]的方式"。

**如果你讲不清楚，说明你还没真正内化。** 讲的过程本身会暴露你没想清楚的地方。

**验证标准：** 你讲完了，对方至少听懂了"你做了什么"和"为什么有用"。

### 第30天：庆祝+决定下一步（10分钟）

回顾这30天：
- 最大的变化是什么？
- 最意外的收获是什么？
- 下一步：继续深化现有场景，还是扩展到新场景？

**验证标准：** 你给自己一个明确的下月方向。

---

## 如果你只能做一件事

如果30天太长了——**只做第1-5天。** 选一个场景，写一份规范，存为记忆文件。

那一份文件，是你从"问答模式"切换到"管理模式"的第一块基石。之后每用一次，改一点。三个月后，这份文件会精准得像另一个你。

**管理模式的启动不需要30天。它只需要一个决心：下一次，不让AI回答你的问题——让AI按你的规则，帮你准备答案。**
