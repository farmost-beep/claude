# Claude Mythos：一个AI找到了存在27年的漏洞

2026年4月，Anthropic 的红队发布了一份安全评估报告。标题很克制，内容不克制。

**Claude Mythos Preview 在自主安全测试中，成功发现并利用了所有主流操作系统和浏览器中的零日漏洞。** 包括一个在 OpenBSD 中埋藏了27年的 bug。

OpenBSD 以安全性著称。它的开发团队以代码审计严格闻名，声称"默认安全"。一个27年无人发现的漏洞，意味着在它被写入代码的时候，Windows 98 还没发布。

## 它找到了什么

漏洞在 OpenBSD 的 TCP SACK（选择性确认）实现中，是1998年加入的代码。实际上是两个 bug 链式利用：

1. SACK 块范围起始位置的边界检查缺失
2. TCP 序列号比较中的**有符号整数溢出**，使得一个"不可能"的条件变得可达

攻击者可以利用这两个漏洞，**远程崩溃任何响应 TCP 的 OpenBSD 主机**（拒绝服务攻击）。

更值得注意的不是漏洞本身，而是发现成本：**不到50美元**（整次评估约1000次运行，总成本低于2万美元）。

## 这不是孤例

截至2026年5月22日，Anthropic 的漏洞披露仪表板显示：

- **1,596个漏洞** 在 **281个开源项目** 中被发现
- **97个已被修复**，其中88个获得了 CVE 或 GHSA 编号
- 涵盖 Python 生态（NumPy/SciPy/Pandas 中56%的报告为有效 bug）
- Firefox 单一评估中发现了 **22个漏洞**（14个高危）

作为对比，传统的代码审计和模糊测试也能发现漏洞，但 Mythos 的差异在于：它**不需要人为指定搜索范围**，它自己理解代码、自己构造利用链、自己验证结果。

## Project Glasswing

Anthropic 同时启动了 Project Glasswing——用 AI 做防御性网络安全，而不是攻击性。核心思路：既然 AI 能找到漏洞，那就让它系统性地帮开源项目修复漏洞。

## 这意味着什么

AI 自主发现漏洞这件事，行业已经讨论了好几年。2025年的共识是"AI 可以辅助人类找漏洞"。2026年的 Mythos 评估把这个结论改写成：**AI 可以独立找到并利用人类专家多年未发现的漏洞。**

对网络安全行业：漏洞发现的门槛正在从"专家经验"变成"算力成本"。一个运行成本50美元的 AI Agent 能完成一个安全团队数周的工作。

对 AI 能力边界：这是"AI 在特定领域超越人类专家"最清晰的证据之一。不是下棋，不是游戏——是真实世界的安全工程。
