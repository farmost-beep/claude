# 社交关系AI管家 — 产品设计方案

> 创意：维护社交账号的AI插件 | 2026-06-13 | v1.0

---

## 一、核心价值

**一句话**：记住你所有社交关系中"最近干了什么"和"下一步该干什么"，顺手帮你拟一条消息，推到该去的地方。

**痛点矩阵**：

| 痛点 | 频率 | 现有方案 | 缺口 |
|:----|:---:|:--------|:----|
| 记不清上次和谁聊了什么 | 每周多次 | 翻聊天记录 | 太慢，跨平台不可查 |
| 知道该联系但不知道说什么 | 每周 | 硬想 | 缺乏上下文触发 |
| 重要关系长期不联系 | 每月 | 完全遗忘 | 无主动提醒 |
| 跨平台切换浪费时间 | 每天 | 手动切换 | 信息割裂 |
| 拟消息措辞费劲 | 每天 | 硬写 | 无AI辅助 |

## 三、四大功能模块

### 模块1：关系图谱（who）

| 字段 | 说明 | 数据来源 |
|:----|:-----|:--------|
| 联系人 | 姓名/备注/标签 | 导入通讯录+手动补充 |
| 关系类型 | 家人/朋友/同事/客户/合作伙伴 | 初始化分类 |
| 关系强度 | 1-5分，基于互动频率自动调整 | 自动计算 |
| 标签 | 自定义多维标签（如"考研/孩子/同乡"） | 手动+AI建议 |
| 上次互动 | 日期+平台+摘要 | 自动记录 |

### 模块2：时间线记忆 (What happened)

```
📅 2026-06-10 微信 张总说项目下周验收，等通知
📅 2026-06-08 飞书 李组长发了新方案PDF，待提修改意见
📅 2026-06-05 邮件 王教授同意引荐，等安排时间
📅 2026-05-28 微信 陈于东期末7/7结束，暑假计划待沟通
```

每条记录自动提取：
- **待办**(pending) → 进入提醒队列
- **承诺**(committed) → 追踪是否兑现
- **事件**(event) → 存入时间线

### 模块3：智能提醒（What to do next）

**三档提醒优先级**：

| 级别 | 条件 | 提醒方式 | 例子 |
|:---:|:----|:--------|:----|
| 🔴 **P0** | 承诺已逾期或承诺日在3天内 | 推送通知 | "你答应张总周三给方案，今天周五了" |
| 🟡 **P1** | 超过14天未联系+关系强度≥3 | 每日简报 | "14天没联系王教授了" |
| 🟢 **P2** | 超过30天未联系+关系强度≥2 | 周报汇总 | "本月未互动：李组长/刘老师/陈总" |

**反遗忘机制**：连续N天不联系的强关系自动升温提醒。

### 模块4：AI拟稿+一键发送（Write & Send）

这是核心能力——根据上下文自动生成消息：

**拟稿流程**：

```
触发条件（提醒/手动/定时）
       ↓
AI提取上下文：上次说了什么 → 当前状态 → 关系基调
       ↓
生成消息草稿（可选风格：正式/亲切/简洁/详细）
       ↓
用户预览→确认/修改→发送
```

**示例场景**：

```
触发：P1提醒"14天没联系王教授"
上下文：5/28在飞书聊过，王教授同意引荐但没安排时间
AI拟稿：
  "王教授好，上次聊到引荐的事，不知道您最近方便安排时间吗？
   我这边的材料已经准备好了，随时可以发您过目。"

用户：确认 ✓ → 选微信 → 发送
```

## 四、技术方案

### 架构选择

| 层 | 技术 | 理由 |
|:---|:----|:-----|
| 插件框架 | **Claude Code Skill** + MCP Server | 已有Claude Code基础设施，AI能力内置 |
| 连接层 | cc-connect / iLink API / Email (SMTP/IMAP) | 已有微信/飞书接入经验 |
| 数据存储 | SQLite（本地）+ JSON备份 | 隐私优先，不上云 |
| 消息总线 | MCP协议（标准化工具调用） | 可扩展，解耦 |
| AI核心 | Claude API（自然语言处理+生成） | 长上下文记忆 |

### 数据流

```
社交工具收到消息
  → 连接层捕获
    → 核心引擎分析：提取实体+待办+关系更新
      → 存入本地SQLite
        → 更新关系图谱+时间线
          → 检查提醒条件
            → 触发AI拟稿
              → 推送给用户确认
                → 发送到目标平台
```

### 隐私设计

- **全部数据本地存储**，不上任何云端
- 用户完全控制哪些对话被追踪
- 消息草稿必须用户确认后才发送（绝不自发）
- 可随时导出/删除所有数据

## 五、MVP路线图

### Phase 1：单机版（2周）

| 功能 | 优先级 | 说明 |
|:----|:-----:|:-----|
| 联系人管理+关系分类 | P0 | JSON手动录入 |
| 手动添加时间线条目 | P0 | 关键对话摘要 |
| AI拟稿 | P0 | 根据上下文生成消息 |
| 微信推送（已有） | P0 | 用桥接API |
| 提醒队列 | P1 | P0/P1/P2三档 |

### Phase 2：联网版（+2周）

| 功能 | 优先级 | 说明 |
|:----|:-----:|:-----|
| 微信消息自动抓取 | P0 | 通过cc-connect/ilink |
| 飞书接入 | P1 | cc-connect已有 |
| 邮件接入（IMAP） | P1 | 读取已发邮件 |
| 时间线自动提取 | P0 | 从对话记录自动解析 |

### Phase 3：智能版（+2周）

| 功能 | 优先级 | 说明 |
|:----|:-----:|:-----|
| 关系强度自动计算 | P0 | 基于互动频率+质量 |
| 反遗忘主动提醒 | P0 | N天未联系自动升温 |
| 行为分析仪表盘 | P1 | 社交健康度/人脉覆盖 |
| 多平台同步 | P2 | iCloud/WebDAV |

## 六、差异化价值

| 对比维度 | 现有CRM | 本方案 |
|:--------|:-------|:------|
| 目标用户 | 销售/商务 | **每个人** |
| 数据录入 | 手动，负担重 | AI自动提取 |
| 消息生成 | 模板，僵硬 | AI根据上下文人拟 |
| 平台覆盖 | 单一 | **全平台打通** |
| 隐私 | 上云 | **本地优先** |
| 提醒 | 机械日历 | **基于关系质量** |
