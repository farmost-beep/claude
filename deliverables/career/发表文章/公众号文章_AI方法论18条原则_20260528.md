## 一个前提：Anthropic用AI的方式和你不一样

大多数人用AI是"问答模式"——你问一句，它答一句。Anthropic的工程师用AI是"管理团队模式"——你设定方向，AI自己规划、执行、检查、修正、进化。

这两种模式的区别，就是今天这篇文章要讲的全部内容。

| 问答模式（大多数人在用） | 管理模式（Anthropic在用） |
|----------|------------|
| 你：帮我写个报告 → AI：好的，这是报告 → 你：改一下第二段 | 你：这是背景和格式要求，完成后自己检查一遍再给我 → AI：规划→执行→自查→交付 |
| 一次做一件事 | 同时启动3个Agent并行工作 |
| AI产出=你要自己验证 | AI产出自带自检结果 |
| 每次对话从零开始 | AI记得你的偏好和项目规范 |
| 你操作AI | 你指挥AI团队 |

我花了一周时间逆向研究了Anthropic使用AI构建软件的方法论——从Claude Code的能力进化轨迹中推导出18条原则，分四层，形成一个自我进化的飞轮。

**今天这篇文章，就是这18条原则的完整拆解。** 每条原则都附带"你怎么用"的具体指令，读完可以直接操作。

## 第1层：建立人与AI的正确关系（原则1-5）

> 这一层解决：你和AI之间谁做什么。答案是——AI负责执行，你负责判断。

### 原则1：AI必须直接操作生产环境

**让AI接触你的真实工作，不给它"模拟环境"。**

Anthropic的工程师不让AI输出建议再让人手动执行。他们让Claude Code直接读写文件、执行命令、修改项目。速度差10倍。

你怎么用：

- 错误示范："帮我分析一下这个方案有什么问题，告诉我怎么改"
- 正确示范："这是我的方案，直接帮我改好，改动的地方列出来让我确认"

### 原则2：给AI完整上下文

**每次布置任务，说清四件事——文件在哪、要做什么、为什么做、什么格式。**

AI的"理解能力"=你给的信息量。信息越少，AI猜得越多，错得越多。

正确示范：

> 我是银行科技金融部门负责人。帮我整理这份智算行业研报的核心数据。要求：①用表格呈现，包含公司名、芯片型号、算力参数、商业化进度 ②长度不超过一页A4 ③数据标注来源页码。格式参考我之前写的行业简报风格。

对比一下你平时是怎么问AI的——"帮我总结一下这个研报"。两者的产出质量差距，就是背景信息量决定的。

### 原则3：信任执行，验证结果

**不要盯着AI做事的过程，只检查它交出来的结果。**

Anthropic发现，盯着AI执行每一步不如直接说"修改第3段"然后检查结果。省下盯过程的时间，用于判断。

发给AI一份50页的文件，说"提炼核心观点"——然后去做别的事。回来只看结果：逻辑通吗？重点对吗？格式OK吗？不对就指出哪里不对，让它重新来。

### 原则4：AI主动工作——不让AI等你

**设好规则，让AI定时或按条件自动执行任务，不等你下指令。**

Anthropic的工程师用Routines让AI自动检查PR、自动修复失败、自动汇报进度。人的角色从"推动者"变成"接收者"。

你可以这样起步：
- 每日定时：每天早上8点，AI自动收集行业动态推给你
- 条件触发：持仓股票涨跌超过5%，AI自动分析原因推给你
- 周末回顾：每周日晚上，AI自动总结本周AI对话中的关键发现

先从一个每日定时任务开始。

### 原则5：随时异步推进

**AI不依赖你坐在电脑前。手机发指令→AI后台执行→回来看结果。**

Claude Code创始人Cherny每天通过手机发起150个PR。通勤、排队、开会间隙——随时推进工作。

今天试试：排队时用手机对AI说"帮我查一下关注的公司今天有什么重大公告"，开会时想到一个点子立刻发给AI"帮我记一下这个想法，晚上展开"。

## 第2层：学会组织任务（原则6-9）

> 这一层解决：复杂任务怎么交给AI。答案是——拆解、规划、给目标不给步骤。

### 原则6：复杂任务拆解+分发

**不要一个AI做所有事。把大任务拆成3-5个子任务，分给不同Agent并行执行。**

Anthropic的Multiagent系统让多个Agent同时工作，各自负责一个维度。5分钟完成一个人需要1小时的工作。

正确示范：

> 同时做三件事：1. 帮我查华为昇腾、寒武纪、海光信息的最新AI芯片参数和商业化进展 2. 帮我找这三家公司最近一个季度的财报核心数据 3. 帮我梳理国家对AI芯片的最新政策。每个方向独立完成，最后汇总成一张对比表。

### 原则7：Agent角色分工

**给每个子Agent设定"角色"，它在这个角色下产出更专业。**

Anthropic的子Agent有研究员、审查员、执行者等不同角色。角色越明确，产出越专业。

试试这样说：

> 你现在扮演三个角色同时工作：角色A"行业分析师"——从市场角度分析；角色B"技术专家"——从技术可行性角度分析；角色C"风险官"——只关注风险点。每个角色独立输出，最后汇总给我。

### 原则8：给目标不给步骤

**说"完成X"，不要说"先做A再做B再做C"。**

这是最重要的原则之一。Anthropic的/goal模式证明——AI自己规划的执行路径，往往比人指定的步骤更优。人擅长定义"要什么"，AI擅长找到"怎么做到"。

- 错误："先打开这份文件，找到营收数据，然后和前年对比，再计算增长率，最后画个表"
- 正确："帮我分析这份财报的营收变化趋势，找出增长最快和最慢的业务线，用表格呈现"

检查一下你今天给AI的指令——有没有"怎么做"的内容？试着删掉它们，只留"要什么结果"。

### 原则9：规划与执行解耦

**重要任务先让AI出规划→你审阅→AI按批准的计划执行。**

Anthropic的Ultraplan模式将规划和执行分离。规划阶段可以反复修改，确认后再执行——避免方向性错误。

> 我需要写一份向领导汇报AI产业链投资机会的报告，约3000字。先不要写正文。先列出：1. 报告的整体结构 2. 每个章节要覆盖的要点 3. 需要我补充什么信息。我确认结构后，你再开始写。

## 第3层：建立质量保障（原则10-14）

> 这一层解决：怎么确保AI的产出靠谱。答案是——自查+交叉审查+版本控制，三层网。

### 原则10：生成后必自查

**每次AI产出后，追加一句"检查你的工作"，让它自己找问题。**

这是最简单的质量提升手段，只需多输入一行字：

> 报告写完了。现在检查你的工作：1. 数据是否准确——所有数字和原文核对一遍 2. 逻辑是否通顺——每段的结论有没有依据支撑 3. 格式是否一致——标题层级、表格样式、术语用法 4. 有没有遗漏——对照我最初的要求逐条核对

### 原则11：多Agent交叉审查

**让另一个AI（或新对话中的AI）审查第一个AI的产出。**

Anthropic用多个独立Agent交叉审查代码。自己写的代码，另一个Agent来审查——发现率远高于自审。

开一个新对话窗口，输入：

> 我有一份AI生成的报告，你是独立审查员。请审查这份报告的：1. 事实准确性 2. 逻辑漏洞 3. 改进建议。[附上报告]

### 原则12：独立评分——质检Agent不知道生成细节

**让AI用你定义的评分标准独立评估产出，不给它看生成过程。**

Anthropic的Outcomes系统——质检Agent不知道生成Agent怎么做的，只能看到最终结果。这种"盲评"防止了"既当运动员又当裁判"。

> 请用以下标准给这份报告打分（1-5分）：信息完整性、逻辑清晰度、可操作性、格式规范性。不要评价报告好不好，只打分+每项给一句理由。[附上报告，不附任何上下文]

### 原则13：失败自动修复

**AI发现错误后，让它自己修复，不要把错误丢回给你。**

> 这份数据分析和原文有几个数字不一致，不要告诉我让我改。你自己找出所有不一致的地方，逐个修正，修正后重新核对一遍。最后给我一个"修正清单"。

### 原则14：版本控制一切

**AI产出的重要文档都纳入版本管理，可追溯、可回滚。**

Anthropic的AI每次产出都git commit。什么时候改了、改了什么、谁改的——全部可追溯。

## 第4层：启动持续进化（原则15-18）

> 这一层解决：怎么让AI越用越好。答案是——反思、记忆、管理、进化。

### 原则15：AI需要反思时间

**定期让AI回顾历史对话，找模式、找矛盾、找可以进化的地方。**

Anthropic的Dreaming功能——AI在两次工作间隙回顾过去100个会话，清理冲突、合并重复、发现规律。灵感来自人类REM睡眠。

> 回顾我们今天所有的对话。告诉我：1. 我反复问了什么类型的问题？ 2. 我对你的哪些回答不满意？为什么？ 3. 有没有我发现不了但我应该注意的模式？

周期：每天花2分钟做快速回顾，每周花15分钟做深度回顾。

### 原则16：长期记忆 > 短期上下文

**把重要信息写入AI能记住的地方，对话结束知识不丢失。**

Anthropic的MEMORY.md系统——项目规范、个人偏好、经验教训都持久化。每次新对话，AI先读记忆，不用从头解释。

起步模板：

> 我是[身份]，我的工作习惯：写报告先要提纲确认后动笔；数据要表格呈现，结论要一句话概括；格式要求A4正文小四/12pt。

### 原则17：Agent是需要管理的持续实体

**不把AI当"用完即弃"的工具。把它当成需要持续管理和优化的团队成员。**

- 监控：每天早上检查一下定时Agent是否正常产出
- 评估：每周评估一次——产出质量是上升还是下降？
- 进化：发现模式后优化指令——"经过这一周，我把日报的格式更新为..."

### 原则18：从使用AI到AI自进化

**终极状态——AI自己发现问题、自己改进、自己验证。人只需要设定方向。**

这是Anthropic飞轮的终点。Dreaming让AI回顾+Outcomes让AI自评+Multiagent让AI协作——三者结合，AI不再依赖人推动。

| 信号 | 说明 |
|------|------|
| AI发现了一个你自己没注意到的问题 | AI的观察力开始超过你 |
| AI主动建议"我们可以这样改进流程" | AI开始思考"怎么做更好" |
| 你早上看AI昨晚的产出，只修改了10% | AI的交付质量达到你的90分标准 |
| 你三天没用AI，回来发现它一直在按规则自动工作 | AI不需要你推了 |

**不需要着急到达。先稳稳走完前三层，这一层是自然发生的。**

## 飞轮启动：你今天就能开始的行动

Anthropic的AI飞轮是：

> 给AI完整上下文+目标 → 拆解分发多Agent并行 → 质量保障三层网 → 回顾反思进化 → 下次更好

| 时间 | 行动 | 对应原则 | 耗时 |
|------|------|---------|------|
| 今天 | 写一份"关于我的AI说明书"保存为AI记忆 | 原则2+16 | 10分钟 |
| 今天 | 下一次不给AI步骤，只给目标 | 原则8 | 即时 |
| 今天 | 让AI做完一件事后追加"检查你的工作" | 原则10 | 10秒 |
| 明天 | 开两个对话窗口，一个写一个审 | 原则11 | 5分钟 |
| 本周 | 设一个每日定时任务让AI自动执行 | 原则4 | 3分钟 |
| 本周 | 让AI回顾你本周的使用模式 | 原则15 | 5分钟 |

**不要等"学完再做"。从现在开始，每一次和AI对话都多做一个动作——多给一点上下文、多让AI自检一次、多拆一个子任务。一个月后，你回头看，会发现你和AI的协作方式已经彻底变了。**

## 使用红线

| 红线 | 说明 |
|------|------|
| 不输入密码/银行卡号/身份证号 | AI对话不是保险箱 |
| 不把AI的回答当权威 | AI会犯错、会编造、会过于自信。关键信息必须核实 |
| 不在情绪极端时做决策 | 不要因为AI说"这个方案很好"就放弃自己的判断 |
| 不替代专业判断 | 医疗、法律、重大财务决策——AI是参考不是替代 |
| 不让Agent越过你的决策权 | Agent可以执行，但批准权永远在你手里 |
