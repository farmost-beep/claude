# 社交关系AI管家 — MVP产品说明书

> 版本 v0.1 | 2026-06-13 | 状态：概念验证阶段

---

## 一、产品定位

**一句话定义**：一个本地运行的AI插件，自动追踪你所有社交关系中的"最近干了什么"和"下一步该干什么"，帮你拟好消息，推到该去的地方。

**目标用户**：需要维护大量社交关系的职场人（尤其适合：银行客户经理、创业者、销售、管理者）

**核心承诺**：不改变你的社交习惯，只是在背后帮你记住。

---

## 二、MVP功能清单

### 2.1 必须有的（P0 — MVP核心）

| 功能 | 说明 | 验收标准 |
|:----|:-----|:--------|
| **联系人管理** | JSON配置联系人姓名/关系类型/标签 | 增删改查可用 |
| **时间线记录** | 每条互动记录：日期/平台/摘要/联系人 | 可手动添加、查看、搜索 |
| **待办提取** | 从时间线中标记"待跟进"事项 | 待办列表可见 |
| **AI拟稿** | 根据联系人+上下文生成消息草稿 | 生成文本可编辑可发送 |
| **微信推送** | 提醒推送+消息发送到个人微信 | 通过现有桥接API |

### 2.2 建议有的（P1 — 体验完整）

| 功能 | 说明 | 优先级 |
|:----|:----|:------|
| **关系图谱** | 按关系类型/强度/标签筛选 | 中 |
| **提醒队列** | P0(逾期)/P1(14天未联系)/P2(30天未联系) | 高 |
| **消息模板** | 支持自定义语气风格（正式/亲切/简洁） | 中 |
| **数据导出** | JSON导出/导入 | 低 |

### 2.3 暂不做的（P2 — 未来迭代）

- 消息自动抓取（需用户授权各平台）
- 关系强度自动计算
- 多端同步
- 仪表盘分析

## 三、技术架构

### 3.1 整体架构

```
┌──────────────────────────────┐
│  CLI / Skill 交互层           │
├──────────────────────────────┤
│  核心引擎（Python）            │
│  ├─ 联系人管理                │
│  ├─ 时间线引擎                │
│  ├─ 待办队列                  │
│  ├─ AI拟稿（Claude API）      │
│  └─ 推送模块                  │
├──────────────────────────────┤
│  数据层（本地JSON/SQLite）     │
├──────────────────────────────┤
│  连接层                       │
│  ├─ 微信（桥接API）            │
│  └─ 飞书（cc-connect，预留）   │
└──────────────────────────────┘
```

### 3.2 数据模型

**联系人（contact.json）**
```json
{
  "id": "zhangzong",
  "name": "张总",
  "relation": "客户",
  "strength": 4,
  "tags": ["科技金融", "并购贷款"],
  "platforms": {"weixin": "zhang123", "feishu": ""},
  "notes": "通过李会长认识，对并购贷感兴趣"
}
```

**时间线（timeline.json）**
```json
{
  "id": "t-001",
  "date": "2026-06-10",
  "contact": "zhangzong",
  "platform": "微信",
  "summary": "项目验收讨论，他说下周等通知",
  "pending": "等张总通知后跟进验收结果",
  "type": "conversation"
}
```

**待办（todos.json）**
```json
{
  "id": "todo-001",
  "contact": "zhangzong",
  "due": "2026-06-17",
  "task": "跟进项目验收结果",
  "priority": "P0",
  "status": "pending",
  "source": "2026-06-10 微信对话"
}
```

### 3.3 数据流

```
手动添加/导入联系人 → 手动记录时间线 → AI自动提取待办
                                          ↓
                                  提醒队列触发
                                          ↓
                                  AI生成消息草稿
                                          ↓
                                  用户确认 → 推送微信
```

## 四、交互设计

### 4.1 CLI命令（MVP交互方式）

```
# 添加联系人
social add-contact 王教授 --relation 导师 --tags "学术,引荐"

# 记录互动
social log 王教授 "聊了引荐的事，同意安排时间"

# 查看待办
social todos
→ P0 张总 跟进攻城项目验收结果 逾期2天
→ P1 王教授 安排引荐时间 14天未联系

# 拟稿
social draft 王教授
→ "王教授好，上次聊到引荐的事，您最近方便安排时间吗？"

# 查看所有联系人状态（仪表盘）
social dashboard
```

### 4.2 关键词触发

在微信中通过关键词触发查询：
```
"帮我看看王教授最近的情况" 
→ AI查询时间线 → 返回摘要

"给张总拟一条消息" 
→ AI提取上下文 → 生成草稿
```

## 七、价值评估

### 对个人的价值

| 场景 | 原来 | 用了AI管家 |
|:----|:----|:----------|
| 客户跟进 | 翻聊天记录，硬想上次说了什么 | 一看就知道"项目验收等通知" |
| 节假问候 | 群发，谁都看得出是群发 | AI根据最近互动拟个性化消息 |
| 跨平台 | 微信/飞书/邮件来回切 | 统一时间线，不分平台 |
| 遗忘 | 半年没联系的老朋友彻底断了 | 30天提醒一次，关系不凉 |

### 与现有方案的区别

| 对比 | 手机通讯录 | CRM软件 | 本方案 |
|:----|:---------|:-------|:------|
| 记录了什么 | 电话号码 | 销售漏斗 | **人际关系的完整记忆** |
| 谁在用 | 所有人 | 销售人员 | **每个人** |
| 主动性 | 无 | 手动提醒 | **AI主动提醒+拟稿** |
| 隐私 | 本地 | 上云 | **本地优先** |

## 八、风险与约束

| 风险 | 应对 |
|:----|:-----|
| 用户不愿手动记录时间线 | MVP验证期手动，V1.5自动抓取 |
| AI拟稿语气不对 | 支持自定义风格+用户确认后才发送 |
| 数据隐私担忧 | 全部本地存储，不上云，可随时删除 |
| 跨平台授权复杂 | MVP只做微信，飞书/邮件后续迭代 |
