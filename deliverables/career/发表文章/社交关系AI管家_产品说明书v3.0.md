> Agent原生 · 微信生态深度嵌入 | 2026-06-13 | 入口即聊天

## 产品定位

**一句话定义**：一个住在你微信里的AI管家——你不需要学任何东西，跟它聊天就够。

**使用方式**：在微信里像跟朋友聊天一样跟它说话，它自动帮你打理所有人际关系。

---

## 一、核心体验：Agent原生，入口即聊天

### 它主动找你（Proactive）

| AI行为 | 触发时机 |
|:------|:--------|
| "张总的TS进展到哪了，需要今天问问吗？" | 约定时间+2天 |
| "你和王教授14天没联系了，上次他同意引荐投资人" | 静默超过14天 |
| "明天下午3点跟李总在国贸见面，这是上次聊的内容" | 会面前一天 |
| "检测到你刚给张总发了消息，要记录这条互动吗？" | 发送消息后自动检测 |

### 你找它说话（Reactive）

跟它聊天就像跟一个特别细心的助理说话：

```
你：张总最近啥情况
🤖：6月10日在国贸聊了TS，法务那边基本没问题了。
   现在等他们出最终条款。建议周五问问进展。

你：帮我拟条消息问问
🤖：张总，TS那边法审有结果了吗？有啥需要我配合的随时说。
    → 发 | 改一下 | 算了
```

### 不需要任何命令

没有 `/` 开头的指令，没有按钮，没有菜单。**对话即操作**：

```
"记一下：和张总聊了TS" → AI自动记录+提取待办
"最近有啥要跟进的"   → 显示待办列表
"帮我想想该联系谁"   → 分析冷却关系+推荐
"给王教授拟个消息"   → 生成→确认→发送
```

## 二、自动导入微信通讯录

首次使用零配置：

```
1. 用户通过某个入口添加AI管家为微信好友
2. AI自动扫描用户的微信通讯录
3. 按群聊频率/私聊频率排序，推荐"哪些人值得关注"
4. 用户勾选确认（或一键全选）
5. AI开始为每个关键联系人建立档案
```

**自动提取的信息**：
- 联系人备注/标签
- 最近聊天摘要（仅限用户已参与的对话）
- 群聊中提及的关键信息
- 共同群聊和共同联系人（识别关系网络）

**隐私边界**：
- 只读取用户**已参与**的对话上下文
- 不扫描用户未参与的群聊消息
- 所有数据本地处理
- 用户可以随时删除某个联系人的全部记录

## 四、对话交互设计

### 4.1 主动提醒（AI主动找用户）

用户每天收到2-3条来自AI管家的消息，不多不少。

```
09:00 ── ☀️ 今日概览
  今天有1个待办：跟进张总TS进度
  14天没联系的：王教授（上次聊了引荐）
  今日天气：晴，宜出门谈事

14:00 ── 💡 建议联系
  检测到你下午在国贸附近，张总公司也在国贸。
  要不要顺便问问TS进展？

21:00 ── 📝 今日回顾
  今天记录了3条互动，2个新待办。
  需要我帮你总结今天的社交成果吗？
```

### 4.2 被动应答（用户主动找AI）

用户发什么，AI都能理解：

```
用户说的话                      AI的理解和反应
──────────────────────────────────────────────────
"张总最近怎么样"                查询时间线→生成摘要
"记一下：和李总吃了顿饭"         提取关键信息→存入时间线
"明天见王教授要注意什么"         调取历史记录→生成注意事项
"帮我约张总周五下午"             生成消息草稿→用户确认→发送
"最近是不是冷落了谁"             分析所有关系→列出冷却列表
"这事儿你怎么看"                 分析上下文→给出建议（看场景）
```

### 4.3 语音支持

语音消息同样支持：

```
用户发语音："记一下，张总说下周二签合同"
AI回复文字：✓ 已记录。张总，下周二签合同。周二早上我会提醒你。

用户发语音："最近忙啥呢"  
AI回复文字：最近完成了3次关键沟通，还有2个待办……
```
