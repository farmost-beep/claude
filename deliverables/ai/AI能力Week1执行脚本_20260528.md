## Day 3 — 6/3 周三：Agent工具使用

### 学习目标
- 理解Agent与直接工具调用的本质区别
- 知道什么时候启动Agent、什么时候自己直接调用工具
- 掌握并行Agent的触发方式和场景

### 任务（45分钟）

**步骤1：理解Agent vs 直接工具（10分钟）**

在Claude Code中做以下对比思考（口头即可）：

```
场景A：你让Claude Code"读取文件A，提取所有数字，写入文件B"
→ 直接工具调用：Read → 你自己看 → Write
→ Agent模式：Agent自主 Read → 处理 → Write

场景B：你让Claude Code"从300篇研报中找出关于AI信贷的论述，生成摘要报告"
→ 直接工具调用：不现实（无法手动处理300篇）
→ Agent模式：Agent分批次Read → 提取 → 汇总 → Write
```

判断标准（写在今日收获里）：
- 任务步骤 <= 3且不依赖上一步结果 → 直接工具调用
- 任务步骤 > 3 或需要自主规划 → Agent模式
- 需要并行处理多个独立任务 → 并行Agent

**步骤2：单Agent练习——投资组合分析（20分钟）**

在Claude Code中执行以下指令（复制粘贴即可）：

```
用Agent模式完成以下投资组合分析任务：

1. 读取 deliverables/investment/ 目录下所有.md文件
2. 从中提取以下信息：
   - 所有出现的股票名称和代码
   - 所有出现的数字型数据（如持仓数量、成本价、目标价等）
   - 所有出现的投资策略或原则表述
3. 按以下结构汇总，写入 deliverables/ai/Week1-Day3-Agent分析结果.md：

   # 投资组合Agent分析结果
   
   ## 覆盖的股票
   （列表，含出现位置）
   
   ## 关键数据点
   （按文件分组的数字型数据表格）
   
   ## 投资策略摘要
   （提取的策略表述）

要求：Agent自主完成文件读取、数据提取和写入，不需要一步步指导。
```

**验收要点**：
- 输出文件存在且包含三个章节
- 覆盖了deliverables/investment/下至少2个文件
- Agent没有中途停下来问"接下来做什么"

**步骤3：并行Agent练习——同时推进两个目标（15分钟）**

在Claude Code中执行：

