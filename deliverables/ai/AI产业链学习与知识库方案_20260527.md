# AI产业链学习与知识库方案

> 面向陈颖芳 | 版本：v1.0 | 2026-05-27
> 核心原则：不是学AI技术细节，是学"AI商业判断力"——这是科技金融事业的核心竞争力

---

## 0. 为什么AI产业链洞察是科技金融从业者的核心能力

### 0.1 你的工作场景决定你需要什么

作为邮储银行科技金融事业部的高级副经理，你日常面对的真实问题是：

- 一家AI芯片企业来申请授信，你能判断它的技术路线是主流还是偏门吗？
- 一家大模型创业公司估值30亿，但收入几乎为零，它的商业模型说得通吗？
- 一家声称"用AI改造传统制造业"的企业，它的技术是真AI还是伪AI？
- 监管要求收紧AI信贷风险敞口，哪些AI细分领域风险最高？

这些问题不是靠读技术论文能回答的，而是靠**理解AI产业链各环节的商业逻辑**来回答的。

### 0.2 AI产业链的五层结构

```
┌─────────────────────────────────────────────────────┐
│ 第5层：智能体层（Agent）                              │
│ 自主规划、执行、迭代的AI系统                          │
│ 代表：Manus、Devin、Claude Code Agent               │
│ 商业关键词：任务完成率、工具调用能力、人机协同模式     │
├─────────────────────────────────────────────────────┤
│ 第4层：应用层（Application）                         │
│ AI在各行业的具体落地产品                              │
│ 代表：AI客服、AI信贷审批、AI医疗影像、AI编程助手      │
│ 商业关键词：场景适配度、用户留存、付费意愿、替换成本   │
├─────────────────────────────────────────────────────┤
│ 第3层：模型层（Model）                               │
│ 大语言模型、多模态模型、垂直领域模型                   │
│ 代表：GPT-5、Claude、Gemini、文心一言、DeepSeek      │
│ 商业关键词：训练成本、推理成本、模型能力边界、定价策略  │
├─────────────────────────────────────────────────────┤
│ 第2层：算力层（Compute）                             │
│ 数据中心、云计算、算力租赁                             │
│ 代表：AWS/Azure/阿里云/华为云、CoreWeave、算力租赁平台  │
│ 商业关键词：GPU利用率、算力单价、电力成本、资本开支     │
├─────────────────────────────────────────────────────┤
│ 第1层：芯片层（Chip）                                │
│ GPU、TPU、NPU等AI加速芯片                             │
│ 代表：英伟达、AMD、华为昇腾、寒武纪、摩尔线程          │
│ 商业关键词：制程工艺、显存带宽、生态壁垒(CUDA)、制裁风险 │
└─────────────────────────────────────────────────────┘
```

这五层之间的关系是：**下层是上层的约束条件，上层是下层的价值出口**。

- 英伟达芯片的产能，决定了全球大模型公司的训练速度
- 模型能力的上限，决定了AI应用能做多"智能"
- Agent的可靠性，决定了AI能在企业场景中承担多少责任

### 0.3 你的学习目标（与程序员不同）

| 程序员学AI | 你学AI |
|-----------|--------|
| 怎么训练一个模型 | 这家模型公司的训练成本合理吗？ |
| 怎么优化推理速度 | 推理成本下降对哪些商业模式是利好？ |
| 怎么写Agent代码 | 这家公司的Agent产品能替代多少人工？ |
| 怎么调参 | 这家AI芯片公司的客户为什么选它不选英伟达？ |
| 看论文 | 看财报、看招股书、看行业研报 |

**你的目标不是"会做AI"，而是"能判断一家AI公司值不值得投/贷"。**

## 1. AI产业链五层全景：非程序员视角

### 1.1 每层的"一句话"

| 层级 | 一句话 | 对金融从业者的意义 |
|------|--------|-------------------|
| 芯片层 | AI计算的"发动机"，谁造发动机谁说了算 | 判断硬件企业的技术壁垒和制裁风险 |
| 算力层 | AI计算的"电厂"，模型训练和运行都需要电 | 理解云服务商和算力租赁的商业模式 |
| 模型层 | AI的"大脑"，决定AI能有多聪明 | 判断模型公司的技术实力和商业可持续性 |
| 应用层 | AI的"手和脚"，在各行各业落地干活 | 评估一家企业"用AI"是真需求还是蹭概念 |
| 智能体层 | AI的"自主行动能力"，从工具到代理的跃迁 | 判断下一代企业级AI的方向和泡沫 |

### 1.2 五层之间的价值流动

```
芯片（英伟达H200）→ 毛利率70%+
    ↓ 卖给
算力（阿里云/火山引擎）→ 毛利率30-40%
    ↓ 租给
模型（DeepSeek/月之暗面）→ 毛利率？（大多数为负）
    ↓ 支撑
应用（AI客服/AI编程）→ 毛利率50-80%
    ↓ 进化为
Agent（自主执行任务）→ 尚未形成稳定商业模式
```

**关键观察**：当前AI产业链的利润集中在最底层（芯片）和最高层（应用），中间层（模型）在烧钱抢市场。这个利润结构会变化，变化的方向决定哪里有投资机会。

### 1.3 中国vs全球的差异

| 层级 | 全球 | 中国 | 差异影响 |
|------|------|------|---------|
| 芯片 | 英伟达垄断（90%+市占） | 华为昇腾/寒武纪追赶中，但受制裁制约 | 中国AI芯片企业面临"技术差距+产能受限"双重压力 |
| 算力 | AWS/Azure/GCP三强 | 阿里云/华为云/火山引擎，国资云崛起 | 中国算力市场有"国产替代"政策红利 |
| 模型 | OpenAI/Anthropic/Google领先 | DeepSeek/智谱/月之暗面/阶跃星辰等"六小虎" | 中国模型公司在成本控制上有优势，但在前沿能力上仍有差距 |
| 应用 | 企业级AI（Salesforce、Microsoft Copilot） | 面向C端超级APP（微信/抖音）+ G端（政务AI） | 中国应用层的落地场景与西方不同，G端是大客户 |
| Agent | 仍在早期，尚未有杀手级产品 | 钉钉AI Agent、Coze（字节） | Agent在中国的企业协同场景可能更快落地 |

## 2. 分层学习路径：非程序员怎么学AI产业链

> 每层学习包含四要素：学什么（知识清单）、怎么学（方法）、时间投入、判断力锚点（学完后能回答什么商业问题）

### 2.1 芯片层（第1-2周）

**学什么**

| 知识点 | 为什么要知道 | 深度要求 |
|--------|------------|---------|
| GPU vs CPU：为什么AI计算用GPU不用CPU | 理解英伟达的护城河本质——不是芯片快，是CUDA生态锁定了开发者 | 能用类比讲清楚（GPU=1000个小学生同时算加减法，CPU=1个数学教授） |
| 英伟达的产品线和迭代节奏 | 判断AI芯片行业的竞争节奏——英伟达每年一代，追赶者跟不上 | 知道H100/H200/B200的区别（显存、带宽、功耗），能用一张表对比 |
| CUDA生态壁垒 | 这是信贷判断的核心——一家AI芯片公司的芯片参数再好，如果没有软件生态（类似Windows/安卓），客户不会用 | 能解释"为什么寒武纪的芯片性能不差但客户少" |
| 美国对华芯片制裁的演变 | 直接影响你评估任何中国AI硬件企业的前景 | 知道三批制裁（2022.10、2023.10、2024.3）的核心限制内容 |
| 华为昇腾 vs 寒武纪 vs 英伟达中国特供版 | 这是信贷评审中最常见的"三家对比"场景 | 能列出一张对比表：制程/显存/生态/客户/制裁风险 |
| 芯片制程（7nm/5nm/3nm）的生意含义 | 知道"几纳米"不重要，重要的是"谁有能力生产几纳米"——全球只有台积电和三星 | 能理解"制程=精度，精度越高芯片越快越省电，但建厂成本是指数级" |

**怎么学**

1. **看3篇行业解读**（而非技术论文）：
   - 找一篇讲"英伟达为什么垄断"的财经分析文章
   - 找一篇讲"华为昇腾能不能替代英伟达"的对比文章
   - 找一篇讲"美国芯片制裁对中国AI产业影响"的时间线文章
   - 用Perplexity搜索：`NVIDIA competitive moat CUDA ecosystem explained for investors`

2. **对比3家公司的财报关键数字**：
   - 英伟达：毛利率70%+、数据中心收入增速、中国市场收入占比
   - 寒武纪：收入规模、亏损额、研发费用占比、政府补贴占比
   - 华为昇腾：公开信息有限，但能看出它不走独立上市路线，是华为体系内的战略业务

3. **做一个"芯片公司授信评估"模拟练习**：
   - 假设寒武纪来申请授信，你会关注哪5个问题？把它们写下来，然后让Claude Code帮你回答

**时间投入**：2周，共约4-5小时
- 第1周：阅读材料 + 理解GPU/CPU差异 + 理解CUDA生态（2-2.5小时）
- 第2周：芯片制裁政策梳理 + 三家对比表 + 模拟授信评估（2-2.5小时）

**判断力锚点**（学完后能回答的问题）
- "这家AI芯片公司的产品对标的是英伟达哪款芯片？差距是性能还是生态？"
- "它的产能受制裁影响吗？有没有被列入实体清单？"
- "它的客户是真的需要它的芯片，还是因为买不到英伟达才用它？"——这个问题区分了"真实需求"和"被动替代"，后者风险更高

### 2.2 算力层（第3-4周）

**学什么**

| 知识点 | 为什么要知道 | 深度要求 |
|--------|------------|---------|
| 训练 vs 推理：两种算力需求的区别 | 训练=造发动机（一次性大量投入），推理=开车（持续消耗）。不同商业模式对应不同算力需求 | 能举例：OpenAI训练GPT-5花了十几亿美元（训练），但每次你问ChatGPT一个问题只花0.01美元（推理） |
| 云算力的单位经济学 | 这是判断算力租赁公司商业可行性的核心：一块H100买来3万美元，租出去每小时多少钱？多久回本？ | 能算一笔"H100回本账"——购入成本/租出价格/利用率/回本周期 |
| 中国云服务市场的竞争格局 | 阿里云/华为云/火山引擎/腾讯云的优劣势 | 知道排名、各自绑定的模型公司（如火山引擎→字节的豆包）、政府客户占比 |
| 算力租赁 vs 自建数据中心 | 什么情况下租？什么情况下建？这个判断影响企业的资本开支结构 | 能解释"大模型公司前期租算力（灵活），训练出好模型后可能自建（降本）" |
| "东数西算"和国产算力替代 | 这是中国特有的政策变量：国家在西部建数据中心，要求政府/国企优先用国产算力 | 理解政策红利→国产算力产业链受益；理解政策限制→纯商业算力商在G端市场受限 |

**怎么学**

1. **算一笔真实的经济账**：
   - 让Claude Code帮你搜"2025年H100云租赁价格"和"H100采购价格"
   - 算：一块H100买3万美元，租出去每小时2-3美元，利用率60%，多久回本？
   - 理解：利用率从60%掉到40%，回本周期翻倍——这就是算力租赁的核心风险

2. **看2家公司的模式对比**：
   - CoreWeave（美国的GPU云）的招股书节选（如果公开的话），看它的收入结构、客户集中度、负债率
   - 阿里云的年报节选——看云业务收入增速和利润率

3. **画一张"算力采购决策树"**：
   - 一家AI创业公司，什么时候该租云算力？什么时候该自建集群？
   - 这个决策树，你以后在尽调中可以直接问企业："你们的算力是租的还是建的？为什么？"

**时间投入**：2周，共约4小时
- 第3周：理解训练vs推理 + 算力经济账 + 云市场格局（2小时）
- 第4周：算力租赁vs自建 + 国有算力政策 + 决策树绘制（2小时）

**判断力锚点**
- "这家公司的算力成本占总成本的比例是多少？是偏高还是行业平均水平？"
- "它绑定了哪个云厂商？如果换云厂商，迁移成本高吗？"
- "如果美国进一步限制中国云厂商获取英伟达芯片，这家公司的算力来源会受影响吗？"

### 2.3 模型层（第5-7周）

这是五层中**最需要深度理解的一层**，因为"大模型"是当前AI浪潮的核心，也是信贷评审中最常遇到的企业类型。

**学什么**

| 知识点 | 为什么要知道 | 深度要求 |
|--------|------------|---------|
| 大模型的基本工作原理（非技术版） | 理解"模型的智能来自哪里"——本质上是从海量文本中学习模式和关联，不是真正的"理解" | 能用一个比喻讲清楚：大模型=一个读过全互联网所有文字的人，他非常擅长"预测下一句话应该说什么"，但他没有真正的"理解" |
| 开源模型 vs 闭源模型的商业模式 | 这是判断模型公司投资价值的关键分歧：开源派（Meta Llama、DeepSeek）认为模型本身不值钱，值钱的是生态和应用；闭源派（OpenAI、Anthropic）认为顶尖模型稀缺，值得付费 | 能列出双方的核心论点和适用条件 |
| 模型能力的评估维度 | 别人说"这个模型很强"，你会追问"什么维度强"：推理能力？代码能力？中文能力？多模态？ | 知道几个主流评测基准（MMLU、HumanEval、C-Eval）分别测什么 |
| 模型训练的成本结构 | 判断一家大模型公司"烧钱速度"合理与否的核心 | 知道：训练一次大模型要几千万到几亿美元（主要是GPU算力+电力+数据成本），推理每次几分钱 |
| 中国"大模型六小虎"竞争格局 | 信贷评审中最常遇到的企业类型：零一万物/智谱AI/月之暗面/阶跃星辰/MiniMax/百川智能 的各自定位和差异化 | 能填一张对比表：创始人背景/主打模型/融资额/估值/收入模式/主要客户 |
| DeepSeek现象：低成本模型的冲击 | 2025年初DeepSeek用极低成本训练出接近GPT-4水平的模型，这个事件改变了整个行业的成本预期 | 理解为什么DeepSeek的案例让"模型层利润薄"的预判加速——既然训练门槛在降低，模型的稀缺性就在下降 |
| 模型公司的收入从哪里来 | API调用费？企业定制？C端订阅？目前大多数中国模型公司还没有稳定的收入模式 | 能分析一家模型公司的收入结构，判断它是"真收入"还是"烧融资的钱" |

**怎么学**

1. **亲自对比3个模型回答同一个问题**：
   - 用一个你熟悉的金融问题（如"分析科创50指数的投资价值"），分别问Claude、ChatGPT、DeepSeek（如果可访问）
   - 比较三个回答的质量、深度、专业度差异——这是最直观的"模型能力"感知方式

2. **用"公司对比表"代替论文阅读**：
   - 搜"中国大模型六小虎2025年对比"或类似文章
   - 自己填一张表：公司/创始人/技术路线/融资/估值/收入/客户/竞争壁垒
   - 注：这类信息变化极快，表是活的，每季度更新一次即可

3. **做一次"大模型公司授信评估"**：
   - 选一家公开融资较多的大模型公司（如月之暗面或智谱）
   - 用你的银行信贷框架问：它有持续经营能力吗？（收入）它的技术会被替代吗？（壁垒）它的烧钱速度能撑到盈利吗？（现金流）
   - 你会发现：大多数中国大模型公司按银行信贷标准，目前是通不过的——但这就是"科技金融"的特殊性：要用不同的框架评估

**时间投入**：3周，共约6-7小时（这是投入最大的一层）
- 第5周：模型工作原理 + 开源vs闭源 + 模型评估维度（2-2.5小时）
- 第6周：成本结构 + 中国六小虎对比 + DeepSeek案例分析（2-2.5小时）
- 第7周：收入模式分析 + 模拟授信评估 + 汇总（2小时）

**判断力锚点**
- "这家模型公司的技术路线是追随还是原创？追随者估值应该打折"
- "它的模型在哪个维度上有差异化？——如果只是在评测基准上'接近GPT'，那差异化为零"
- "它的收入是从模型能力直接来的（API费），还是从模型生态间接来的（应用分成）？——前者天花板低，后者天花板高但还不成立"
- "按它当前的烧钱速度，现有融资能撑几个月？下一轮融资在什么时候？"

### 2.4 应用层（第8-9周）

应用层是"AI商业价值"最直接体现的地方，也是你作为银行从业者最能发挥判断力的地方——因为"行业+AI"的场景判断，需要用行业经验。

**学什么**

| 知识点 | 为什么要知道 | 深度要求 |
|--------|------------|---------|
| AI应用的四类商业模式 | 按token收费、按效果收费、SaaS订阅、项目制——不同模式对应不同的收入质量和可预测性 | 能分析一个AI应用属于哪一类，这一类的毛利率范围是多少 |
| 银行业AI应用全景 | 这是你的主场——智能客服、信贷审批、反洗钱、合规审查、智能投顾……你每天都在这个行业里 | 能把银行AI应用按"成熟度"和"投入产出比"两个维度分类；能判断哪些是"真需求"哪些是"伪需求" |
| AI应用的"护城河"来源 | 数据壁垒、客户切换成本、行业know-how、网络效应——这四点决定了AI应用能不能持续赚钱 | 用巴菲特的护城河框架分析AI应用公司 |
| C端AI应用 vs B端AI应用 | C端（用户付费）：天花板高但留存难；B端（企业付费）：单价高但销售周期长。中国和西方的差异很大 | 能解释为什么中国AI应用在B端G端有优势，在C端面临字节/腾讯/阿里等平台的竞争 |
| 判断一家公司是"真AI"还是"伪AI" | 这是信贷审批中最实用的能力——很多传统企业把"AI"当融资噱头 | 掌握三个鉴别标准：(1) AI是否核心业务流程 (2) 是否有专有数据训练 (3) AI替代的是成本还是创造了新收入 |

**怎么学**

1. **做一个"银行AI应用成熟度矩阵"**：
   - X轴：技术可行性（从"已成熟可用"到"5年之后才有可能"）
   - Y轴：业务价值（从"锦上添花"到"核心业务依赖"）
   - 把银行10+个AI应用场景填进去，这是你对银行业AI应用最系统的认知

2. **分析2家AI应用上市公司**：
   - 一家成功（如某家确实用AI大幅提升了效率的企业软件公司）
   - 一家失败/争议（如某家被质疑"伪AI"的公司）
   - 对比它们的收入结构、客户留存率、研发投入方向

3. **用"伪AI检测三问"测试3家公司**：
   - 选3家你因工作关系知道的企业（它们都声称"用AI"）
   - 用三个问题测试：(1) AI在核心业务流程中还是边缘功能？(2) 数据是它自己的还是调用别人的？(3) AI带来的价值是降本还是增收？降本有天花板，增收才有想象空间

**时间投入**：2周，共约4-5小时
- 第8周：四类AI商业模式 + 银行AI应用全景 + 银行应用成熟度矩阵（2-2.5小时）
- 第9周：AI应用护城河分析 + 真伪AI鉴别 + 上市公司案例分析（2-2.5小时）

**判断力锚点**
- "这家公司的AI用的是自己的数据还是公开数据？——自己的数据=壁垒，公开数据=人人能做"
- "如果GPT-6明天发布，这家公司的产品是受益还是被淘汰？——被淘汰说明它只是'套壳'，没有真正价值"
- "这家公司的AI产品被客户替换的成本高吗？——切换成本=真正的护城河"

### 2.5 智能体层（第10-11周）

这是最新、最前沿的一层，也是最不确定的一层。对金融从业者来说，不需要追最新的Agent产品，但需要理解Agent改变了什么。

**学什么**

| 知识点 | 为什么要知道 | 深度要求 |
|--------|------------|---------|
| Agent是什么：和聊天机器人的本质区别 | Chatbot=你问它答（被动），Agent=你给目标它执行（主动）。这是从"工具"到"代理"的跃迁 | 能举例：ChatGPT给你写一封邮件大纲（被动），Agent帮你完成整个客户接待方案——订会议室、写议程、发邀请、准备PPT、跟进纪要（主动） |
| Agent的核心能力：规划、工具调用、记忆、执行 | 评估一家Agent公司的技术实力，就看这四项 | 能解释Manus为什么比普通AI助手"更像一个员工"——因为它有规划+工具调用+记忆+执行的完整链条 |
| Agent的失败模式 | 金融从业者最需要关注的：Agent不可靠会怎样？——在银行场景中，Agent犯错的代价远高于聊天机器人犯错 | 能解释"为什么AI信贷审批不能全权交给Agent"——不是技术问题，是责任归属问题 |
| Agent的商业化路径 | Agent-as-a-Service、Agent嵌入已有SaaS、Agent Platform——三种路径各有优劣 | 能分析一款Agent产品的商业模式属于哪种 |
| Claude Code Agent模式（已在使用） | 这是你理解Agent的第一手经验：你给Claude Code一个目标，它自主规划步骤、调用工具、生成结果 | 回顾你使用Claude Code Agent的经历，哪些任务Agent做得好，哪些需要你干预 |

**怎么学**

1. **用Claude Code Agent做3个不同类型的任务**（你已经有了使用基础，这步是"有意识地观察"）：
   - 任务1：数据获取型（"帮我搜一下最近AI行业的融资新闻"）
   - 任务2：分析决策型（"分析这只AI股票该不该卖"）
   - 任务3：创作交付型（"帮我写一份AI行业周报"）
   - 记录：Agent在哪类任务中表现好，哪类需要你频繁干预

2. **读2篇关键文章**：
   - Anthropic的"Building effective agents"博客（2024年底发布，是Agent领域的奠基性文章）
   - OpenAI或Google关于Agent的公开博客

3. **设计一个"你的工作中哪个流程可以用Agent替代"的场景**：
   - 比如：尽调准备Agent——输入企业名称→自动收集工商/司法/舆情→生成尽调前简报
   - 不要求真的实现，但要把流程画出来，标注哪一步AI能做、哪一步必须人做

**时间投入**：2周，共约3-4小时
- 第10周：Agent概念 + Agent四能力 + 失败模式（1.5-2小时）
- 第11周：Agent商业化 + Claude Code Agent实践观察 + 工作场景设计（1.5-2小时）

**判断力锚点**
- "这家Agent公司有没有自己的'护城河数据'？还是只是一个'套壳'调用别人的模型？"
- "它的Agent在真实场景中的完成率是多少？不是demo视频里的，是真实客户的使用数据"
- "如果Agent搞砸了，谁来承担责任？——这个问题目前没有法律答案，但银行不能赌"

## 5. Claude Code Agent编排模板（3个）

这三个Agent编排模板设计好之后，保存在 `deliverables/ai/agent_orchestration/` 目录下。使用时，将模板内容粘贴给Claude Code，即可触发对应的多Agent协作流程。

### 5.1 模板一：AI公司尽调Agent链

**适用场景**：你即将对一家AI企业做现场尽调或授信评审，需要快速生成一份多维度分析简报。

**触发指令**（粘贴给Claude Code）：

```
启动多Agent并行模式，对 [企业名称] 进行AI公司尽调分析。请同时启动以下6个Agent并行工作：

Agent 1（基本面分析）：搜索该企业的基本信息（成立时间、注册资本、股权结构、融资历史、估值变化）。重点标注：创始人背景（技术还是商业出身？连续创业还是首次？）、最近一轮融资的时间/金额/估值、主要股东及其退出记录。

Agent 2（技术评估）：基于公开信息，评估该企业在AI产业链中的位置（芯片/算力/模型/应用/Agent，属于哪一层？）。分析其技术路线的差异化程度——是与行业主流一致还是独辟蹊径？标注技术壁垒的可持续性。

Agent 3（商业验证）：分析该企业的商业模式——收入从哪里来？客户是谁？（B端大客户/G端/C端？）收入增长率、毛利率、客户集中度如何？如果查不到公开财务数据，请基于同行业上市公司做合理推估，并标注推估的不确定性。

Agent 4（竞争格局）：列出该企业在细分领域的主要竞争对手（国内外各列3家）。分析竞争格局——它是领先者、追赶者、还是差异化者？市场份额趋势如何？替代威胁来自哪里？

Agent 5（风险审查——信贷视角）：这是我的主场。请用银行授信审批的视角，审查该企业的五大风险：
1) 经营风险：客户集中度、单一行业依赖度
2) 财务风险：现金流状况（烧钱速度）、负债结构、下一轮融资窗口
3) 技术风险：技术路线被替代的可能性、核心技术人员的留存
4) 政策风险：是否受中美科技制裁影响？国内AI监管政策变化的影响
5) 市场风险：行业景气度变化、竞争加剧的可能性
每个风险给出"高/中/低"评级和一句话理由。

Agent 6（魔鬼代言人——反方视角）：专门找出不投/不贷给这家企业的理由。至少提5条有据可查的风险或缺陷。这是最重要的理性校验——其他Agent可能倾向于正面描述，你需要找出所有红灯信号。

所有Agent完成后，请汇总为一份"AI公司尽调简报"，结构如下：
1. 一句话结论（放在最前面：是否值得进入下一步深度尽调？）
2. 企业概览（Agent 1）
3. 技术评估（Agent 2）
4. 商业验证（Agent 3）
5. 竞争格局（Agent 4）
6. 风险审查（Agent 5，含五类风险评级）
7. 反方意见（Agent 6，必须列出至少5条）
8. 下一步建议（需要补充哪些信息？尽调中应重点关注哪三个问题？）

输出文件：deliverables/ai/agent_orchestration/outputs/[企业名称]_尽调简报_[日期].md
预计耗时：30-40分钟
```

**使用方式**：开新Claude Code对话，粘贴以上指令，将 `[企业名称]` 替换为实际企业名。注意：Agent并行执行需要Claude Code Agent模式的并行能力。

**适用企业类型**：芯片/算力/模型/应用/Agent各层的AI企业均适用。如果企业信息较少（如未上市未公开财务），Agent会自动标注信息缺失并给出推估。

### 5.2 模板二：AI行业周报Agent链

**适用场景**：每周一早上用15-20分钟生成一份AI行业动态速报，作为个人知识更新和团队分享的基础素材。

**触发指令**（粘贴给Claude Code）：

```
启动多Agent并行模式，生成本周AI行业动态周报（覆盖过去7天，即 [起始日期] 至 [结束日期]）。请同时启动以下4个Agent并行工作：

Agent 1（大模型动态）：搜索过去一周全球大模型领域的重大进展，重点关注：
- 新模型发布（哪家发了什么模型？能力提升在哪？）
- 模型评测变化（LMSYS排行榜有无变动？）
- 模型公司融资/估值变化
- 开源模型的重大更新

Agent 2（芯片与算力）：搜索过去一周AI芯片和算力领域的动态，重点关注：
- 英伟达/AMD/Intel的新品或战略动态
- 美国对华芯片制裁的新进展（如有）
- 华为昇腾/寒武纪/国产GPU的动态
- 云厂商算力价格变化
要求：至少找到3条有效信息，每条标注来源

Agent 3（AI应用与监管）：搜索过去一周AI应用落地和政策监管动态，重点关注：
- 银行业AI应用的新案例（国内外均可）
- AI+金融/科技金融相关的政策新规
- AI监管/数据安全/模型合规的新政策
- AI应用的重大商业合作/落地案例
要求：至少找到3条有效信息，每条标注来源

Agent 4（AI行业投融资）：搜索过去一周AI行业的重要投融资事件，重点关注：
- 融资超过1亿美元的交易
- 新晋独角兽（估值超10亿美元的未上市公司）
- AI上市公司的重大股价变动及原因
- 中国AI企业的融资/上市进展
要求：至少找到3条有效信息，每条标注来源

所有Agent完成后，请汇总为一份"AI行业周报"（总长度不超过1500字），结构如下：
1. 本周一句话总评（AI行业最重要的1-2个变化）
2. 本周AI大事速览（列表形式，每条不超过2行，标注来源）
3. 与科技金融业务的关系（这些动态对银行科技金融业务意味着什么？——2-3条）
4. 下周关注预告（根据本周动态，下周最可能出现什么变化？）

输出文件：deliverables/ai/agent_orchestration/outputs/AI行业周报_[日期].md
预计耗时：15-20分钟

注意：如果某Agent找不到足够信息（少于3条），注明"本周该领域无重大动态"即可，不要硬凑。
```

**使用方式**：每周一早上开新对话，粘贴以上指令，替换日期范围。建议与 `/loop 7d` 组合使用，设置为每周自动提醒。

### 5.3 模板三：AI技术评估Agent链

**适用场景**：面对一项AI技术或产品，需要快速判断它的"技术成熟度"和"商业可行性"。这是科技金融信贷评审中最核心的判断。

**触发指令**（粘贴给Claude Code）：

```
启动多Agent并行模式，对 [技术/产品名称] 进行技术成熟度与商业可行性评估。请同时启动以下5个Agent并行工作：

Agent 1（技术原理解析——用非技术语言）：
用金融从业者能理解的语言（类比、比喻、避免术语），解释这项技术/产品：
- 它做什么？（一句话）
- 它是怎么做的？（用类比解释技术原理，不超过200字）
- 它比现有方案好在哪里？（不是技术参数，是商业价值——更快？更便宜？更准？）
- 它的关键技术假设是什么？（什么条件不满足它就做不好？）

Agent 2（技术成熟度评估）：
用TRL（技术就绪度）框架评估：
- TRL 1-3：基础研究阶段（离商业应用5年以上）
- TRL 4-6：技术验证阶段（离商业应用2-5年）
- TRL 7-8：产品验证阶段（离商业应用1-2年）
- TRL 9：已进入商业部署
给出一个TRL评级，并说明依据。标注：达到下一级TRL需要克服的关键障碍是什么？

Agent 3（商业可行性评估）：
- 目标市场：市场规模（TAM/SAM/SOM，粗略估计即可）、增速
- 客户画像：谁会为这项技术/产品付费？付费意愿的证据是什么？
- 收入模式：可能的收费方式？毛利率预期？
- 竞争态势：已有替代方案是什么？为什么客户现在不用那些替代方案？
- 单位经济学：单个客户的获客成本 vs 生命周期价值的粗略估算

Agent 4（替代威胁分析）：
- 技术替代：未来3年内，是否有更优技术可能替代它？
- 大厂替代：如果腾讯/阿里/字节/华为决定做同样的事，它会怎样？
- 模型升级替代：如果底层模型能力大幅提升（如GPT-6），这项技术/产品是受益还是被淘汰？
- 政策替代：政策变化（如监管收紧、数据安全新规）会否使这项技术变得不可行？

Agent 5（信贷风险评估）：
用银行授信视角，评估如果一家以此为唯一核心业务的企业来申请授信：
- 持续经营能力：这项技术/产品能独立支撑一家公司的收入吗？
- 现金流预测：在什么时间点可能实现经营性现金流为正？
- 抵押/担保可能性：这家企业的核心技术/专利能作为授信的"软抵押"吗？
- 授信建议：如果必须给这家企业一个授信评级（AAA到B），给什么级别？为什么？

所有Agent完成后，请汇总为一份"技术评估报告"，结构如下：
1. 一句话评估结论
2. 技术是什么（Agent 1）
3. 离商业落地还有多远（Agent 2 + TRL评级）
4. 商业上说得通吗（Agent 3 + 单位经济学）
5. 会不会被替代（Agent 4 + 替代威胁排序）
6. 能贷/投吗（Agent 5 + 授信评级）
7. 需要进一步验证的3个关键问题

输出文件：deliverables/ai/agent_orchestration/outputs/[技术名称]_评估报告_[日期].md
预计耗时：30-40分钟
```

**使用方式**：遇到一项新AI技术/产品时，粘贴以上指令。或在对一家AI公司做尽调前，先对它的核心技术跑一遍这个评估。

## 9. 关键提醒

### 9.1 你不是在和程序员竞争

你不需要理解Transformer架构的数学公式。你需要理解的是：
- 英伟达的毛利率为什么是70%（硬件+生态的双重锁定）
- DeepSeek为什么只用600万美元就能训练出接近GPT-4的模型（MoE架构的技术选择+工程优化）
- 为什么大多数AI应用公司不赚钱（模型API成本高+客户获取成本高+替代成本低）

这些不是技术问题，是商业判断问题。**你的金融背景在理解这些问题上有天然优势**——算成本、算现金流、算竞争壁垒，这是你每天都在做的事。

### 9.2 学习节奏比学习速度重要

AI行业变化极快，任何一份"行业全景图"三个月后就过时了。你的目标不是"一次学会所有"，而是：
- 用五层框架把碎片信息装入结构化框架（信息来了知道放哪）
- 通过AI行业周报Agent链保持每周信息更新（不需要你手动搜）
- 每季度更新知识卡片中的关键数字（市场规模、估值、竞争排名）

**你不是在做一份"AI产业链百科全书"，你是在训练一种"AI商业判断直觉"。**

### 9.3 你的独特壁垒

大多数做科技金融信贷的人，有两类：
1. 金融出身，不懂技术→看AI企业时只能看财务，看不懂技术风险
2. 技术出身，不懂金融→看AI企业时只能看技术，看不懂现金流和信贷风险

你的路径是第三条：
- 金融背景（20年银行经验） + AI产业链商业理解（本方案） + AI工具熟练使用（Claude Code + Agent编排）= **能同时从信贷视角和技术视角评估AI企业**

这个组合在中国银行业科技金融领域极为稀缺。
