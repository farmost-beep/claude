## 零、一个前提：49岁+金融背景+AI工具=稀缺组合

大多数懂AI的人不懂金融业务，大多数懂金融的人畏惧AI。你站在交叉点上。

**AI不会取代银行家，但会用AI的银行家会取代不会用的。** 这不是五年后的事，是正在发生的事。

| 大多数同龄同行 | 你的起点 |
|-------------|---------|
| 漠视："离我还远" | 中科大理科训练 → 对技术有底层理解力 |
| 焦虑但不动："知道重要但不知从何下手" | 20+年银行业经验 → 知道业务痛点在哪 |
| 浅尝辄止："用过ChatGPT，也就那样" | 已在重度使用Claude Code → 在加速而非起步 |

**AI能力不是"又多了一个目标"，而是其他四个目标的效率倍增器：**

```
投资成功 ← AI加速研究、辅助决策
事业进阶 ← AI赋能科技金融框架 → 建立行业影响力
AI能力 → 家庭支持 ← AI辅助子女教育、家庭事务管理
百岁健康 ← AI分析体检数据、运动饮食跟踪
```

**检查点**：你能不能一句话说清"我为什么要学AI"？（提示：不是因为焦虑，是因为____）

## 一、第1层：建立正确认知——AI能做什么、不能做什么、怎么学

> 这一层解决的是：知道边界在哪，不做无用功。

### 1.1 AI擅长的 vs 不擅长的

| AI擅长（交给它） | AI不擅长（你来） |
|---------------|---------------|
| 信息汇总、文本生成、翻译润色 | 需要真实判断力的决策（信贷最终签批） |
| 结构化分析（对比、归纳、分类） | 需要人际关系洞察的沟通（客户谈判、团队管理） |
| 代码编写和数据分析脚本 | 需要承担责任的判断（合规意见、风险敞口决策） |
| 已知模式的识别和预警 | 真正原创性的战略思考 |

**记住**：AI是你的分析师和笔杆子，决策者仍然是你。

### 1.2 当前能力基线

| 维度 | 自评 | 一句话 |
|------|------|--------|
| AI概念理解 | 4/5 | 中科大理科底子，理解LLM工作机制、RAG、Agent概念 |
| 大语言模型使用 | 4/5 | Claude Code为绝对主力，日均1-2小时，每周5-7天 |
| AI辅助写作 | 3/5 | 已在用AI写报告和文档，但"先让AI出初稿再改"的肌肉记忆尚未形成 |
| AI辅助研究 | 3/5 | 研究流程偏即兴，尚未形成标准化SOP |
| AI辅助编程/数据分析 | 3/5 | 能通过自然语言让AI生成Python脚本，但pandas/可视化系统掌握尚浅 |
| AI行业动态跟踪 | 2/5 | 信息获取较随机，无固定渠道和频率 |

### 1.3 核心工具链（不贪多，4个够用）

| 工具 | 角色 | 覆盖场景 |
|------|------|---------|
| **Claude Code** | AI中枢 | 投资研究、报告撰写、数据脚本、知识管理、健康数据分析 |
| **Claude/ChatGPT 网页版** | 轻量助手 | 邮件起草、家庭咨询、快速问答 |
| **Perplexity/秘塔搜索** | AI搜索 | 行业动态、竞品调研、政策原文（标注来源） |
| **WPS AI（行内）** | 办公文档 | 会议纪要、报告排版、PPT大纲 |

**选工具三原则**：①不追新，新工具观望2-4周再决定 ②优先选能打通现有工作流的 ③一个月用不到3次的不付费

**检查点**：你现在能用一句话说清每个工具在你工作流中的角色吗？

## 二、第2层：组织学习路径——四层能力模型 + 百日计划

> 这一层解决的是：分阶段怎么学、每周做什么、什么时间出什么成果。

### 2.1 四层能力模型

```
第4层 领导层：带领团队用AI，在行内推动AI应用文化
         "我们部门用AI做了这个，效果很好"
                    ↑
第3层 整合层：将AI融入日常工作流，形成稳定方法论
         "每周固定用AI完成X、Y、Z三类工作"
                    ↑
第2层 应用层：熟练使用核心AI工具，解决具体问题
         "这个报告用AI先出初稿，我再改"
                    ↑
第1层 认知层：理解AI能做什么、不能做什么、发展趋势
         "这个场景适合用AI，那个不适合"
```

## 三、第3层：质量保障——验收标准 + 合规红线

> 这一层解决的是：怎么知道自己在进步、怎么确保不出合规问题。

### 3.1 三个月验收标准（2026年9月底）

| 验收项 | 合格标准 | 验证方式 |
|--------|---------|---------|
| **个股深度分析** | 3000字+，五模块（业务/财务/竞争/风险/估值），每个风险有数据支撑 | 选1只持仓股，从头到尾AI辅助完成 |
| **竞聘演讲稿** | 1500-2000字 + PPT大纲 + 5-10个答辩Q&A | 按真实竞聘场景准备全套材料 |
| **监管政策解读** | 3页以内速读笔记：要点+影响+合规建议 | 选一份真实文件，在时间压力下完成 |
| **体检报告分析** | 对比历史趋势，标注异常，给具体建议 | 用真实体检数据跑一遍 |

达标：四项全部完成。基本合格：完成3项 + 1项进行中。

### 3.2 产出目标追踪

| 产出类型 | 百日期目标 | 当前进度 |
|----------|-----------|---------|
| 自动化脚本 | ≥1个稳定运行 | 0/1 |
| Prompt模板 | ≥10个场景 | 0/10 |
| 研究报告 | ≥3篇（≥3000字/篇） | 0/3 |
| AI应用案例 | ≥5个 | 0/5 |

### 3.3 AI使用合规红线

作为银行业从业者，以下红线不可逾越：

| 红线 | 说明 |
|------|------|
| **不输入客户个人信息** | 客户姓名、身份证号、银行卡号、联系方式等不得输入任何外部AI工具 |
| **不输入行内机密文件** | 内部评级模型参数、风险敞口数据、未公开的业务策略不得输入AI |
| **不依赖AI做合规判断** | 授信审批、合规意见、风险敞口决策的最终判断必须是人 |
| **不将AI输出直接作为工作结论** | AI产出须经过你审阅和确认，关键数据须核实来源 |
| **不使用行内数据训练外部模型** | 外部AI工具的对话可能被用于模型训练，注意数据安全 |

### 3.4 薄弱环节针对性补强

| 薄弱环节 | 优先级 | 补强方法 |
|----------|--------|---------|
| 系统化AI工作流 | 高 | W9建立流程图，先写3个核心流程，跑两周再调整 |
| 提示词工程 | 高 | W2专项练习 + 对比实验 + W10-11建模板库 |
| AI+金融业务方法论 | 高 | W3-8投资+事业实战，所有练习用真实业务场景 |
| 合规意识 | 高 | 每次用AI处理工作前，过一遍红线清单 |
| 数据思维 | 中 | 用Python+AI逐步建立"用数据说话"的习惯 |

**检查点**：你能背出五条合规红线中的三条吗？

## 四、第4层：持续进化——从使用AI到AI自进化

> 这一层解决的是：百日期满后怎么继续往上走。

### 4.1 从使用者到倡导者

百日计划的目标不是"你会用AI"，而是"你能让身边的人感受到AI的价值"。

**行内分享准备**（W13重点）：

| 环节 | 时长 | 内容 |
|------|------|------|
| 开场 | 3分钟 | 我为什么要学AI（不是焦虑，是发现它真的能帮上忙） |
| 三个真实案例 | 15分钟 | 每个案例：没有AI时怎么做→有了AI后→省了多少时间 |
| 我的方法 | 5分钟 | 每周三动作+Prompt模板库+脚本自动化 |
| 给同事的建议 | 5分钟 | 从哪个任务开始、避免踩坑、合规注意事项 |
| Q&A | 5-10分钟 | 提前准备10个可能问题 |

### 4.2 建立影响力（2028-2030）

| 年份 | 方向 | 可衡量成果 |
|------|------|-----------|
| 2028 | 团队AI赋能 | 2-3人形成AI使用习惯；1个AI提效案例入选行内交流 |
| 2029 | 制度参与 | 行内AI相关会议/项目中发出业务部门的声音 |
| 2030 | 个人品牌 | 至少1次公开分享；1-2篇"AI+科技金融"署名文章 |

### 4.3 长期迭代节奏

| 周期 | 动作 | 说明 |
|------|------|------|
| 每周 | 三动作保底 + 周日5分钟复盘 | 最忙时也不跳过 |
| 每月 | 回顾Prompt模板库，淘汰低效的，补新发现的 | 模板库是活文档 |
| 每季 | Claude协助做一次AI能力自评 + 调整下季度重点 | 用路线图的评估表 |
| 每年 | 年度AI应用总结 + 制定下一年计划 | 对比上一年，看进步 |

**到这个阶段的信号**：

| 信号 | 说明 |
|------|------|
| AI发现了一个你自己没注意到的问题 | AI的观察力开始超过你 |
| 同事主动来问"你这个是怎么做的" | 影响力开始外溢 |
| 你早上看一下AI昨晚的产出，只修改了10% | AI交付质量达到你的90分标准 |
| 三个月没学新工具，但效率还在提升 | 不是工具在进步，是你在进步 |

**检查点**：百日期满后，你最想分享给同事的一个故事是什么？
