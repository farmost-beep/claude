# 快速学习Claude Code手册

> 适用对象：已会用Claude对话，想进阶到Claude Code提高效率的人
> 前置要求：已安装Claude Code CLI，在终端中可以使用`claude`命令

---

## 零、 一句话定位

**Claude Code = 终端里的AI程序员。日常对话用Claude App。写代码、改文件、跑命令、管项目——用Claude Code。**

### 它改变了什么

| 以前你做这些事... | 现在你只需要... |
|-----------|------------|
| 手动打开文件、查找、修改、保存 | 说一句话："把XX文件中所有YY改成ZZ" |
| 自己跑命令、看报错、搜索修复方法 | 说一句话："跑测试，有失败就分析原因并修复" |
| git add、git commit、写commit message | 说一句话："检查改动并提交" |
| 一件事做完再做下一件 | 同时启动多个Agent，并行推进 |
| 有想法→找程序员实现→等两天 | 有想法→对Claude Code说→5分钟看到结果 |

**核心变化**：你不再"操作电脑"，而是"指挥AI操作电脑"。你的角色从操作者变成目标设定者+结果检查者。**编程能力不再是瓶颈，描述问题的能力才是。**

## 一、 认识Claude Code

### 1.1 它在哪

```
Claude App（网页/桌面） → 聊天对话，通用助手
Claude Code（终端CLI）  → 编程代理，可以读写文件、执行命令
```

### 1.2 它能做什么（你可能用到的高频场景）

| # | 场景 | 举例 |
|---|------|------|
| 1 | 让它写代码并直接运行 | "写一个Python脚本分析这只股票的历史波动率" |
| 2 | 让它改多个文件 | "把项目中所有'女性'改成'男性'，检查所有文件" |
| 3 | 让它跑命令 | "检查git状态，把所有新增文件加入staging" |
| 4 | 让它生成文档并转PDF | "把这个.md文件生成PDF，用专业中文排版" |
| 5 | 让它查找和搜索项目 | "找到所有引用了XX函数的文件" |
| 6 | 让它管理任务 | "列出今天要做的事项，按优先级排" |
| 7 | 让它部署/运行服务 | "启动开发服务器" |

### 1.3 和普通Claude的区别

| 能力 | Claude App | Claude Code |
|------|-----------|-------------|
| 读取文件 | 需要你上传 | 自己读取项目文件 |
| 修改代码 | 输出代码让你复制 | 直接编辑文件 |
| 运行命令 | 不能 | 可以执行shell命令 |
| 管理项目 | 不能 | 可以理解整个项目结构 |
| 多步骤任务 | 需要你一步步引导 | 可以自主规划执行 |

## 二、 第一个命令

打开终端，输入：

```bash
claude
```

你会进入一个对话界面。试试：

```
帮我看看这个项目的目录结构，告诉我每个文件夹是干什么的
```

Claude Code会自动运行`ls`、`find`等命令来了解项目，然后告诉你。

**这就是Claude Code的核心使用方式——用自然语言给它布置任务，它会自己决定用什么工具来完成。**

## 三、 每天必用的5个技巧

### 技巧1：让它"先看再做"

```
❌ "帮我修bug"
✅ "先看看src/目录下的代码，理解项目结构。然后找到XX功能的bug并修复。"
```

关键：让AI先探索再行动，比直接让它做，准确率高得多。

### 技巧2：给它完整的上下文

```
❌ "改一下这个函数"
✅ "在src/utils.py的第50-80行，format_report函数输出的日期格式是
   YYYY/MM/DD，需要改成YYYY-MM-DD。同时检查所有调用这个函数的地方
   是否兼容新格式。"
```

关键：文件路径、行号、改什么、为什么改——四个信息给齐。

### 技巧3：一个任务分步骤来

```
第一步："先帮我找到所有引用了旧API的文件"
第二步："把第X行到第Y行的旧API调用换成新API"
第三步："运行测试确认改动没有破坏现有功能"
```

关键：用小步骤代替大任务，每一步都能验证。

### 技巧4：用/clarify确认理解

在消息后加一句：

> "在你开始改代码之前，先用你自己的话复述一遍你理解的任务——改什么文件、改什么内容、为什么。确认你理解对了再动手。"

### 技巧5：让它自己检查自己

> "改完了后，检查一下你的改动：(1)是否正确 (2)是否完整 (3)有没有引入新的问题"

## 七、 斜杠命令速查

在Claude Code对话中，输入`/`可以使用的命令：

| 命令 | 作用 | 什么时候用 |
|------|------|-----------|
| `/help` | 查看所有可用命令 | 不确定有什么命令时 |
| `/clear` | 清空对话历史 | 话题换了，不想AI受之前对话影响 |
| `/compact` | 压缩对话上下文 | 对话很长了，想释放token |
| `/memory` | 查看/管理AI对你的记忆 | 检查AI记住了你的哪些信息 |
| `/tasks` | 查看后台任务 | 有任务在后台跑，想看进度 |
| `/doctor` | 检查Claude Code状态 | 碰到奇怪的问题时 |

## 八、 进阶技巧（第2周开始用）

### 8.1 让它记住你的偏好

```
"记住：我是银行科技金融部门负责人，以后写正式文档用'行内'而非'银行'"
"记住：我偏好简洁直接的风格，不要写长篇大论的背景说明"
```

AI会把这些存入长期记忆，之后的对话会自动适配。

### 8.2 Multi-Agent：把你的效率再乘3倍

这是2026年Claude Code最强大的能力——同时启动多个子Agent并行工作：

```
"同时做三件事：(1)检查所有.md文件有没有'女性'这个词
 (2)检查所有.py文件有没有语法错误 (3)生成一个代码变更摘要"
```

每个子Agent独立运行、互不干扰。原来30分钟的任务，**3个子Agent并行5分钟完成**。这是从"一个助理"到"一个团队"的跨越。

### 8.3 定时任务：让AI在后台持续工作

```
"/loop 5m 检查测试是否通过"
"/loop 1h 抓取昆仑万维最新股价并记录"
```

Claude Code可以按你设定的频率自动执行任务——监控、定期检查、持续跟踪。**你不需要盯着屏幕，AI替你盯着。**

### 8.4 建立项目记忆（CLAUDE.md）

在项目根目录创建`CLAUDE.md`文件，写入项目说明、编码规范、你的偏好——Claude Code每次会先读这个文件。

你的项目已经有一个：`/Users/cyingfang/claude/CLAUDE.md`。

### 8.5 让它管理任务列表

```
"我今天要做5件事：1.修bug 2.跑测试 3.生成PDF 4.提交git 5.写周报。
帮我规划执行顺序，然后一个个完成"
```

### 8.6 让它变成你的代码审查员

```
"审查我最近一次commit的所有改动，指出安全问题、性能问题和风格问题"
```

## 九、 常见错误与应对

| 错误 | 应对 |
|------|------|
| AI改错了文件 | 立即说"回退你刚才的改动"，或用`git checkout -- <文件>`恢复 |
| AI不理解你的任务 | 追加一句"用你自己的话复述一遍，看我是不是说清楚了" |
| AI做了一半停了 | 发送"继续"，它会从中断的地方继续 |
| AI改了不该改的文件 | "你改了X文件，但我不需要改它。撤销对X的改动" |

## 十、 5天入门计划

### Day 1：环境顺畅
- 打开终端，输入`claude`，进入对话界面
- 让它介绍一下自己能做什么
- 让它总结一下你的项目结构

### Day 2：第一个实际任务
- 找一个你本来要手动做的事情（改文件、搜内容、跑命令）
- 让Claude Code来做

### Day 3：多步骤任务
- 找一个需要3步以上完成的任务
- 让Claude Code一步步完成，每步验证

### Day 4：代码生成
- 描述一个你需要的脚本/功能
- 让它生成代码并运行测试

### Day 5：项目管理
- 让它查看最近的改动
- 生成一份工作摘要
- 提交git
