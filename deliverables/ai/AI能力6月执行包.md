## 每周计划

### 第1周 (6/1-7)：基础夯实

- [ ] 回顾已有Python脚本（至少3个），用自己的话写出每个脚本"输入→处理→输出"
- [ ] 学习Claude Code hooks机制，写一个简单hook（如：写完文件后自动提醒生成PDF）
- [ ] 用Claude辅助完成1次代码调试（取一个之前报错的脚本，让AI帮你定位并修复）
- [ ] 产出2张知识卡片（建议主题：Agent模式工作流程、Edit工具精确修改）

### 第2周 (6/8-14)：脚本实战

- [ ] 独立写一个自动化脚本（建议方向：Markdown文件字数统计器，扫描deliverable目录汇总）
- [ ] 探索MCP插件生态，安装1个新插件并做一次实际使用
- [ ] 产出3张知识卡片（建议主题：Python文件读写模式、Python列表与字典、CSV数据处理）
- [ ] 选定深度报告主题，用AI辅助撰写大纲

### 第3周 (6/15-21)：Agent入门

- [ ] 学习Managed Agents概念，设计1个Agent任务（如：让Agent自动扫描本周更新的文档生成周报）
- [ ] 完善已有脚本的错误处理（给至少2个脚本加上try/except和友好错误信息）
- [ ] 产出3张知识卡片（建议主题：Managed Agents概念、Python异常处理、自动化链设计）
- [ ] 深度报告初稿完成（用AI辅助研究+写作，你负责判断和修改）

### 第4周 (6/22-30)：整合输出

- [ ] 将本月写的脚本整理为一个工具集（统一放在`scripts/`下，写一个README列出每个脚本的功能）
- [ ] Agent任务复盘：记录成功/失败/改进点（至少3条经验教训）
- [ ] 产出2张知识卡片（建议主题：6月学习总结、Claude Code高级技巧汇总）
- [ ] 深度报告定稿

## 知识卡片说明

知识卡片是本月的产出物之一，用途：把学到的概念/技巧/踩坑经验写成可检索的卡片，存入Obsidian Vault。

**每张卡片格式**：
- 标题：一句话说清这是什么
- 分类：AI概念 / 工具技巧 / 脚本经验 / 踩坑记录
- 内容：200-400字，用自己的话写
- 关联：链接到相关卡片或文档

**本月10张卡片建议分布**：

| 类别 | 数量 | 示例主题 |
|------|------|----------|
| AI概念 | 3张 | Agent模式、MCP协议、记忆系统 |
| 工具技巧 | 3张 | Edit精确编辑、并行Agent、/loop定时任务 |
| 脚本经验 | 3张 | Python文件读写、CSV处理、错误处理 |
| 踩坑记录 | 1张 | 本月遇到的1个典型坑 |

## 每日时间安排

| 时间段 | 优先级 | 时长 | 说明 |
|--------|--------|------|------|
| 工作日早晨（6:30-7:15） | 首选 | 30-45分钟 | 头脑清醒，不受工作打扰 |
| 午休时间（12:00-12:45） | 备选 | 30-45分钟 | 注意不影响下午工作 |
| 晚上（21:00-21:45） | 备选 | 30-45分钟 | 不要太晚影响睡眠 |

**掉队处理原则**：
- 某天任务超过1小时未完成：暂停，记录卡点，周末复盘时集中处理
- 连续两天没时间做：跳过，不补，从当天继续
- 某周验收标准未达标：不回溯，下周围绕弱项优先补强
- 核心：完成60%也比放弃好
