# Agent的演化路径（v2）：分化、共存与不可替代性

> 陈颖芳 | 2026年5月 | 约6000字
>
> v1版本的叙事是"操作型Agent → 治理型Agent"的线性替代。这个版本修正一个关键判断：**操作型Agent不会被完全替代。** 演化不是替代，是生态分化——操作型Agent在垂直场景、成本敏感、高可靠性需求中找到了治理型Agent和裸模型都无法占据的生态位。两种Agent长期共存，各自服务不同层次的需求。

---

## 一、修正一个判断

Agent系列的v1版本（演化路径）描绘了一条从操作型到治理型的转型曲线。核心叙事是：模型能力跃迁 → 操作型Agent的存在理由消融 → 治理型Agent崛起。

这个叙事大体正确，但有一处关键偏差：**它暗示操作型Agent会被治理型Agent替代——就像哺乳动物替代了爬行动物。**

真实情况更像：**森林分化成了不同的生态层。** 操作型Agent没有被替代——它分化了。一部分升维成治理型Agent。但更大的一部分在自己的生态位上扎得更深——在治理型Agent和裸模型都覆盖不到的领域找到了不可替代的位置。

这篇文章修正这个偏差。不是推翻v1——是在v1的线性叙事上增加"分化"和"共存"两个维度。

## 二、为什么操作型Agent不会被完全替代：四个理由

### 2.1 成本：不是所有任务都需要Mythos级的推理

Mythos的推理成本是Opus的很多倍。你让它分析投资组合的风险——值得。你让它帮你把一段JSON格式化——浪费。

操作型Agent在大量"不需要深度推理但需要稳定执行"的任务上，比裸模型有结构性优势：

- **稳定输出格式** — 裸模型可能在格式上自由发挥。操作型Agent的模板+验证层确保每次输出严格遵守schema。这不是"模型不够强所以需要Agent"——这是"格式严格性"天然适合Agent来做，无论模型多强。
- **批量处理** — 处理1000条记录的标准化任务。裸模型做1000次推理——成本高、速度慢、格式可能不一致。操作型Agent的流水线（数据分片→并行调用→结果汇总→格式校验）比裸模型便宜且可靠。
- **低延迟要求** — 客服Agent需要在500ms内响应。Mythos的深度推理可能超过2秒。操作型Agent用更小更快的模型+规则引擎——不是因为大模型做不了，是因为太慢。

### 2.2 可靠性：领域知识的积累不能每次都从头推理

Mythos能分析任何代码——但它不知道你的七个飞轮为什么被设计成现在这个样子。它不知道投资飞轮的四维度评估中，"行为纪律"这个维度的权重为什么高于"资本配置"。它不知道家庭飞轮中"端午节点"的判断阈值是基于你过去五年的家庭实践。

操作型Agent积累的是**领域知识和工作流记忆**。这些不是"模型推理一次就能替代的"——因为推理可以给你一个看似合理的答案，但它不知道"你的系统的设计决策和取舍"。

你的七个飞轮脚本中，有大量的判断阈值、权重系数、决策树分支——这些不是"最优解"，是"适合你的解"。Mythos可以重写它们，但无法在不理解"为什么是这个值"的情况下正确校准它们。

操作型Agent的价值之一是：**把人和模型之间的隐性知识固化到可重复执行的工作流中。** 这个价值不会因为模型变强而消失——反而因为模型变强而更珍贵（因为"为什么是这个而不是别的"的理由更不容易被模型捕捉）。

### 2.3 安全性：操作型Agent的"简"是治理型Agent的"可信"的基础

治理型Agent的可靠性建立在操作型Agent的简单性之上。

你的投资飞轮的黑天鹅检测——如果检测逻辑本身依赖一个复杂模型的推理链，那么当检测结果说"有问题"时，你不知道是"真的有问题"还是"检测器自己出了幻觉"。

但如果检测器是基于规则的操作型Agent——"持仓集中度>X%→标记为风险"——那么当它说"有问题"，你100%确定是规则触发了，不是幻觉。你可以不同意规则（阈值设得对不对是另一个问题），但你知道它是规则驱动的，不是推理驱动的。

**治理型Agent需要操作型Agent作为可靠的信号源。** 规则驱动的简单Agent提供"硬信号"——确定性的、可复现的、不需要解释的判断。推理驱动的治理Agent提供"软信号"——需要解释、可能不确定、需要交叉验证的判断。两套信号各有用处。硬信号校准软信号。软信号扩展硬信号的范围。

### 2.4 生态位：操作型Agent在"模型太贵/太慢/太不可控"的地带找到了位置

生物进化中，哺乳动物没有替代爬行动物。爬行动物在自己的生态位上同样成功：变温动物能耗低、适合稳定环境、不需要高频进食。

Agent生态有同样的分化逻辑：

| | 裸模型 | 操作型Agent | 治理型Agent |
|:---|:---|:---|:---|
| 适合的任务类型 | 单轮、深度推理、创造性 | 多步、规则化、批量、格式严格 | 监控、验证、仲裁、边界管理 |
| 不适合的任务类型 | 多步状态管理、格式严格输出 | 需要Mythos级推理深度的分析 | 执行类任务（它的职责是治理） |
| 成本敏感度 | 高（大模型推理贵） | 低（可用小模型+规则引擎） | 中 |
| 可靠性要求 | 中（单轮输出，错了好发现） | 高（多步链，错了难发现） | 极高（它错了意味着治理失效） |
| 生态位 | 大脑 | 手和脚 | 免疫系统和神经系统 |

这不是"治理型Agent比操作型Agent高级"。操作型Agent是手和脚——它们没有被大脑（裸模型）替代，也没有被免疫系统（治理型Agent）替代。它们在自己的生态位上不可替代。

## 三、演化路径修正：从"线性替代"到"生态分化"

### 3.1 v1的线性叙事

```
2023-2025     →    2026+
操作型Agent   →   治理型Agent
(被替代)           (崛起)
```

### 3.2 v2的分化叙事

```
                2023-2025                    2026+
                
裸模型 ─────────────────────────────────────────────→ 越来越强，替代了最浅层的操作职能
               
操作型Agent ──────┬─→ 浅层操作职能（任务拆解、简单记忆）
                  │      ↓ 被模型内部化吸收
                  │
                  ├─→ 中层操作职能（格式管理、批量处理、工具编排）
                  │      ↓ 仍然需要，Agent框架提供模板+验证
                  │
                  └─→ 深层操作职能（领域工作流、知识固化、规则引擎）
                         ↓ 不可替代——这是你系统的"肌肉记忆"
                         
治理型Agent ───────────────────────────────→ 从无到有，独立于操作型Agent的新分支
               
三者在2026年后的关系：
  裸模型：执行推理
  操作型Agent：固化领域工作流，提供稳定、低成本、可复现的"手和脚"
  治理型Agent：监控、验证、仲裁，确保模型和操作型Agent都在正确轨道上
```

### 3.3 不是"操作型→治理型"——是"操作型分化"

操作型Agent演化到2026年，分化成了三层：

**（1）被模型吸收的浅层操作** — 任务拆解、简单状态追踪、基本错误恢复。这些曾经是操作型Agent的核心职能。Mythos之后，这些在模型内部完成。操作型Agent不需要再做这些了——不是因为"被替代了"，是因为"这份工作模型做得更好了，Agent可以腾出手来做别的"。

**（2）持续存在的领域操作** — 这是操作型Agent最稳固的生态位。你的投资飞轮的黑天鹅生成规则、知识库飞轮的幽灵链接检测、家庭飞轮的端午节点提醒——这些不会被模型内部化，因为：

- 它们不是你"每次让模型推理一次"得到的结果——是你的工作流固化
- 它们的价值不在于"单次推理的质量"——在于"每次以相同标准执行，持续积累数据"
- 它们的判断阈值和业务流程是你几个月迭代出来的——不是通用推理可以替代的

**（3）规则驱动的硬信号源** — 这是v1没有覆盖的功能。操作型Agent提供的"确定性判断"（X>阈值→标记）是治理型Agent做更复杂判断的基础。治理型Agent可以说"操作型Agent标记了风险X，让我用更深度的推理来检查这个标记是不是误报"——但它自己不会产生这个初始标记。因为治理型Agent不适合做"规则检查"这件事——它是做"推理验证"的。

## 四、三种Agent的共存模型

### 4.1 在一个飞轮运行中的三层协作

以你的投资飞轮一次运行为例：

```
第1层：操作型Agent（规则引擎）
  → 数据抓取（Sina API → 价格数据）
  → 格式校验（数据完整性检查 → 缺失标记）
  → 规则检查（集中度>阈值？PE偏离历史均值>X%？）
  → 产出"硬信号"：3只股票触发风险阈值

第2层：模型（深度推理）
  → 基于操作型Agent的硬信号，做深度分析
  → "这3只股票的集中度风险是否被其他因素抵消？"
  → "这3只股票的相关性背后有没有共同的宏观因子？"
  → 产出"分析判断"：其中2只是真实的集中度风险，1只是数据噪声

第3层：治理型Agent（验证与校准）
  → 第2层分析的质量验证（第二实例交叉审查）
  → 第1层规则阈值的有效性回溯（过去3次标记中，误报率是多少？）
  → 第1层和第2层的产出一致性检查（操作型Agent说3只，模型说2只——差异原因是什么？）
  → 产出"治理报告"：本次分析的可信度评级+规则阈值是否需要调整的建议
```

三层各做各的事。没有一层替代另一层。没有一层比另一层"高级"。

### 4.2 三层之间的关系

```
操作型Agent ←→ 模型
  操作型Agent提供"硬信号"给模型做深度分析的起点
  模型的分析结果反馈给操作型Agent来校准规则阈值
  
模型 ←→ 治理型Agent
  治理型Agent验证模型的产出质量
  模型的"不确定性声明"触发治理型Agent的额外检查
  
治理型Agent ←→ 操作型Agent
  治理型Agent监控操作型Agent的误报/漏报率
  操作型Agent的简单可靠是治理型Agent做复杂判断的基础
```

**这是一个制衡系统，不是一条流水线。**

## 五、这对你的飞轮系统意味着什么

### 5.1 你不需要把操作型Agent"升级"成治理型Agent

你的七个飞轮中，大部分代码是操作型Agent——规则检查、数据收集、格式校验、报告生成。v1可能给人一种"这些都是旧时代的东西，需要升级"的印象。

v2的立场是：**这些不升级。** 它们做得好好的——格式稳定、运行可靠、成本低。它们不是"旧时代需要被替换的东西"——是你的系统的"肌肉记忆"。Mythos不需要替你格式化报告。Mythos应该做的是你当前系统做不了的事——深度分析、跨飞轮模式发现、多路径模拟。

### 5.2 但你需要增加治理型Agent作为新的一层

不是在操作型Agent外面包一层——是在操作型Agent和模型之上，增加一层治理层。

当前你的飞轮系统缺少的是：
- **操作型Agent的规则阈值多久没校准了？** 家庭飞轮中"文档30天未更新=陈旧"——这个阈值是什么时候设的？上次验证它的有效性是什么时候？
- **模型分析的质量谁在检查？** 投资飞轮的黑天鹅分析由一个模型实例完成→直接输出。没有第二实例验证。
- **操作型Agent的误报/漏报率是多少？** 家庭飞轮标记了多少次"风险"，其中多少次被验证为真风险、多少次是误报？

这些不是操作型Agent的问题——操作型Agent做了它该做的（触发了规则）。这些是需要治理型Agent来做的事——检查和校准。

### 5.3 操作型Agent需要"规则有效性回溯"

这是操作型Agent自身进化的机制——不需要变成治理型Agent，但需要被治理型Agent定期检查和校准。

具体来说，在每个飞轮中增加一个元数据字段：

```
[规则ID: FAMILY-003]
规则：文档超过30天未更新 → 标记为"陈旧"
最近10次运行：触发8次 / 实际陈旧6次 / 误报2次
误报率：25%
校准建议：考虑将阈值调整为45天，或增加"内容是否实质性变化"的检查
```

这不是让操作型Agent自己校准自己——是让治理型Agent（AI能力飞轮）定期扫描、发现偏差、建议校准、人确认。

## 六、演化规律的修订

v1提出了三条规律。v2保留它们的内核，但修正"替代"为"分化"。

### 规律一（修订）：Agent的存在理由由模型的短板和成本结构共同定义

v1说"Agent的存在理由永远由模型的短板定义"。这个表述不够完整。

短板只是存在理由的一半。另一半是**成本结构**。当模型能做某件事但成本高昂、延迟太高、或可靠性不够时——操作型Agent仍然有存在理由，不是因为模型"做不到"，是因为模型"做起来不划算"。

这解释了为什么操作型Agent在批量处理、格式检验、规则检查等场景中仍然不可替代——不是模型做不了，是用模型做这些事性价比太低。

### 规律二（修订）：每次模型跃迁，操作型Agent失去一部分"能做但代价高"的职能，治理需求增加

v1说的是"操作层价值缩水，治理层价值升值"。修订后：**操作型Agent不是在"缩水"——是在专业化。** 它从"补模型的短板"转向"做模型做起来不划算的事"。

浅层操作（拆任务、简单记忆）→ 被模型吸收（因为模型做起来更高效）
领域操作（工作流、规则引擎）→ 操作型Agent的稳固生态位（因为模型做起来不划算——不是做不到，是每次从头推理太贵/太不稳定）
治理职能 → 全新分支（因为需求是从模型能力增长中产生的）

### 规律三（修订）：Agent的终极形态不是模型+工具或模型+制衡，是操作+模型+治理的三层共存

v1说"Agent的终极形态是模型+制衡"。v2修正为：**不存在"终极形态"。** 三层长期共存，因为三个层次对应的是三种不同的需求类型——不是"低级→高级"的进化阶梯。

操作型Agent解决"怎么做"——高效、稳定、低成本地执行已知任务。
模型解决"怎么想"——在已知任务中注入深度推理。
治理型Agent解决"怎么确保对"——确保前两者在正确的轨道上。

三个需求不会消失。三者也永远不会合并成一个——因为高效执行、深度推理、独立监督，在结构上不能由同一个实体承担。

## 七、v1 vs v2：关键差异

| | v1 | v2 |
|:---|:---|:---|
| 核心叙事 | 线性替代（操作型→治理型） | 生态分化（浅层被吸收+领域操作保留+治理新分支） |
| 操作型Agent的命运 | 存在理由消融，价值递减 | 浅层操作被吸收，领域操作持续存在且不可替代 |
| 治理型Agent的出现 | 替代操作型Agent | 新增的独立分支，与操作型Agent共存 |
| "不要在操作型Agent上继续堆功能" | 对 | 部分对——不要在"被模型吸收的浅层操作"上堆，但要在"领域操作"上深耕 |
| 你的七个飞轮 | 本质上是治理型Agent系统 | 大部分是操作型Agent（规则引擎），需要增加治理层，而非替换 |
| 终极形态 | 模型+制衡 | 操作+模型+治理三层共存，没有终极形态 |
| 隐喻 | 哺乳动物替代爬行动物 | 森林的生态分层——灌木、乔木、藤本各占一层 |

## 八、后记：为什么做这个修正

v1写完后，两件事让我意识到偏差。

第一件：重新看你的七个飞轮代码。投资飞轮的黑天鹅检测、家庭飞轮的端午节点提醒、知识库飞轮的幽灵链接检查——这些都不是"可以被模型内部化"的东西。它们是规则引擎。规则引擎的价值不是"它比模型更聪明"——是"它是确定的、可复现的、不需要解释的"。

第二件：想清楚了制衡系统的结构——它必须是**三层**，不是两层。权力制衡在AI系统中的应用，需要的不是"执行者+监督者"两层，而是"操作者+推理者+监督者"三层。操作型Agent提供硬信号（规则触发），模型提供软信号（推理分析），治理型Agent对比两者并仲裁。

如果在"执行者+监督者"的两层框架中把操作型Agent等同于"执行者"——那就错了。操作型Agent不是执行者（模型才是）。操作型Agent是规则驱动的硬信号源——它和监督者（治理型Agent）是两种完全不同的东西，不会被彼此替代。

修正后的框架是三层共存。不是v1错了——是v1不够完整。它只描述了演化中的"替代"部分，漏掉了"分化"和"共存"的部分。

---

> Agent的演化不是一个物种替代另一个物种的故事。是一个生态系统逐渐丰富、分层的过程。
>
> 2023年：只有操作型Agent——它做全部的事
> 2024-2025年：模型越来越强，替代了操作型Agent最浅层的工作
> 2026年：治理型Agent出现——不是因为操作型Agent失败了，是因为模型太强，需要在操作型Agent之外建立一层独立的监督管理
>
> 结果是三个层次的共存：操作（手和脚）+ 模型（大脑）+ 治理（免疫系统）。
>
> 三层各做各的事。没有一层是多余的。没有一层是过渡性的。
>
> 这不是演化的终点。但下一阶段的演化不会是"其中一层替代另一层"。下一阶段的演化会是：**每一层内部的专业化加深，以及三层之间的接口变得更清晰。**
