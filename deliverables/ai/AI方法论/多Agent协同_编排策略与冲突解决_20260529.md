# 多Agent协同：编排策略、冲突解决与你的七个飞轮

> 陈颖芳 | 2026年5月 | 约5500字
>
> 一个Agent是工具。多个Agent是组织。你的七个飞轮各自独立运行，但它们之间有隐含的数据依赖和信号矛盾。当投资飞轮说"加大风险"、事业飞轮说"压力偏高"时——谁听谁的？这不是一个代码问题。这是一个组织设计问题。本文以你七个飞轮的跨飞轮耦合为案例，定义多Agent协同的四种模式、冲突解决的三级机制、以及一套可以直接编入AI能力飞轮（元治理）的仲裁规则。

## 一、为什么现在需要多Agent协同

### 1.1 你的飞轮已经是一个多Agent系统

七个飞轮分别监控七个不同的目标域：

```
投资飞轮     → 4维决策质量 → 黑天鹅检测 → 还本路径
AI能力飞轮   → 5维自进化   → 扫描所有飞轮代码
Coshine飞轮  → 4维竞争适配 → 竞对情报追踪
WeLinkData飞轮 → 竞争情报   → 供应链金融科技
知识库飞轮   → 4维规范检查 → 跨卡片模式发现
事业飞轮     → 3层结构扫描 → 方法论产出追踪
家庭飞轮     → 4维联动判断 → 风险预警
健康飞轮     → 3层因果链   → P0行动提醒
```

每个飞轮有自己的评估维度、自己的判断逻辑、自己的输出频率。它们是独立设计的——投资飞轮不知道家庭飞轮在做什么，事业飞轮不知道投资飞轮输出了什么判断。

但它们在数据层面不是独立的。它们共享：
- **你** — 同一个人，同样的时间预算，同样的注意力容量
- **你的资金** — 投资飞轮管理资金配置，家庭飞轮产生资金需求（赡养、子女教育），事业飞轮产生资金流入
- **你的状态** — 健康飞轮的身体状态影响投资飞轮的风险承受能力，家庭飞轮的压力水平影响事业飞轮的产出质量

### 1.2 已经出现的跨飞轮矛盾信号

即使飞轮之间没有显式的通信，矛盾已经开始出现：

| 场景 | 飞轮A的信号 | 飞轮B的信号 | 矛盾 |
|:---|:---|:---|:---|
| 资金配置 | 投资飞轮：可以加大权益仓位 | 家庭飞轮：暑假培训班+考研辅导预算待确认 | 资金在两个方向上被拉扯 |
| 时间投入 | 事业飞轮：投稿进度正常，建议加速 | 家庭飞轮：6月考研支持窗口即将打开，需大量时间投入 | 同一时间不能既多写文章又多辅导考研 |
| 风险评估 | 投资飞轮：黑天鹅分析显示集中度风险可控 | AI能力飞轮：投资飞轮代码质量评分偏低，分析结论可靠性待验证 | 投资飞轮的结论本身可能不可靠 |
| 优先级 | Coshine飞轮：竞对FIS有新动作，建议优先跟进分析 | WeLinkData飞轮：竞对也有新动作 | 两个商业分析同时需要你的注意力 |

这些矛盾信号目前没有被系统化地处理。它们会在你的头脑中被非正式地解决——"今天先看投资，明天再看家庭"。但随着飞轮数量增加、输出频率提高、Mythos让每个飞轮的分析都更深——非正式处理会越来越不够。

## 二、多Agent协同的四种模式

对你七个飞轮的不同耦合关系，匹配不同的协同模式。

### 模式A：层级式 — 明确的上报关系

**适用场景：** 一个Agent的输出需要被另一个Agent验证或审批。

**你的场景：AI能力飞轮 vs 其他六个飞轮**

AI能力飞轮是"元飞轮"——它扫描所有其他飞轮的代码质量、输出质量、进化状态。这意味着它天然处于其他飞轮的"上一层"——不是因为它更重要，是因为它的职能是"检查其他飞轮是否正常运行"。

层级式协同的运作方式：
```
AI能力飞轮（元治理）
  ├── 扫描投资飞轮代码 → 质量评分 → 如果低于阈值 → 标记"投资飞轮结论可靠性待验证"
  ├── 扫描家庭飞轮代码 → 质量评分 → 如果正常 → 不干预
  ├── 扫描Coshine飞轮 → 竞对情报新鲜度 → 如果过期 → 提醒"情报已过期，飞轮结论基于过时数据"
  └── ...
```

**关键设计：**
- AI能力飞轮只做"异常标记"——不做"纠正"。它告诉人"投资飞轮可能有问题"，不自己改投资飞轮的代码
- 被标记的飞轮不会自动停止——它继续运行，但在输出报告中附加"AI能力飞轮对本次分析的可靠性评级为B"
- 只有人有权限"根据AI能力飞轮的标记暂停某个飞轮"

### 模式B：平等协商式 — 两个Agent独立判断后比较

**适用场景：** 两个Agent从不同角度分析同一个问题，结论可能一致或矛盾。

**你的场景：投资飞轮 vs 家庭飞轮（资金配置问题）**

投资飞轮关注"怎么配置资金回报最高"。家庭飞轮关注"家庭支出需求是什么"。两个飞轮对"当前可用资金"的看法可能不同。

平等协商式的运作方式：
```
投资飞轮输出：建议可配置资金X万用于权益投资
家庭飞轮输出：6月-9月家庭预期支出Y万（暑假班+考研辅导+端午+暑假）
         ↓
    比对：X和Y的关系？
      X >> Y → 一致（投资空间充足）
      X ≈ Y  → 协商：家庭支出优先，剩余资金投资
      X < Y   → 矛盾：资金不足，需要优先级排序
         ↓
    冲突 → 升级给人，附带两个飞轮的分析依据
```

**关键设计：**
- 两个飞轮保持独立运行——各自的评估维度、各自的判断逻辑互不干扰
- 只在"冲突信号"出现时才触发协商——不是每次运行都协商
- 协商的结果是"识别矛盾并呈现"——不是"决定谁对"

### 模式C：流水线式 — Agent的输出是下游Agent的输入

**适用场景：** 一个Agent的分析结果被另一个Agent用作输入数据。

**你的场景：事业飞轮 → 投资飞轮（收入预期影响风险承受）**

事业飞轮追踪你的职业进展、投稿进度、方法论产出。这些影响你的收入预期。收入预期影响投资的风险承受能力。

但目前，投资飞轮的风险承受参数是人手动设的——它不知道事业飞轮对收入稳定性的判断。

流水线式协同的运作方式：
```
事业飞轮输出：
  - 6月投稿：3篇（按计划）
  - 方法论产出：Agent系列4篇完成（超前）
  - 职业稳定性评估：高 → 收入乐观

      ↓（数据流入投资飞轮）

投资飞轮接收：
  - 收入预期 = 乐观
  - 建议风险承受 = 适度上调（从"保守"→"中性"）
  - 附注：此调整基于事业飞轮的收入稳定性评估
```

**关键设计：**
- 不是"事业飞轮告诉投资飞轮怎么做"——是"事业飞轮把自己的判断作为数据输入给投资飞轮，投资飞轮在自己的框架内决定是否调整参数"
- 下游飞轮（投资）必须显式声明"我的结论中哪个参数受上游飞轮（事业）数据的影响"
- 如果下游飞轮决定**不**采纳上游飞轮的数据——需要写一句原因

### 模式D：联邦式 — 各自独立，定期汇总

**适用场景：** 多个Agent各自负责独立领域，不需要实时协调，但需要定期汇总给人类。

**你的场景：Coshine飞轮 + WeLinkData飞轮（两个独立的竞对分析）**

两个飞轮追踪不同的竞争格局（支付处理器 vs 供应链金融科技），各自独立更新。但它们共享"需要你的注意力"——当两个飞轮同时输出"需要注意"的信号时，联邦式协同确保你不会漏掉其中一个。

联邦式协同的运作方式：
```
每周末汇总：
  Coshine飞轮本周信号：
    ✅ FIS无新动作
    ⚠️ Fiserv agentic commerce更新——需关注
    ✅ 其他5家正常
  
  WeLinkData飞轮本周信号：
    ✅ 竞对A正常
    ⚠️ 竞对B新融资——需关注
    ✅ 其他正常

汇总后：2项需关注（Fiserv + 竞对B），优先级标注 + 建议处理时间
```

**关键设计：**
- 每个飞轮独立运行，互不干扰
- 联邦层（汇总层）只做"收集+去重+优先级排序+呈现"——不做判断
- 人类每周只需要看一份联邦汇总，不需要逐一检查七个飞轮

## 三、冲突解决的三级机制

当两个飞轮给出矛盾信号时，怎么处理？

### 三级机制总览

```
第一级：自动比对（Agent做）
  两个飞轮的输出被自动比对 → 检测到矛盾信号
  → 自动检查：矛盾是"数据层面"还是"判断层面"？
  → 数据层面矛盾：通常是数据源不一致 → 自动标注+请求数据源统一
  → 判断层面矛盾：进入第二级

第二级：结构化呈现（Agent做人确认）
  矛盾的双方判断 + 各自的依据链 + 冲突的核心假设
  → 生成"冲突升级包"推送给人
  → 人在24小时内确认：A对/B对/两者都不对/需要更多信息
  → 人的判断被记录为"先例"

第三级：规则进化（人做，Agent辅助）
  同一类型的矛盾出现3次以上
  → 触发规则更新：定义"当A和B在这个问题上矛盾时，默认采用X规则"
  → 新规则编入第一级自动比对
  → 同类矛盾以后不再升级到第二级
```

### 3.1 第一级：自动比对

**触发条件：** 两个飞轮在同一次联邦汇总中输出了涉及同一资源的矛盾信号（资金、时间、注意力）。

**Agent做的事：**
1. 识别矛盾类型：资金冲突/时间冲突/信号可靠性冲突/优先级冲突
2. 判断矛盾是"数据层面"还是"判断层面"
   - 数据层面："飞轮A说资金X万，飞轮B说资金Y万"→ 可能是一个数据源没更新
   - 判断层面："飞轮A建议加大风险（因为市场机会多），飞轮B建议减少风险（因为家庭支出大）"→ 两个合理但矛盾的判断
3. 数据层面矛盾 → 自动标注，请求数据源统一，不升级
4. 判断层面矛盾 → 进入第二级

### 3.2 第二级：结构化仲裁

**触发条件：** 第一级判定为"判断层面矛盾"。

**Agent做的事：**
生成一个"冲突升级包"，包含：

```
═══════════════════════════════
🏠 跨飞轮冲突升级 | 2026-06-07
═══════════════════════════════

冲突类型：资金配置优先级冲突

飞轮A（投资）的判断：
  "建议将可配置资金从30%提升到50%，当前市场估值偏低，机会窗口存在"
  依据链：
    ① 持仓PE中位数处于历史25分位
    ② 宏观指标未显示系统性风险
    ③ 黑天鹅压力测试：最差情景可承受

飞轮B（家庭）的判断：
  "6-9月家庭预期支出X万，需保留40%流动资金"
  依据链：
    ① 暑假考研班预算Y万（待陈于东确认）
    ② 两位母亲赡养费用+端午/中秋节点
    ③ 国庆家庭旅行预算待确认

冲突的核心假设：
  投资飞轮假设"当前流动性需求低"
  家庭飞轮认为"6-9月流动性需求高"
  两个假设都合理，但对"流动性需求"的判断不同

需要你判断的问题（不是一个选项——是一个精确的问题）：
  >> 6-9月的最低流动性需求是多少？<<
  （确定了最低需求后，剩余资金可以用于投资，两个飞轮的矛盾自动消解）

═══════════════════════════════
```

**关键设计：**
- 不是"两个选项你选哪个"——是"一个精确的问题，回答了它，矛盾自动消解"
- 飞轮A和飞轮B的分析依据完整呈现——但"需要人判断的问题"是仲裁Agent提炼的
- 附带先例参考："上次类似冲突（5月15日），你的判断是……"

### 3.3 第三级：规则进化

**触发条件：** 同一类型的矛盾出现3次以上。

**人做的事（Agent辅助）：**
- 回顾3次先例中的判断：每次判断的逻辑是什么？有没有模式？
- 定义新规则：例如"所有飞轮在资金相关判断中，统一使用'最低流动性需求优先'原则"——即家庭飞轮的流动性需求先满足，剩余资金再进入投资飞轮的配置逻辑
- 新规则编入第一级自动比对——同类矛盾以后不再升级

**Agent辅助的事：**
- 从3次先例中提取判断模式，生成"建议规则"
- 规则生效后，监控新规则是否造成了新的矛盾

## 四、跨飞轮耦合图谱：你当前需要建立的四条数据流

基于以上分析和你的七个飞轮的当前状态，四条跨飞轮数据流需要从隐性变为显性：

### 数据流1：AI能力飞轮 → 所有飞轮（元治理）

**当前状态：** AI能力飞轮扫描所有飞轮代码，但扫描结果不反馈给被扫描的飞轮。

**建议：** AI能力飞轮的扫描结果作为"可靠性评级"附加到每个飞轮的输出中。投资飞轮报告开头显示"本次分析的AI系统可靠性评级：A"。

### 数据流2：家庭飞轮 → 投资飞轮（资金约束）

**当前状态：** 投资飞轮的资金配置不考虑家庭支出需求。

**建议：** 家庭飞轮的"预期支出"数据流入投资飞轮，投资飞轮在资金配置逻辑中增加"扣除预期支出后的可配置资金"。

### 数据流3：事业飞轮 → 投资飞轮（收入稳定性）

**当前状态：** 投资飞轮的风险承受参数由人手动设定。

**建议：** 事业飞轮的"职业稳定性评估"作为投资飞轮风险承受参数的调整因子。不是自动调整——是在投资飞轮报告中显示"基于事业飞轮的收入稳定性评估，建议风险承受参数从保守调整为中性"。

### 数据流4：健康飞轮 → 投资飞轮（风险偏好校准）

**当前状态：** 完全不相关。

**建议：** 这是最轻量级的耦合。健康飞轮的P0行动提醒中如果出现"睡眠严重不足"或"压力显著偏高"——投资飞轮的报告附注"注意：健康飞轮检测到状态异常，重大投资决策建议推迟到状态恢复后"。

## 五、在AI能力飞轮中实现联邦汇总

AI能力飞轮是你的"元治理Agent"。在它里面增加一个联邦汇总模块，是这个多Agent协同方案的最轻量实现。

### 联邦汇总模块的设计

当前AI能力飞轮的5维度扫描（规范力/编排力/验证力/扩展力/感知力）是"向内看"——检查AI系统本身的质量。联邦汇总是"向外看"——收集所有飞轮的本周信号，汇总、去重、优先级排序。

```python
def 联邦汇总():
    # 收集所有飞轮的本周输出
    signals = []
    for flywheel in [投资, AI能力, Coshine, WeLinkData, 知识库, 事业, 家庭, 健康]:
        report = flywheel.latest_report()
        signals.append({
            "飞轮": flywheel.name,
            "状态": report.status,
            "关键信号": report.key_signals,
            "需关注项": report.action_items,
            "可信度": AI能力飞轮.可靠性评级(flywheel)
        })
    
    # 检测跨飞轮矛盾
    conflicts = detect_conflicts(signals)  # 第一级自动比对
    
    # 优先级排序
    prioritized = sort_by_urgency(conflicts + all_action_items)
    
    # 生成联邦汇总报告
    return build_federation_report(signals, conflicts, prioritized)
```

### 每周联邦汇总报告应该包含什么

1. **本周所有飞轮的状态一览**（一张表：飞轮名+状态emoji+关键信号+可信度评级）
2. **跨飞轮矛盾**（如果有——按三级机制处理后的结果）
3. **本周需人关注的top 3事项**（跨飞轮去重+优先级排序后的结果）
4. **上周事项的回溯**（上周top 3事项的完成状态）

## 六、四种协同模式在你飞轮中的部署

| 协同关系 | 模式 | 优先级 | 当前状态 | 关键动作 |
|:---|:---|:---|:---|:---|
| AI能力飞轮 → 所有飞轮 | A（层级式） | P0 | 已有基础，缺反馈 | AI能力飞轮输出可靠性评级，附加到各飞轮报告 |
| 投资飞轮 ↔ 家庭飞轮 | B（平等协商） | P0 | 无 | 建立资金配置的矛盾检测+升级机制 |
| 事业飞轮 → 投资飞轮 | C（流水线式） | P1 | 无 | 事业飞轮的收入稳定性→投资飞轮的风险参数 |
| Coshine + WeLinkData | D（联邦式） | P1 | 已有独立运行 | 周末联邦汇总去重+优先级排序 |
| 健康飞轮 → 投资飞轮 | C（轻量流水线） | P2 | 无 | 仅异常状态触发提醒，非定期 |
| 家庭飞轮 ↔ 事业飞轮 | B（平等协商） | P2 | 无 | 时间投入的矛盾检测（考研辅导 vs 投稿写作） |

## 七、多Agent协同不是让系统更复杂——是让矛盾可见

多Agent协同最容易犯的设计错误是：**过度集成。** 把所有飞轮串成一条流水线，A输出→B输入→C验证→D汇总——结果是一个高度耦合的系统，一个飞轮出问题，全链条停摆。

这篇文章建议的方案是相反的：**最小耦合。** 每个飞轮保持独立运行。只在以下三种情况下触发跨飞轮通信：

1. **矛盾信号出现时** — 两个飞轮在资金/时间/注意力上给出了相反的建议
2. **数据自然流动时** — 一个飞轮的输出是另一个飞轮的天然输入（事业→投资）
3. **每周汇总时** — 联邦层收集所有信号，去重，排序

其他时间——每个飞轮自己跑自己的。不协同。不耦合。

多Agent协同的目标不是"让系统更聪明"——是**让矛盾可见，让人能在充分信息下做出跨飞轮的判断。**

矛盾以前也存在。投资飞轮想加大风险，家庭飞轮需要更多资金——这个矛盾不需要飞轮协同也能发生。区别在于：以前它在你的脑子里非正式地解决。现在它被显式化、结构化、附带完整的依据链和先例参考——**你仍然做判断，但你现在能看到两个飞轮的完整分析，而不是凭直觉在两个方向中选一个。**

---

> 七个飞轮各自是一套独立的判断系统。它们不需要合并成一个更大的系统——但它们的判断信号需要有一个地方交汇、比对、去重、冲突升级。
>
> 这个地方就是AI能力飞轮中的联邦汇总模块。
>
> 多Agent协同的终点不是"全自动协同"——是"矛盾可见、依据可追溯、人的跨飞轮判断建立在充分信息上"。
>
> 不是让Agent替你决定投资和家庭怎么平衡。是让Agent确保你面对这个平衡时，两个方向的分析都摆在你面前。
