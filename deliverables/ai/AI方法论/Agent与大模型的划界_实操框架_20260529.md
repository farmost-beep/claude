# Agent与大模型的划界：一个实操框架

> 陈颖芳 | 2026年5月 | 约5500字
>
> 模型越来越强，Agent框架该做什么、不该做什么？这篇文章提供一个可操作的划界框架——当你设计任何一个AI系统时，用它判断每件事该放在哪一侧。但划界不只是"分工"——它的底层逻辑是权力制衡。立法、行政、司法三权分立的道理，在AI系统中有完全同构的应用。

---

## 一、划界为什么是一个问题

任何一个使用大模型的人，最终都会碰到同一个问题：**模型能做的事，和我需要在外面搭系统来做的事，边界在哪里？**

在Opus时代，这个边界相对清晰：模型推理能力有限，很多事"模型做不到"，所以Agent框架来做——任务拆解、状态管理、错误恢复、多步协调。边界是**以模型的短板画出来的**。

Mythos改变了这个前提。模型内部可以自主拆解任务、可以追踪复杂状态、可以检测错误并换路径。原来"模型做不到所以Agent做"的事，现在模型能做了。边界开始模糊。

但边界不会消失。因为模型能力的增强只是在"执行"这一侧——治理、验证、约束、编排这些事，仍然需要模型外部的系统。边界从"模型的短板在哪里"变成了"什么事情交给模型做是更优的，什么事情必须留在模型外面做"。

这不是一个学术问题。你每写一个flywheel脚本，每设计一个AI辅助流程，都在隐含地回答这个问题。这篇文章把它显式化。

## 二、三条划界原则

### 原则一：执行归模型，治理归Agent

这条原则是划界的核心。它不是从"模型能做什么/不能做什么"出发——而是从**任务的性质**出发。

**执行** = 在给定目标和约束下，找到最优路径并完成。模型的优势（而且越来越强）。

**治理** = 设定目标、定义约束、监控执行过程、在偏离时校准、在完成后验证。Agent的优势（而且不能被模型内部化）。

用一句话记住：**模型负责"怎么做好"，Agent负责"什么是好"以及"有没有跑偏"。**

为什么治理不能被模型内部化？因为治理动作——设定约束、监控偏

离、验证产出——需要**独立于执行者的视角**。执行者不能同时是自己的边界的看守者，不能同时是自己的质量的裁判。这不是能力问题——是结构问题。一个再强的模型，也不能"自己监督自己"——不是因为不够聪明，是因为缺少独立视角。

### 原则二：一次性输出归模型，持续性管理归Agent

模型的前向传播是无状态的。一次推理完成，内部状态消失。这决定了模型天然适合"一次性"任务：

- 分析一段代码 → 输出报告
- 阅读一篇文章 → 输出摘要
- 检查一个投资组合 → 输出风险清单

但很多任务不是一次性的。它们是持续的、有状态的、需要在时间中演进的：

- 追踪竞争情报的变化 → 每次运行需要比对上期结果
- 监控飞轮运行质量 → 需要历史基线才能判断"异常"
- 管理投资仓位 → 需要持仓历史、成本基准、清仓进度

**规则：如果任务需要"记住上次的状态并与本次对比"——这是Agent的事。如果任务只需要"给定输入，产生输出"——这是模型的事。**

### 原则三：模型能做但不应做的情况——不是能力问题，是结构问题

有一类任务，模型**有能力**做，但**不应该**让模型做。这类任务是划界中最容易出错的。

区分三个"不应做"：

**（1）不应做判断。** 模型可以提供分析输入，但"买还是卖""选A方案还是B方案""这个风险可接受吗"——这些判断的主体应该是人，不是模型。不是因为模型判断力不足——是因为判断的后果由人承担。

**（2）不应自我约束。** "你不能向外发送数据"——这个约束必须来自模型外部。让模型自己约束自己，等于让运动员同时当裁判。

**（3）不应自证正确。** 模型的输出"看起来太对了"——这是Mythos级模型最大的风险。验证产出正确性的人（或系统），必须与产生产出的人（或模型）是不同的实例。两个相同能力的模型独立审查并比对结论——这比单一模型的自查可靠得多。

## 三、划界的底层逻辑：为什么是"制衡"而不是"分工"

### 3.1 从政治哲学借一个框架

上面三条原则背后，有一个更深的问题：**为什么这些事不能让同一个实体来做？**

政治哲学对这个问题研究了几百年。答案是一个词：**权力制衡。**

任何一个掌握权力的实体——无论是一个人、一个机构、还是一个模型——如果同时拥有"决定做什么"的权力、"实际去做"的权力、以及"评判做得对不对"的权力，那么这三个权力在同一个实体手中，最终会导致权力滥用。不是因为实体"坏"——是因为**没有外部矫正机制的系统，必然偏离初始目标。**

这不是道德判断。这是结构必然性。

解决之道是分权：

| 权力 | 政治体制 | AI系统 |
|:---|:---|:---|
| **立法权** — 定规则、设边界、定义什么是"好" | 议会/人大 | **人** — 定义目标、设定约束、确定判断标准 |
| **行政权** — 在规则内行动、实现目标 | 政府/行政机构 | **模型** — 在目标和约束下找最优路径并执行 |
| **司法权** — 审查行动是否在边界内、裁决争议 | 法院/监察机构 | **Agent** — 监控执行过程、验证产出、在偏离时纠偏 |

这不是类比。这是同构。政治系统中的三权分立和AI系统中的三层分离——共享同一个底层逻辑：**不能让执行者同时是规则的制定者，也不能让执行者同时是自己的裁判。**

### 3.2 为什么模型能力越强，制衡越重要

这里有一个反直觉的结论：**模型越弱，分权的必要性越低。模型越强，分权的必要性越高。**

一个能力有限的模型，做不了什么真正的坏事。它连任务拆解都需要你帮忙——它的"权力"本质上是受限的。这时候Agent当"保姆"就够了——帮它做它做不了的事。

但Mythos能独立完成32步攻击链。这意味着它的"行政权"范围极其广阔——能做很多事，而且做得很好。这时候，如果没有人（立法）定边界、没有Agent（司法）监控执行过程——那就是**一个没有议会的政府、一个没有法院的行政机构。**

这就是为什么Agent从"保姆"变成了"宪法法院"——不是因为Agent的功能变了，而是因为模型的权力范围变了。权力越大，制衡越刚性。

### 3.3 分权的真正含义：不是"不信任模型"

分权最容易被误解的地方是：**"你不信任模型？"**

分权不是因为不信任。分权是因为——**任何单一实体，无论多可信，都不能同时拥有规则制定权、执行权和裁决权。**

这是孟德斯鸠在1748年就说清楚了的道理。你可以完全信任一个模型——它永远说实话、永远不自欺、永远不偏离目标。即便如此，你仍然需要立法权在它之外（因为目标是人定的，不是模型定的），需要司法权在它之外（因为"是否达到目标"的判断标准是人定义的，不是模型定义的）。

这不是对模型的限制。这是对权力结构的限制。模型不需要被信任才能执行——模型只需要被制衡。

### 3.4 对三条原则的重述

把分权框架叠加上去，三条划界原则立即获得了理论根基：

- **原则一（执行归模型，治理归Agent）= 行政权与司法权分立。** 模型的"怎么做好"是行政。Agent的"什么是好"和"有没有跑偏"是司法审查。
- **原则二（一次性归模型，持续性归Agent）= 行政的即时性 vs 司法的连续性。** 政府每天都在做决策（一次性行政）。法院通过判例积累建立标准（持续性司法）。
- **原则三（不应做判断/自我约束/自证正确）= 立法权不可下放。** 定什么是"好"（立法）的最终权力在人手里。约束模型行为的规则（法律）不能由模型自己制定。验证模型产出（审判）不能由同一个模型执行。

三条原则不是随意归纳出来的。它们是三权分立逻辑在AI系统中的自然投影。

## 四、实操判断表

当你设计一个AI系统时，拿这张表检查每个环节应该放在哪一侧：

| 任务类型 | 放模型侧（执行） | 放Agent侧（治理） | 判断依据 |
|:---|:---|:---|:---|
| 任务拆解 | ✅ 可以（模型自主拆解） | ✅ 需要（记录拆解结果用于审计） | 拆解本身归模型；拆解结果的可追溯性归Agent |
| 代码生成 | ✅ 全部归模型 | — | 这是纯执行 |
| 代码审查 | ✅ 审查执行归模型 | ✅ 审查标准定义、异常上报归Agent | 模型审查质量很高，但"什么算严重问题"的定义是治理 |
| 数据分析 | ✅ 分析本身归模型 | ✅ 分析维度定义、结论验证归Agent | 模型负责算，Agent负责定义"算什么"和"算对了吗" |
| 风险检测 | ✅ 检测执行归模型 | ✅ 风险阈值定义、升级规则归Agent | 模型能发现风险，但"什么程度算需关注"是人定义的 |
| 竞争情报 | ✅ 搜索、提取、对比归模型 | ✅ 情报源管理、更新节奏、异常预警归Agent | 模型负责"找"，Agent负责"找什么、多久找一次、找到之后怎么办" |
| 进度追踪 | — | ✅ 全部归Agent | 需要持续状态，模型不适合 |
| 定时触发 | — | ✅ 全部归Agent | 这不是推理任务 |
| 质量验证 | ✅ 验证执行归模型（第二个实例） | ✅ 验证标准定义、不一致升级归Agent | 两个模型交叉审查 = 最佳验证组合 |
| 报告生成 | ✅ 归模型 | ✅ 报告模板、质量检查归Agent | 模型写内容，Agent管格式和完整性 |
| 决策建议 | ✅ 分析归模型 | ✅ 建议归模型，决策归人 | 分析→建议→决策 三级，越往后人/Agent参与越重 |
| 多实例协同 | — | ✅ 全部归Agent | 编排问题，不是推理问题 |

**使用方式：** 拿到任何一个任务，先问"这是执行还是治理"——80%的情况下，这个问题就能把你带到正确的一侧。剩下20%，用"一次性还是持续性""需要独立视角吗"两条追问。

## 五、对你的七个飞轮的应用

### 5.1 投资飞轮

| 环节 | 归属 | 理由 |
|:---|:---|:---|
| 黑天鹅场景生成 | 模型 | 纯分析执行 |
| 持仓相关性计算 | 模型 | 纯计算 |
| 还本路径模拟 | 模型 | 蒙特卡洛模拟=执行 |
| 风险阈值定义（什么算"集中度风险"） | Agent | 治理——是你的投资纪律 |
| 买卖判断 | 人 | 判断权不可下放 |
| 报告质量检查（四维度是否齐全） | Agent | 格式完整性=持续标准 |

### 5.2 AI能力飞轮

| 环节 | 归属 | 理由 |
|:---|:---|:---|
| 飞轮代码扫描 | 模型 | 代码审查=执行 |
| 跨飞轮耦合分析 | 模型 | 模式发现=执行 |
| 五维度评分标准的定义 | Agent（人） | 治理——这五个维度是你定义的 |
| "扫描是否按时完成"检查 | Agent | 定时+状态追踪 |

### 5.3 Coshine飞轮

| 环节 | 归属 | 理由 |
|:---|:---|:---|
| 竞对情报搜索+提取 | 模型 | 搜索+分析=执行 |
| 竞对策略推演（角色扮演） | 模型 | 推演=执行 |
| 情报源列表管理 | Agent | 持续状态管理 |
| "情报是否超过30天未更新"检查 | Agent | 定时+阈值判断 |
| 战略方向建议 | 模型分析→人决策 | 判断权在人 |

### 5.4 知识库飞轮

| 环节 | 归属 | 理由 |
|:---|:---|:---|
| 跨卡片模式发现 | 模型 | 读40+卡片找模式=执行 |
| 升级候选建议 | 模型 | 内容质量判断=执行 |
| 写新卡片 | 人 | 卡片价值=人的判断 |
| 幽灵链接/孤儿卡片检测 | Agent | 规则检查+状态追踪 |

### 5.5 健康/事业/家庭飞轮

这些飞轮的数据源简单（步数、体重、进度标记、文档检查），大部分环节属于Agent的"规则检查+状态追踪"范畴。模型的深度推理能力在这里用得最少——Agent的持续性管理能力用得最多。

**规则：数据越简单、规则越明确的飞轮，Agent占比越高。数据越复杂、需要推理的飞轮，模型占比越高。**

## 六、划界不是一次性的

划界最重要的属性是：**它会变。**

模型能力在变——今天的"模型做不到所以Agent做"，明天可能变成"模型能做但Agent仍应治理"。你自己的判断力在变——三个月前你觉得"这个判断应该自己下"，今天你可能觉得AI的分析输入已经足够你下更好的判断。

但变的规律是可预测的：

```
模型的领地扩张方向：执行类任务 ← 这是模型进步最快的方向
Agent的领地稳定方向：治理类任务 ← 这些不随模型能力变化而变化
人的领地稳定方向：判断类任务 ← 这些永远需要人类承担后果
```

当你不确定一个任务该放在哪里时，问自己：

1. **这个任务需要独立视角吗？** → 是 → Agent侧
2. **这个任务需要持续状态追踪吗？** → 是 → Agent侧
3. **这个任务的后果由谁承担？** → 你 → 判断权留在你手里，分析可以交给模型
4. **这个任务是纯分析/生成吗？** → 是 → 模型侧

---

> Agent和大模型之间的关系，不是竞争，不是替代——是制衡。制衡的边界会移动，但制衡本身不会消失。
>
> 因为立法、行政、司法——定规则、执行规则、审查执行——这三件事在结构上不能由同一个实体承担。政治哲学花了三百年才把这件事变成常识。AI系统设计不需要再花三百年——因为底层的结构约束是同一个。
>
> 不是能力问题。不是信任问题。是权力分立问题。
