## 1. 为什么AI能力是第五个长期目标

### 1.1 AI正在重塑金融业

银行业是AI落地最直接、最广泛的行业之一。以下领域已经在发生实质性改变：

| 领域 | 现状 |
|------|------|
| **信贷审批** | AI辅助尽调、企业经营分析、信用评分模型，传统信贷员经验正在被算法补充甚至替代 |
| **风险管理** | 实时异常交易监测、压力测试自动化、舆情预警系统，人力无法覆盖的维度AI能覆盖 |
| **财富管理** | 智能投顾、个性化资产配置建议，正在从高净值客户向大众客户渗透 |
| **客户服务** | 智能客服处理80%以上标准咨询，人工坐席转向复杂问题处理 |
| **合规与报告** | 监管文件自动生成、反洗钱筛查、合同条款AI审查 |

**关键判断**：AI不会取代银行家，但会用AI的银行家会取代不会用的。这不是五年后的事，是正在发生的事。

### 1.2 49岁金融管理者的独特窗口

大部分同龄同行对AI的态度停留在三种状态：
- **漠视**："离我还远，基层先试"
- **焦虑但不动**："知道重要但不知从何下手"
- **浅尝辄止**："用过ChatGPT，也就那样"

这三种状态的共同结果是——**没有形成系统性的AI应用能力**。

而陈颖芳的起点截然不同：
- 中科大理科训练 = 对技术有底层理解力，不会被术语唬住
- 银行业20+年经验 = 知道业务痛点在哪里，AI用在刀刃上
- 已在使用Claude Code等工具 = 不是从零开始，是在加速

**结论**：49岁+金融背景+AI工具，这是稀缺组合。大多数懂AI的人不懂金融业务，大多数懂金融的人畏惧AI。你站在交叉点上。

### 1.3 AI能力与四大目标的关系

```
投资成功 ← AI加速研究、辅助决策
    ↑
事业进步 ← AI赋能科技金融框架 → 建立行业影响力
    ↑
AI能力 → 家庭幸福 ← AI辅助子女教育规划、家庭事务管理
    ↓
健康管理 ← AI分析体检数据、运动饮食跟踪
```

AI能力不是"又多了一个目标"，而是**其他四个目标的效率倍增器**。

## 2. AI能力全景图

### 2.1 四层能力模型

```
┌─────────────────────────────────────────────┐
│ 第4层：领导层                                │
│ 带领团队用AI，在行内推动AI应用文化             │
│ "我们部门用AI做了这个，效果很好"               │
├─────────────────────────────────────────────┤
│ 第3层：整合层                                │
│ 将AI融入日常工作流，形成稳定的方法论            │

├─────────────────────────────────────────────┤
│ 第2层：应用层                                │
│ 熟练使用核心AI工具，解决具体问题               │

├─────────────────────────────────────────────┤
│ 第1层：认知层                                │
│ 理解AI能做什么、不能做什么、发展趋势            │
│ "这个场景适合用AI，那个不适合"                 │
└─────────────────────────────────────────────┘
```

### 2.2 核心工具链（面向金融管理者）

| 工具类别 | 代表工具 | 适用场景 | 优先级 |
|----------|----------|----------|--------|
| **大语言模型** | Claude、GPT | 写作、研究、分析、头脑风暴 | 最高 |
| **AI编程助手** | Claude Code、Cursor | 数据分析脚本、自动化工具 | 高 |
| **AI搜索/研究** | Perplexity、秘塔搜索 | 行业动态、竞品调研、政策解读 | 高 |
| **AI办公协同** | WPS AI、Notion AI | 文档、会议纪要、演示文稿 | 中 |
| **AI数据分析** | 自然语言问数、AI+Excel | 业务数据分析、报表生成 | 中 |
| **AI知识管理** | 个人知识库+AI检索 | 学习笔记、工作经验沉淀 | 中 |

### 2.3 能力边界：知道"不做什么"比"做什么"更重要

AI当前**擅长**的事：
- 信息汇总、文本生成、翻译润色
- 结构化分析（对比、归纳、分类）
- 代码编写和数据分析脚本
- 已知模式的识别和预警

AI当前**不擅长**的事：
- 需要真实判断力的决策（信贷审批的最终签批）
- 需要人际关系洞察的沟通（客户谈判、团队管理）
- 需要承担责任的判断（合规意见、风险敞口决策）
- 真正原创性的战略思考

**记住**：AI是你的分析师和笔杆子，决策者仍然是你。

## 7. 百日AI实战计划（2026年6月-9月）

> 不是让你多学一样东西，是帮你用AI把已经在做的事做得更快更好。

### 7.1 四阶段递进

#### 第1-2周：工具精通（6月1日-14日）

**Week 1：Claude Code 高级功能**

| 练习项 | 具体操作 | 验证方式 |
|--------|----------|----------|
| Agent 模式 | 给一个多步骤任务（如"把这份尽调报告的Markdown生成PDF，用STHeiti字体，中文字体注册要处理好"），让Claude Code自主规划→执行→验证 | 看Agent是否完成了完整链条，中间有没有需要你插手的地方 |
| 并行Agent | 同时启动两个Agent，一个更新投资路线图文档，一个更新健康计划文档。两个任务互相独立 | 两个Agent都完成，输出文件都有更新 |
| 记忆系统实操 | 把自己的一个偏好（如"所有PDF用STHeiti字体、12pt字号"）写入CLAUDE.md，然后开新对话验证Claude Code是否记住了 | 新对话中不提字体要求，让Claude Code生成PDF，看它是否主动用了STHeiti |

**Week 2：Prompt Engineering 实战**

| 练习项 | 具体操作 | 验证方式 |
|--------|----------|----------|
| 目标描述 vs 步骤描述 | 同一个任务写两遍：一次用"先做A，再做B，然后C"的步骤描述；一次用"最终产出是X，验收标准是Y"的目标描述。对比输出质量 | 目标描述的版本是否更完整、更少需要你纠正 |
| 角色扮演提示词 | 给Claude Code指定一个角色（"你是邮储银行科技金融事业部的评审委员，正在评估一家半导体设备企业的授信申请"），看它是否自动切换到那个角色的视角来分析 | 输出是否体现了评审委员的视角（关注现金流、抵押物、行业周期），而不是泛泛的分析 |
| 验收标准嵌入 | 在提示词末尾加一句验收条件（如"报告中每个风险提示必须有具体数据支撑，不能空泛地说'市场竞争激烈'"），看输出是否达标 | 输出是否符合了验收条件 |

#### 第3-4周：投资场景实战（6月15日-28日）

**任务1：个股深度分析（选1只你持仓或关注的股票）**

不追求覆盖13位大师，聚焦一件事：让AI帮你把一只股票分析透彻。

流程：你选定股票和你的关注点 → AI出业务模式+财务分析+行业竞争+风险提示+估值五个模块 → 你用金融经验审阅 → 标注"哪些分析到位、哪些你需要补充"

关键：这是你的主场。AI的分析框架是工具，你的金融判断是灵魂。比如AI可能只看财报数字判断"现金流健康"，但你在银行系统里知道这家企业的实际回款周期远超同业——这就是AI做不到、你能做到的地方。

**任务2：持仓复盘**

把当前持仓清单（股票名称+仓位占比+持有成本）给AI，让它做：
- 行业集中度分析：你重仓了哪个行业？单一行业敞口是否过大？
- 市值分布：大小盘比例是否合理，和你投资框架里的风险偏好是否匹配
- 大师框架对照：用你最认可的1-2位大师的原则（如巴菲特的护城河、格雷厄姆的安全边际），检查持仓

**任务3：行业横向对比**

选一个你因工作关系已经有所了解的行业（如光伏、半导体设备、新能源），让AI对比3-5家代表性公司。输出一张对比表，至少包含：收入规模、毛利率趋势、研发投入占比、应收账款周转天数、核心客户集中度。

#### 第5-8周：事业场景实战（6月29日-7月26日）

**任务1：科技金融业务报告**

选题建议（三选一，选你当前工作中最可能用到的）：
- "科技型中小企业授信的难点与对策"——这个你每天都在做
- "AI赋能科技金融的风险评估框架"——紧扣你的AI能力建设
- "XX行业（光伏/半导体/生物医药）信贷风险评估要点"——选你部门当前重点关注的行业

流程：你和AI讨论确定报告框架 → AI出初稿 → 你审阅修改 → AI润色定稿。目标：3000-5000字，达到可直接用于行内汇报的水准。

**任务2：竞聘演讲稿（选一个真实或假设场景）**

场景可以是：行内竞聘答辩、季度工作总结汇报、行业交流会发言。

流程：
1. 让AI先分析你的背景和目标岗位的匹配点（优势在哪、哪些需要补充论述）
2. AI出演讲框架（开场→个人优势→工作设想→收尾）
3. AI生成逐页演讲稿（1500-2000字）
4. AI模拟5-10个评委会问的问题，你准备回答

额外练习：把演讲稿读给AI听（粘贴文字），让它从评委视角提修改建议。

**任务3：监管政策解读**

选一份最近发布的与你工作直接相关的监管文件（金融监管总局、人民银行、证监会的新规），让AI做三件事：
- 政策要点提炼（一页纸，只列跟科技金融业务相关的条款）
- 对业务的影响分析（具体到"这个条款意味着我们的尽调流程要增加什么/减少什么"）
- 合规建议（列出3-5条你部门应该注意的事项）

目标是产出一份3页以内的政策速读笔记，格式清爽，可以直接转发给同事。

**任务4：PPT生成**

把任务1的报告或任务2的演讲稿转成PPT大纲+逐页内容。不用纠结AI排版（那方面AI还不成熟），重点产出是：每页的标题、3-5个要点子弹、需要配什么数据/图表。

#### 第9-12周：体系化（7月27日-9月30日）

**任务1：建立个人AI工作流**

拿出一张纸（或用Markdown），画出你的"AI工作流程图"：

```
每周一早上：AI扫描上周行业动态 → 生成速览（5条以内）
每次写报告前：AI出框架 → 你确认 → AI填充 → 你定稿
每次尽调前：AI跑一遍企业公开信息汇总（工商、司法、舆情、行业地位）
每月底：AI汇总本月持仓变动+行业轮动 → 生成投资月报
```

不需要一口气设计完美。先写3个你最常用的流程，实践两周，再调整。

**任务2：建立个人Prompt模板库（≥10个场景）**

从过去三个月的实战中，把每次用过的有效提示词整理出来。建议用以下格式：

```

提示词："请从业务模式、财务分析、行业竞争、风险提示、估值五个维度，
分析[股票名称]。每个风险提示必须有具体数据支撑，估值部分需给出保守/
中性/乐观三种情景。报告不少于3000字。"
典型输出：[链接到一次实际输出]
使用心得：AI的估值部分偏乐观，实际使用时需要自己调低增长率假设。
```

10个建议场景（从你最常遇到的开始）：

4. 最新政策解读

7. 邮件/公文起草
8. 体检报告分析

10. 会议纪要整理

**任务3：搭建1个自动化脚本**

从你当前最重复的劳动中选一个，让AI写脚本替代。候选：
- 周报自动汇总：从多个Markdown文件中提取"本周更新"标记的内容→汇总成周报初稿
- 持仓看板：输入持仓Excel/CSV→输出行业分布饼图+市值分布+收益率排序表
- 体检指标年度对比：输入最新体检数据→自动对比历史记录→标注变化>10%的指标

目标：跑通"源数据→脚本→输出结果"的闭环。只要一个脚本能稳定运行就算成功。后续可以在这个基础上复制模式做更多。

**任务4：准备行内分享**

主题："AI赋能科技金融——一个业务管理者的实战心得"

内容框架：

**提前准备**：分享前一周，让AI模拟"听众可能问的10个问题"，你准备好回答。最可能出现的问题——"AI会不会取代我们的工作""客户数据能输入AI吗""AI分析错了谁负责"——提前想好答案。

### 7.3 AI工具栈建议

不贪多。以下4个工具覆盖你的四大场景，就已经够了。多一个工具就多一份维护成本，工具在精不在多。

| 工具 | 角色定位 | 覆盖场景 | 费用参考 |
|------|----------|----------|----------|
| **Claude Code** | 你的AI中枢。承担投资分析、脚本自动化、文档生成、知识管理等复杂任务 | 投资研究、报告撰写、数据脚本、知识管理、健康数据分析 | Claude Pro $20/月 |
| **Claude/ChatGPT 网页版** | 轻量级助手。承担日常问答、快速写作、头脑风暴 | 邮件起草、家庭事务咨询、快速信息查询 | ChatGPT Plus $20/月（可选，与Claude Code组合已能覆盖大部分需求） |
| **Perplexity / 秘塔搜索** | AI搜索引擎。特点是会标注信息来源，你作为金融从业者需要核实信息出处 | 行业动态追踪、竞品调研、政策原文检索、子女教育信息查询 | Perplexity Pro $20/月 或 秘塔搜索免费版 |
| **WPS AI（行内办公环境）** | 办公文档AI辅助。在银行办公电脑上直接使用，不需要切换设备 | 会议纪要整理、Word报告排版润色、PPT大纲生成 | 视行内IT采购情况，通常已有 |

**各场景最佳组合**：

| 场景 | 推荐组合 | 典型流程 |
|------|----------|----------|
| **投资研究** | Perplexity + Claude Code | Perplexity查最新行业动态和公司公告（标注来源）→ Claude Code做深度分析、多公司对比、生成报告 |
| **事业提效** | Claude Code + WPS AI | Claude Code出报告/演讲稿/政策解读的初稿（内容深度）→ WPS AI做格式润色和排版（行内环境可直接用）→ 你在WPS里最终定稿 |
| **家庭管理** | Claude Code / ChatGPT网页版 | 场景分散且低频（子女教育规划、升学分析、家事备忘），不需要固定工具链。按需打开网页版提问即可 |
| **健康管理** | Claude Code | 体检报告数据输入→AI对比历史趋势→标注异常指标→给出针对性建议。这种需要保留历史数据做对比的场景，Claude Code（有文件记忆）优于一次性对话工具 |

**选工具的三条原则**：

1. **不追新**。新工具出来先观望2-4周，等有真实用户评价再决定。你不需要做"最早吃螃蟹的人"，做"最早把螃蟹吃出营养的人"
2. **优先选能打通现有工作流的**。Claude Code已经是你工作流的中枢，新工具能否和它配合是最高权重
3. **一个月用不到3次的工具不付费**。"看起来很好"不等于"真的会用到"。先用免费版试一个月，确认高频使用再付费

### A. 核心理念（常读常新）
