# 快速掌握AI手册

> 适用对象：非技术背景，想用AI加速工作的成年人
> 核心理念：不是学AI，是用AI。不学原理，学用法。

---

## 零、 一个前提：你的工作方式将被彻底改变

你不需要学编程。不需要学数学。不需要理解神经网络。

你只需要接受一件事：**你的角色从"做事的人"变成"指挥事的人"。**

| 以前你的工作方式 | 以后你的工作方式 |
|----------|------------|
| 自己读50页文件，花1小时 | 把文件给AI，30秒拿到摘要 |
| 自己写报告，从空白页开始，花3小时 | 给AI大纲和参考，5分钟出初稿，你花20分钟润色 |
| 自己搜索对比信息，花2小时 | 给AI一个对比指令，3分钟拿到表格 |
| 自己检查文档错误，逐行看 | AI自动扫描，列出所有问题供你确认 |
| 一天做1-2件"产出型"工作 | 一天做5-8件，因为你只管判断和决策 |

**AI不是替代你——是替代你工作中"不需要判断力"的部分。** 你的判断力、经验、人脉——这些AI没有的东西——变得更重要。

三条铁律：
1. **交代背景** — 你是谁，你要做什么，为什么
2. **说清要求** — 什么格式、什么长度、什么风格
3. **给例子** — 哪怕一个很烂的例子，也比没有强

```
❌ "帮我写个报告"
✅ "我是银行科技金融部门负责人，需要写一份2000字的报告，
   向行领导汇报AI产业链的信贷机会。参考我之前写的这份报告的风格。
   [附上参考报告]"
```

## 一、 第一周：用起来（每天15分钟）

### Day 1：聊天式入门

打开Claude或任何AI对话工具。用自然语言聊天，像你在和同事说话。

今天做这3件事：

**1. 让它自我介绍**
> "你能帮我做什么？列出10个我最可能用到的功能。我是银行对公业务人员。"

**2. 让它帮你改写一段文字**
> "下面这段是我写的工作汇报开头，帮我改得更简洁有力：[粘贴文字]"

**3. 让它总结一份文件**
> "帮我用三句话总结这份文件的核心内容：[粘贴文件内容]"

**Day 1 检查点**：你完成了3次对话，拿到了3个有用的结果。

### Day 2：学会上传文件

AI最强大的能力是"读文件"。

**今天做这件事**：把正在看的一份工作文档（PDF/Word/图片）上传给AI，说：
> "这是我明天开会要用的一份文件。帮我：①用200字总结核心内容 ②列出3个可能被领导追问的问题 ③给我一个1分钟的口头汇报稿"

**Day 2 检查点**：你上传了第一份文件，AI帮你处理了。

### Day 3：让AI帮你写东西

**今天做这件事**：让AI帮你写一份你本来打算自己写的文字。

> "帮我写一份给客户的邮件：我们要约他下周见面，讨论智算项目的融资方案。语气专业但不过于正式。他知道我是谁，所以不用自我介绍。"

**Day 3 检查点**：AI帮你写了一份你本来要花30分钟写的文字。你只需要修改润色。

### Day 4：让AI帮你做研究

**今天做这件事**：问一个你本来需要花1小时搜索的问题。

> "帮我对比一下华为昇腾和寒武纪的AI芯片，从三个维度：①技术路线 ②商业化进展 ③和英伟达A100的差距。数据截至2026年5月。用表格呈现。"

**Day 4 检查点**：你在5分钟内拿到了相当于1小时搜索的信息。

### Day 5：让AI分析你的工作

**今天做这件事**：把AI变成你的"外部大脑"，分析你自己。

> "我想梳理一下我工作中最浪费时间的三件事。帮我问5个问题来帮我诊断。一次问一个。"

回答完5个问题后，让它输出分析结果和改进建议。

**Day 5 检查点**：AI帮你看清了你自己没注意到的问题。

### Day 6-7：自由练习

用这两天尝试把AI用在你想用的任何地方。写孩子的家长会发言稿、计划周末行程、分析一只股票、修改一段文字——什么都行。

**唯一的要求**：每天至少有1次对话超过3轮。因为AI在对话中会越来越懂你。

## 二、 第二周：掌握7个高频场景（每天1个）

### 场景1：帮我读
> "这是[文件类型]。帮我提炼：[具体要求]"

- 读合同 → "标出对我方不利的条款"
- 读研报 → "用表格列出核心数据和观点"
- 读财报 → "和去年同期对比，哪三项变化最大"

### 场景2：帮我写
> "帮我写一份[文体]，给[读者]看的，长度[字数]，语气[正式/口语/...]"

- 工作汇报、客户邮件、朋友圈文案、演讲稿、会议纪要

**关键技巧**：写完第一版后，追加指令——"第二段太啰嗦了，压缩到一半"、"换个更直接的开头"

### 场景3：帮我搜
> "帮我整理关于[主题]的信息。重点看[维度1/2/3]。给出信息来源。"

- 行业动态、公司背景、政策梳理、竞争对手分析

### 场景4：帮我想
> "帮我从[角度]分析[问题]。列出至少3种可能的解释/方案，各附优缺点。"

- 问题诊断、方案生成、决策辅助、盲点扫描

### 场景5：帮我改
> "下面这段文字，帮我改进[具体方面]：[粘贴文字]"

- 改简洁、改准确、改语气、改结构、找漏洞

### 场景6：帮我翻译
> "帮我把这份[原文语言]文件翻译成[目标语言]，要[正式/口语]，保留[原文的什么特质]"

AI的翻译质量已接近专业水平，且可以同时处理几十页文档。

### 场景7：帮我对
> "下面A和B是两个版本的[方案/数据/表述]，帮我对比差异，找出不一致的地方"

- 合同版本对比、数据一致性检查、文档与文档之间的交叉验证

## 三、 第三周：进阶技巧

### 3.1 用"角色设定"提升输出质量

在任务前加一句话，AI的输出质量会显著不同：

```
"假设你是麦肯锡的资深顾问，专长是金融行业战略。帮我分析..."
"假设你是科技公司CTO，帮我评估这个技术方案的可行性..."
"假设你是《财经》杂志的编辑，帮我审这篇稿子..."
```

### 3.2 用"思维链"解决复杂问题

不要一次问一个巨大的问题。分步骤来：

```
第一步："帮我梳理银行科技金融面临的5个核心挑战。不着急分析，先列出是什么。"
第二步："针对第3个挑战，深挖一层：这个挑战的根因是什么？"
第三步："针对这个根因，如果有3种方案，分别是什么？"
第四步："方案B的风险是什么？"
```

### 3.3 用"多版本"做决策

让AI给出多个版本而不是一个答案：

```
"针对这个问题，给我3种不同风格的回复：
 A版：大胆激进
 B版：稳妥保守
 C版：两者之间
我选一个，或者让你融合两个版本。"
```

### 3.4 建立你的提示词库

把你用得好的指令保存下来。每次用完AI后，如果效果特别好，花10秒把这句话复制到备忘录。

一周积累5-10条你的专属提示词。一个月后，你会有一个高度个性化的"AI话术库"。

## 四、 认识AI Agent——从工具到搭档

### 4.1 什么是AI Agent

你之前用的AI是"问答模式"——你问，它答。AI Agent是"执行模式"——你给目标，它自己规划步骤、调用工具、检查结果、修正错误。

```
传统AI（问答模式）：你→提问→AI→回答→你→再提问→AI→再回答...
AI Agent（执行模式）：你→给目标→AI自己：规划→执行→检查→修正→完成→报告给你
```

**一句话**：Agent不是"你问一句它答一句"的助手，而是"你交代一个目标，它自己想办法完成"的搭档。

### 4.2 Agent能做什么（你今天就能用的）

| 能力 | 传统AI | AI Agent |
|------|--------|----------|
| 读文件 | 你上传一份，它分析一份 | 自己搜索整个项目，找到相关文件，交叉比较 |
| 写代码 | 输出代码让你复制粘贴 | 自己写代码、运行、看报错、修正、再运行 |
| 多步骤任务 | 你一步步引导 | 你布置目标，它自主拆解并执行 |
| 并行工作 | 一次做一件事 | 同时启动多个子Agent，各自负责一部分 |
| 使用工具 | 只能对话 | 可以调用外部工具、数据库、API |

### 4.3 你已经在用Agent了

如果你用过Claude Code（见《快速学习Claude Code手册》），你就已经在用Agent了。Claude Code就是一个Agent——它能：

- 自己读取项目文件，理解结构
- 自己决定用什么命令来完成任务
- 修改多个文件，跑测试，确认改动正确
- 启动多个子Agent并行处理不同任务

**你不需要学新工具。你只需要学会"给Agent布置目标"而不是"给Agent布置步骤"。**

### 4.4 给Agent布置任务的技巧

```
❌ "帮我搜索文件"（这是给工具下指令）
✅ "帮我找到项目中所有和昆仑万维持仓相关的数据，检查数据一致性"（这是给Agent布置目标）

❌ "运行test.py"
✅ "运行所有测试，如果有失败的，分析原因并尝试修复"

❌ "git commit"
✅ "检查今天的改动，写一条合适的commit message并提交"
```

**核心转变**：从"告诉它怎么做"到"告诉它你要什么结果"。Agent会自己解决"怎么做"。

### 4.5 Multi-Agent与MCP：不止一个Agent，还能接外部工具

2026年AI的两个最新发展：

**Multi-Agent（多Agent并行）**：多个Agent同时工作，各自负责一部分：

```
你：同时做三件事
  ├─ Agent 1：搜索所有研报中关于AI芯片的数据
  ├─ Agent 2：对比华为昇腾和寒武纪的最新进展
  └─ Agent 3：生成一份投资分析报告框架
```

5分钟后，三个Agent都完成了，你拿到整合后的结果。**这相当于你有了一个团队，而不是一个助理。**

**MCP（Model Context Protocol，模型上下文协议）**：让AI连接外部工具和数据源的开放标准。比如AI可以直接查数据库、调用API、读取企业系统——不再只靠对话。

> 你不需要理解MCP的技术细节。你只需要知道：**2026年的AI不再是"对话机器人"，而是能接入你工作系统的"数字员工"。**

### 4.6 你的角色变了——从操作者到指挥者

| 以前你 | 以后你 |
|--------|--------|
| 自己查资料、写初稿、改格式 | 给Agent目标，它产出初稿，你判断和润色 |
| 一次做一件事 | 同时启动多个Agent，并行推进 |
| 操作工具（搜索、编辑、运行） | 指挥Agent（描述目标、检查结果、决策） |
| 工作时间=产出时间 | 工作时间=判断时间，产出在后台自动进行 |
| 能力上限=个人能力 | 能力上限=个人判断力×Agent执行力 |

**Agent是执行力，你是判断力。速度×方向=力量。**

### 4.7 Agent的边界（知道什么不能交给它）

| 能交给Agent的 | 不能交给Agent的 |
|-------------|---------------|
| 信息搜集、整理、对比 | 最终投资决策 |
| 文档撰写、修改、格式化 | 需要人际信任的沟通 |
| 代码编写、测试、调试 | 价值判断和优先级排序 |
| 数据分析和可视化 | 需要承担责任的签字决策 |
| 多文件批量修改 | 需要直觉和经验的判断 |

## 五、 长期：AI能力的三个段位

### 青铜（第一个月）：AI是你的"超级实习生"

- 你能熟练用AI读文件、写东西、搜信息
- 每天都用AI，但主要是"你给我做什么"的单向指令
- 关键标志：你不需要搜索怎么提问，本能就能说出指令

### 白银（第三个月）：AI是你的"外部大脑"

- AI了解你的工作上下文——它记得你的行业、你的角色、你的偏好
- 你开始和AI"对话"而不是"下指令"——它会追问、会质疑、会给意外的好建议
- 关键标志：有几次AI说出来的话，让你觉得"我没想到这个角度"

### 黄金（第六个月）：AI是你的"能力放大器"

- 你的工作效率比不用AI的同事快2-5倍（尤其是读写分析类工作）
- AI帮你做掉了"必须做但不喜欢做"的80%工作
- 你把省下来的时间投在AI做不到的事——见人、决策、感受、判断
- 关键标志：如果有一天不能用AI，你会感觉像少了一半团队

### 钻石（第十二个月）：Agent是你的"执行团队"

- 你不再给AI下指令，而是给Agent布置目标——它自己规划并执行
- 你同时启动多个Agent并行工作：一个做研究、一个写方案、一个检查数据
- 你的角色从"操作者"变成"指挥者"——你判断优先级，Agent负责执行
- 关键标志：你早上布置3个目标，下午看结果。你的产出量等于一个小团队

## 六、 使用红线

| 红线 | 说明 |
|------|------|
| **不输入密码/银行卡号/身份证号** | AI对话不是保险箱 |
| **不把AI的回答当权威** | AI会犯错、会编造、会过于自信。关键信息必须核实 |
| **不在情绪极端时做决策** | 不要因为AI说"这只股票前景很好"就买入 |
| **不替代专业判断** | 医疗、法律、重大财务决策——AI是参考，不是替代 |
| **不依赖AI思考** | AI帮你整理思路，但最终的判断和决策必须是你自己的 |
