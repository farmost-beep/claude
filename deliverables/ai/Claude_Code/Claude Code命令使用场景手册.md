## 场景1：修改文档后重新生成全部PDF {#场景1}

**触发条件**：你改了任意 deliverable 目录下的 Markdown 文件。

**指令**：

```
运行 scripts/build_goal_pdfs.py 生成最新PDF
```

**预期结果**：15个PDF全部重新生成，Claude Code 会列出每个PDF的文件路径和大小。

**变体**：

```
只生成投资目标的PDF
```

```
运行 build_goal_pdfs.py，但只生成 investment 和 career 两个
```

**注意事项**：
- 生成后检查 PDF 文件大小是否正常（通常5-50KB）
- 如果报字体错误，检查 STHeiti Medium.ttc 是否在系统字体目录

## 场景2：批量更新六大目标文档 {#场景2}

**触发条件**：你有一个改动需要同时反映到多个目标的文档中。

**指令模板**：

```
同时更新投资/事业/家庭/健康/AI/知识库六个目标的路线图和操作手册，改动内容：[描述]
```

**实际案例**：

```
同时更新六个目标的路线图，在每个文档末尾追加 "## 6月执行重点" 章节，
其中投资目标写"9月信用贷60万还本准备"，事业目标写"中级经济师报名准备"，
家庭目标写"子女考研暑假计划"，健康目标写"基线测量完成"，
AI目标写"Week 1-4执行"，知识库目标写"MOC框架填充"
```

**注意事项**：
- 六个目标之间无依赖时可并行，有依赖时分两批
- 并行完成后，核对每个 Agent 的输出是否真的写入了文件（用 `ls -la` 验证）

## 场景4：单只股票投资分析 {#场景4}

**触发条件**：你想对某只持仓或候选股票做深度分析。

**指令模板**：

```
参考13位投资大师的分析框架，对[股票名称/代码]做一次完整的投资分析。重点从以下维度评估：
1. 商业模式（波特五力 + 护城河）
2. 财务健康（ROE、自由现金流、负债率、近5年趋势）
3. 估值（PE/PB历史分位、DCF粗略估算）
4. 管理层（持股比例、历史决策记录、诚信度）
5. 风险（行业风险、政策风险、汇率风险）
请给出明确的持有/增持/减仓建议，并说明置信度。
```

**变体——快速估值**：

```
用格雷厄姆公式和PE历史分位法快速估值[股票名称]，告诉我是低估/合理/高估
```

**变体——财务排雷**：

```
用[股票名称]的最新财报数据，检查是否有财务造假迹象：
应收账款增速 vs 收入增速、经营现金流 vs 净利润、存货周转天数变化趋势
```

## 场景5：多只股票并行研究 {#场景5}

**触发条件**：你要同时评估持仓组合中的多只股票。

**指令模板**：

```
同时开6个Agent，分别研究以下6只股票，每只输出一份500字以内的分析摘要：
1. 昆仑万维——AI业务进展和估值
2. 腾讯——游戏+广告+云三大业务线拆分估值
3. 茅台——消费降级对高端白酒的影响
4. 美的——出海业务进展
5. 招商银行——净息差和不良率趋势
6. 宁德时代——产能过剩和海外市场
所有Agent完成后汇总，给出组合层面的结论。
```

**注意事项**：
- 明确让每个Agent输出"500字摘要"，避免被Agent的长篇输出淹没
- 汇总阶段关注跨股票矛盾（如一个说消费复苏、一个说消费降级）

## 场景8：更新操作手册中的具体数据 {#场景8}

**指令模板**：

```

```

**实际案例**：

```

```

```

```

**建议模式**：一次性把同一目标的所有新数据一起说，避免多次小改动。比如上面的第二个例子就是一次说完4个数据点。

## 场景11：开启一个干净的新对话 {#场景11}

**触发条件**：对话太长、上下文太杂、或者Claude Code开始混淆前面的内容。

**做法**：直接关闭当前对话窗口/终端，打开一个新的。第一句话带上关键上下文：

**模板**：

```
继续推进 [目标领域]。当前状态：[1-2句关键背景]。
今天要做：[具体任务]。
```

**实际案例**：

```
继续推进投资目标。我是陈颖fang，49岁，当前净资产383万，目标60岁前金融资产900万+。
今天要：更新昆仑万维的持仓分析，基于Q1财报重新评估。
```

**什么时候必须开新对话**：
- 对话已跨了3个以上不相关的目标领域
- Claude Code 反复引用几百条之前的旧信息来回答新问题
- 你发现自己在Prompt里写了很长的"前情提要"

## 场景12：定时循环任务 {#场景12}

**触发条件**：你希望某个任务被定期自动提醒/执行。

**指令模板**：

```
/loop [间隔] [任务描述]
```

**实际案例**：

```
/loop 5d 并行推进目标，直到目标达成
```

```
/loop 1w 检查投资组合是否触发再平衡条件
```

```
/loop 3d 更新Claude Code使用手册
```

**支持的间隔**：
- `Nm`：每N分钟（如 `30m`）
- `Nh`：每N小时（如 `6h`）
- `Nd`：每N天（如 `5d`）
- `Ns`：每N秒（会向上取整到分钟）

**注意事项**：
- 定时任务7天后自动过期，需要重新设置
- 可以在任意对话中用 CronDelete 取消
- 非持久化——关闭Claude Code后任务就没了

## 场景14：跨文档一致性检查 {#场景14}

**触发条件**：你怀疑多个文档里的同一个数据不一致。

**指令模板**：

```

```

**重点检查项**：
- 净资产数字（383万）
- 投资目标（900万+）
- 六大目标状态描述
- 关键日期（审批日、还本日、考试日）

**实际案例**：

```
检查投资自由路线图.md、投资自由操作手册.md、Obsidian Vault投资自由路线图.md，

## 场景15：代码/脚本排错 {#场景15}

**触发条件**：脚本报错，你需要排查修复。

**指令模板**：

```

```

**实际案例**：

```
运行 build_goal_pdfs.py 时报错：
  File "build_goal_pdfs.py", line 197, in markdown_to_pdf
    pdf.output(pdf_path)

- 如果是在调试迭代中，告诉Claude Code"这是第X次尝试，上一次我们做了Y改动"

## 场景17：git 提交流程 {#场景17}

**触发条件**：你完成了一批改动，要提交到 git。

**指令**：

```
git commit
```

**系统行为**：Claude Code 会：
1. 运行 `git status` 查看变更文件
2. 运行 `git diff` 看具体改动
3. 运行 `git log` 参考历史提交风格
4. 起草提交信息，让你确认后提交

**你不会看到的行为**：Claude Code 不会主动提交、不会push、不会force push——这些都需要你明确指令。

## 场景18：快速问答 vs 深度任务 {#场景18}

**判断标准**：这句话能不能一句话说清楚？

| 你的话 | 判定 | Claude Code的做法 |
|--------|------|-------------------|
| "这个PDF有多大" | 快速问答 | 直接回答 |
| "帮我查昆仑万维的PE" | 快速问答 | 直接回答 |
| "分析昆仑万维是否值得持有" | 深度任务 | 规划步骤→读财报→跑大师模型→输出报告 |
| "更新六个目标的6月计划" | 深度任务 | 可能启动并行Agent |
| "帮我把这句话改正式一点" | 快速问答 | 直接回答 |
| "写一份支行科技金融业务拓展方案" | 深度任务 | 先问清楚受众/篇幅/时间，再动手 |

**规则**：快速问答不用Agent，直接对话即可。深度任务Claude Code会自动判断是否用Agent。你不需要指定——它会自己选。
