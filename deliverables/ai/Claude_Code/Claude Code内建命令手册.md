# Claude Code 内建命令使用场景手册

> 读者：陈颖芳 | 版本：v1.0 | 2026-05-27
>
> Claude Code 的内建命令以 `/` 开头，在对话中直接输入即可触发。本手册逐个说明每条命令的功能、使用时机和实战示例。

## 一、工作流核心命令

### 1.1 /clear — 清空对话，重新开始

**做什么**：清除当前对话的全部上下文，回到空白会话。

**什么时候用**：
- 当前对话太长了，Claude Code 开始混淆上下文、引用很久之前的信息
- 你完成了任务A，现在要做一个完全无关的任务B
- Claude Code 的行为变奇怪了（跑偏、偷懒、反复出错）
- 你想确保 Claude Code 从干净的 CLAUDE.md 出发思考

**什么时候不用**：
- 你还在同一个任务上迭代（清空了 Claude Code 就忘了之前的讨论）
- 错误可以通过一句纠正解决（不需要清空重来）

**实际场景**：

```
你：/clear
```
然后在新会话中说：
```
继续推进投资目标。我是陈颖芳，49岁，净资产~383万。
今天要：将昆仑万维减仓方案细化为可执行的逐日交易计划。
```

**注意事项**：`/clear` 不会删除文件，你的所有 deliverable 文档和记忆都在。它只是清空对话历史。

---

### 1.2 /compact — 压缩上下文

**做什么**：将当前对话历史压缩为摘要，释放上下文空间，但保留关键信息。Claude Code 会在上下文即将耗尽时自动建议压缩，你也可以主动触发。

**什么时候用**：
- 对话很长但你不想完全重来（有些中间结果还想保留）
- Claude Code 提示"上下文即将用完"
- 你刚完成了一个大阶段，准备进入下一阶段，但想保留部分上下文

**实际场景**：

```
你：/compact
```

压缩后，Claude Code 会保留：
- 你的身份信息（从 CLAUDE.md）
- 本轮对话中生成的关键文件路径
- 你做出的重要决策
- 当前未完成的任务

会丢弃：
- 中间的尝试和修正过程
- 已读过的文件内容（需要时会重新读取）

**与 /clear 的区别**：

| | /clear | /compact |
|------|---------|-----------|
| 保留信息 | 无 | 摘要保留 |
| 适用场景 | 换任务 | 同一任务继续 |
| 后续成本 | 需要重新描述背景 | 关键信息自动恢复 |

## 二、定时与任务管理

### 2.3 /loop — 定时循环执行任务

**做什么**：设置一个定时循环任务，Claude Code 每隔指定时间自动执行你设定的指令。

**语法**：

```
/loop [间隔] [要执行的任务描述]
```

**间隔写法**：
- `Nm`：每 N 分钟（如 `30m`，最大 59m）
- `Nh`：每 N 小时（如 `6h`）
- `Nd`：每 N 天（如 `5d`）
- 不写间隔默认 `10m`

**实际场景**：

```
/loop 5d 并行推进六大目标，检查进度并输出状态报告
```

```
/loop 1w 检查投资组合是否触发再平衡条件（单票偏离目标仓位±5%）
```

```
/loop 3d 提醒我更新Claude Code使用手册
```

**注意事项**：
- 定时任务 7 天后自动过期，需要重新设置
- 任务只在对话空闲时触发，不会打断正在进行的对话
- 终止方法：`CronDelete [任务ID]` 或对话结束（任务未持久化）
- 设置后立即执行一次，不会等到第一个间隔

---

### 2.4 /tasks — 查看后台任务状态

**做什么**：查看当前有哪些后台 Agent 在运行、有哪些定时任务在等待。

**什么时候用**：
- 你启动了多个并行 Agent，想知道进度
- 你想确认某个定时任务是否还在生效
- 你想停止某个跑偏的后台任务

**实际场景**：

```
你：/tasks
```

你会看到类似：

```
活跃的 Agent：
  [a0219313] 推进投资目标：9月还本方案 — 进行中
  [a13ca706] 推进事业目标：入库+职称策略 — 进行中
  ...

定时任务：
  [97ad3cb2] 每5天：更新Claude Code使用手册
  [0eb044dc] 每5天：并行推进目标，直到目标达成
```

---

### 2.5 /status — 当前状态总览

**做什么**：显示当前会话的基本信息——项目路径、模型、会话时长、token使用量。

**什么时候用**：
- 你感觉对话有点长，想看看是不是该 `/clear` 或 `/compact` 了
- 你想确认当前使用的是哪个模型

---

## 三、记忆与配置

### 3.6 /memory — 管理持久记忆

**做什么**：查看或操作 Claude Code 的 Auto Memory 系统。

**什么时候用**：
- 你想确认之前记住的规则还在不在
- 你想清理过时的记忆
- 你想查看当前所有的记忆文件

**实际场景**：

```
你：/memory
```

Claude Code 会展示当前项目的记忆索引（MEMORY.md 的内容），包括：
- 用户档案
- 反馈规则
- 项目约定
- 参考信息

**记忆的写入**：你不需要用 `/memory` 来写入记忆。直接说"记忆：[内容]"即可，Claude Code 会自动归类写入对应文件。

---

### 3.7 /config — 修改配置

**做什么**：打开或修改 Claude Code 的配置文件（settings.json）。

**什么时候用**：
- 你需要新增命令的自动放行权限
- 你需要修改环境变量
- 你需要调整 Claude Code 的行为设置

**实际场景**：

```
你：/config — 把 python3 scripts/build_goal_pdfs.py 加到自动放行列表
```

---

### 3.8 /fast — 切换快速模式

**做什么**：开关 Claude Opus 的快速输出模式。快速模式输出更快（生成 token 速度提升），但思考深度不变。

**什么时候用**：
- 任务比较简单、不需要深度推理（如"把这些文件重命名"）
- 你在快速推进多个简单任务，希望响应更快

**什么时候不用**：
- 复杂的投资分析、多步骤任务规划——让模型充分思考

---

## 四、代码审查与 IDE

### 4.9 /review — 审查当前变更

**做什么**：让 Claude Code 审查当前会话中的所有代码变更，输出一份变更总结。

**什么时候用**：
- 你改了几个文件，想在提交前做最后一次检查
- 你启动了多个并行 Agent，想统一审查它们的产出
- 你想确认没有引入安全漏洞或 破坏性变更

**实际场景**：

```
你：/review
```

Claude Code 会：
1. 检查本次会话修改了哪些文件
2. 审查代码质量和安全性
3. 检查是否违反了你设定的规则（如 Karpathy 原则）
4. 列出需要注意的问题

---

### 4.10 /ide — IDE 集成

**做什么**：在 VS Code 或 JetBrains IDE 中打开当前项目。

**什么时候用**：
- 你想用 IDE 的图形化 diff 工具查看变更
- 你想在 IDE 中手动编辑某个文件（Claude Code 的 Edit 工具不够直观时）

## 五、帮助与初始化

### 5.11 /help — 查看帮助

**做什么**：显示 Claude Code 的使用帮助和命令列表。

**实际场景**：

```
你：/help
```

---

### 5.12 /setup — 初始化向导

**做什么**：首次安装 Claude Code 后，引导你完成初始设置——包括连接账户、配置权限、了解基本用法。

**什么时候用**：首次使用时自动触发。之后基本不需要手动调用。

## 命令使用频率优先级

按你的实际使用模式，各命令的优先级排序：

### 每天都会用的

1. **`/loop`** — 你已经在用 `5d 并行推进目标`，这是最核心的自动化引擎
2. **`/tasks`** — 并行启动多个 Agent 后查看进度

### 每周用几次的

3. **`/compact`** — 长时间推进同一目标时
4. **`/clear`** — 切换目标领域时开新对话
5. **`/review`** — 批量改动后提交前审查

### 偶尔用到的

6. **`/memory`** — 检查或调整记忆规则
7. **`/status`** — 检查会话状态
8. **`/fast`** — 根据任务复杂度切换

### 基本不用管

9. **`/config`** — 权限放行已经配好了
10. **`/setup`** — 已完成
11. **`/help`** — 备用
12. **`/ide`** — 你有自己的编辑器习惯
