一、整体架构鸟瞰


核心公式

Agent = Model + Harness — Mitchell Hashimoto, 2026. Claude Code 约51.2万行TypeScript（~1,332 .ts + ~552 .tsx），基于Bun运行时。仅约1.6%是AI决策逻辑，98.4%是确定性基础设施[VILA-Lab, arXiv:2604.14228]。


六条核心设计原则

#原则一句话
1Generator-based 流式架构从API到UI全链路使用async function*
2分层防御安全7层独立防御，每层假设上一层可能被突破
3编译时特性门控Bun的feature()宏在构建时物理删除条件代码
4集中式状态与不可变更新Zustand式不可变状态更新
5渐进式压缩最便宜策略最先执行，类似CPU多级缓存
6工具作为扩展点统一Tool泛型接口，新工具只需实现接口

五层子系统

层名称职责
1SurfaceCLI、Headless、SDK、IDE — React + Ink自研渲染器
2CoreQueryEngine + query() 双层生成器
3Safety/Action7层纵深防御 + 权限规则 + 29 Hook事件 + 工具池
4StateAppend-only JSONL + CLAUDE.md四层 + Auto Memory
5BackendShell执行 + MCP连接 + 66+工具目录

9阶段启动流程

模块求值 → 模块加载 → CLI解析 → Commander设置 → preAction → 14步初始化 → Action处理 → 延迟预取 → 遥测记录（~235ms关键路径）


二、上下文工程管线


八个上下文来源

系统提示词→环境信息→CLAUDE.md层级→路径作用域规则→Auto Memory→工具元数据→对话历史→工具执行结果→压缩摘要


CLAUDE.md 四层体系

Managed（企业级）> User（全局）> Project（项目级，纳入版本控制）> Local（本地级，gitignored）。可靠容量≤200行。


5阶段压缩管线

阶段策略触发
1预算控制始终激活
2历史裁剪Snip功能开关
3微型压缩Microcompact始终激活（基于时间/缓存双路径）
4上下文折叠Context Collapse功能开关
5自动压缩Auto-Compact断路器，连续3次失败后停止

Microcompact双路径：缓存热时用缓存编辑路径（不修改本地消息），缓存冷时用基于时间路径（直接修改消息）。


无向量数据库记忆系统

基于LLM对记忆文件头部元数据扫描，每次最多5个文件。4种封闭类型：user/feedback/project/reference。MEMORY.md作为索引（200行/25KB双层截断）。语义召回管道：scanMemoryFiles() → formatMemoryManifest() → selectRelevantMemories()。


4层缓存稳定性防御

  1. 分割：系统提示词拆分确保前缀稳定
  2. 锁存：会话级beta header / TTL粘性
  3. 排列策略：工具数组中内置工具在前，MCP在后
  4. 缓存断裂检测：两阶段快照对比

三、工具系统


统一Tool接口 + buildTool工厂

所有工具通过Tool<Input, Output, P>泛型接口。buildTool()工厂函数使用TOOL_DEFAULTS合并安全默认值：默认非并发安全、默认非只读、默认需权限检查。

三层装配管道

Layer 1 getAllBaseTools() — 编译期裁剪，核心~20直接import + 条件~46通过Bun宏。Layer 2 getTools() — 运行时4层过滤（SIMPLE/REPL/Deny/isEnabled）。Layer 3 assembleToolPool() — 分区排序（内置前缀+MCP后缀）保缓存命中率。


8阶段工具执行生命周期

查找→验证→并行启动（PreToolUse Hook + Bash分类器）→权限检查（6来源+对话框）→执行→结果处理→PostToolUse Hook→消息发射


四、安全与权限体系


5外部+2内部权限模式

default（日常）/ acceptEdits（编辑自动） / plan（只读） / bypassPermissions / dontAsk / auto（ML分类器） / bubble（协调器）


权限规则系统

3匹配类型（精确/前缀通配/通配符），3行为（allow/deny/ask），8层优先级（policySettings→session），deny永远优先。拒接降级：连续3次或累计20次拒绝后强制回退。


三种权限处理器竞速

UI对话框+Hook检查+ML分类器同时启动，第一个有效决定生效。200ms防误触宽限期。用户操作后自动化结果丢弃。


7层纵深防御

  1. 工作区信任确认
  2. 权限模式
  3. 权限规则匹配
  4. Bash多层安全（AST+23项检查+12种命令替换模式+sed验证）
  5. 工具级安全（危险文件保护）
  6. 沙箱隔离（macOS Seatbelt/Linux命名空间）
  7. 用户确认

Fork子Agent提示词缓存经济学

Fork子Agent继承父级字节精确请求前缀，所有fork共享一个提示词缓存。buildForkedMessages()构建占位tool_result保持缓存一致性。

新增模块（v2.0）


技能系统

技能 = AI Shell脚本。6个来源：内置>文件系统>工作流>插件>MCP>外部。双重调用：用户手动/模型自动。5级信任模型。SKILL.md目录格式。

4核心工具。状态机：pending→in_progress→completed。依赖追踪。文件级存储。2种粒度锁。3层变更检测。多Agent协调。


UX设计

自研React终端渲染器（251KB）。StreamingMarkdown增量解析。Spinner 5模式状态机。Vim 4模式。REPL虚拟滚动。成本精度分档展示。
