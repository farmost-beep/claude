## 零、开始前：一句话理解Claude Code

**Claude Code = 终端里的AI程序员。** 你说话，它读写文件、执行命令、完成任务。

打开终端，输入 `claude`，回车。看到对话界面，就可以开始了。

但今天你学的不是一个工具。你学的是 Anthropic 公司内部用 AI 开发软件的方法论——18条经过验证的原则，浓缩到一天的实践中。

### 今天你会经历什么

| 今天之前 | 今天之后 |
|---------|---------|
| 修改文件 → 打开编辑器 → 找到位置 → 手动改 → 保存 | 说一句话，改完（原则1：AI直接操作生产环境） |
| 搜索内容 → 逐个文件打开看 | 说一句话，结果列出（原则3：信任执行，验证结果） |
| 跑命令 → 复制粘贴 → 看报错 → 搜索怎么修 | 说一句话，自动运行+修复（原则13：失败自动修复） |
| 提交git → 想commit message → git add → git commit | 说一句话，提交完成 |
| 一次只做一件事 | 同时交代多件事，AI并行完成（原则6：拆解+分发） |

**你今天不只是学工具，是换一种工作方式——从"操作电脑"到"指挥AI操作电脑"。**

### Anthropic的18条原则速览

| 优先 | 原则 | 一句话 | 案例中的应用 |
|------|------|--------|-------------|
| ⭐⭐⭐ | 原则2：给AI完整上下文 | CLAUDE.md是你的项目说明书 | 告诉AI日志文件的路径和格式要求 |
| ⭐⭐⭐ | 原则8：给目标不给步骤 | "完成X"而非"先A后B再C" | "帮我从零创建一个工作日志项目" |

## 上午：建立人与AI的正确关系（2小时）

> 覆盖原则：1-5（人与AI的关系层）

### 9:00-9:15 第一个对话——体验原则1

**原则1：AI必须能直接操作生产环境，而非仅输出建议。**

Claude Code 不是给你建议，是直接帮你做。你现在在一个空目录里，什么都没有。让AI帮你迈出第一步。

> **案例第1步**——打开终端 → `claude` → 输入下面这句话。你会看到AI自动创建项目结构。这是你和AI的第一次握手。

**在Claude Code中输入：**

```input
我当前在一个空目录里。帮我从零创建一个工作日志项目：
1. 初始化git仓库
2. 创建一个 工作日志_202605.md 文件，写入标题"# 每日工作日志"和今天的日期
3. 告诉我项目结构是什么样的
```

Claude Code 会自动运行 `git init`、创建文件、写入内容，然后回答你。你不需要知道具体命令——这就是"给目标不给步骤"。

### 9:15-9:45 三个核心指令

**指令1：帮我读**

> **案例第2步**——刚才创建了日志文件，现在让AI读一下，确认格式符合预期。这也是后面追加内容前必须的步骤。

**在Claude Code中输入：**

```input
读取 工作日志_202605.md 的完整内容，告诉我当前的格式结构是怎样的
```

**指令2：帮我找**

**在Claude Code中输入：**

```input
在当前目录下搜索所有包含"TODO"或"待办"的文件，列出文件名和行号
```

**指令3：帮我写**

**在Claude Code中输入：**

```input
在当前目录下创建一个 日志模板.md 文件，内容是每日日志的标准格式：
日期、今日完成、遇到的问题、明日计划。每项用二级标题。
```

### 9:45-10:15 让AI理解你——实践原则2

**原则2：AI需要完整上下文，而非片段。CLAUDE.md = 给AI的项目README。**

每次给Claude Code布置任务，说清四个信息：

| 信息 | 为什么重要 | 案例中的例子 |
|------|----------|------------|
| 文件在哪 | 原则2：完整上下文 | "在当前目录下的工作日志_202605.md中" |
| 改什么 | 原则8：明确目标 | "整理今天的工作要点，筛选最重要的3-5条，追加到文件末尾" |
| 为什么 | 原则2：理解意图 | "这是我每日必做的工作记录，格式要保持一致" |
| 什么格式 | 原则3：验收标准 | "每条包含：日期+事项标题+一句话总结+下一步行动，用---分隔" |

> **案例第3步**——发出你的第一个"四信息完整指令"，让AI帮你写今天第一条工作日志。这就是你今天的第一个正式产出。

**❌ 模糊指令（不要这样写）：**

```input
帮我写个工作日志
```

**✅ 完整指令（在Claude Code中输入）：**

```input
帮我整理今天的工作要点，包括完成的任务、参加的会议、遇到的问题。
筛选最重要的3-5条，以"日期+事项标题+一句话总结+下一步行动"格式，
追加到当前目录下的 工作日志_202605.md。
格式参照文件中已有的条目，保持风格一致。
```

### 10:15-10:45 第一项实战任务——体验原则3

**原则3：信任AI执行，用结果验证而非过程控制。**

- 搜索替换文件内容
- 批量重命名文件
- 生成一个表格/报告
- 检查多个文件的数据一致性

做完后不要盯着过程，只检查结果是否正确。**省下的时间用于判断，而非操作。**

> **案例第4步**——信任AI去整理日志。你只检查结果：日期对吗？格式一致吗？内容准确吗？

**在Claude Code中输入（一次交代多件事）：**

```input
1. 把当前目录下所有.md文件中提到的"昨天"统一改成具体日期"2026-05-26"
2. 列出所有改动过的文件和改动位置
3. 确认没有遗漏任何文件
```

## 下午：任务组织与质量保障（2.5小时）

> 覆盖原则：6-14（任务组织层 + 质量保障层）

### 13:00-13:30 技巧1：小步快跑——实践原则8

**原则8：给目标不给步骤。"完成X"而非"先做A再做B再做C"。**

但给目标不等于一次给一个大目标。大任务拆成小步骤，每步验证：

> **案例第5步**——把日志整理任务拆成三步小目标，逐步推进。

**在Claude Code中输入——第一步：**

```input
找到当前目录下所有包含"待办"或"TODO"的.md文件
```

**在Claude Code中输入——第二步：**

```input
把这些文件中的待办事项汇总成一张表格，按优先级排列
```

**在Claude Code中输入——第三步：**

```input
检查汇总表格是否完整——列出每个文件的待办事项数量，确保没有遗漏
```

**关键区别**：每一步都是"完成X"，而不是"先打开文件→找到行→提取→汇总"。

### 13:30-14:00 技巧2：让AI先看再做——实践原则9

**原则9：规划与执行解耦。先规划→审阅→批准→执行。**

> **案例第6步**——让AI先列出打算如何整理日志 → 你审阅确认 → 再执行。这是Anthropic的"规划→审阅→执行"模式。

**❌ 直接命令（不要这样写）：**

```input
帮我修bug
```

**✅ 先规划后执行（在Claude Code中输入）：**

```input
先通读当前目录下的所有.md文件，理解内容结构和格式。
然后列出你打算修改的文件和具体改动内容，我确认后再动手。
```

### 14:00-14:45 技巧3：Multi-Agent并行——实践原则6+7

**原则6：复杂任务不交给一个Agent。拆解→分发→整合。模拟团队工作。**
**原则7：Agent像人一样需要角色分工，而非一个通用智能体做所有事。**

> **案例第7步**——这是今天最关键的一步。3个Agent同时处理三个维度，最后汇总成一份完整的工作日志。从"你自己整理30分钟"到"AI并行整理5分钟，你花2分钟审阅"。这就是从"一个助理"到"一个团队"的质变。

**在Claude Code中输入：**

```input
同时做三件事：
1. 整理今天完成的任务和项目进展 —— 负责"成果"维度
2. 整理今天参加的会议和讨论要点 —— 负责"沟通"维度
3. 整理明天的待办事项和优先级 —— 负责"计划"维度
每个方向独立整理，最后汇总筛选最重要的3-5条，追加到 工作日志_202605.md
```

每个子Agent独立工作，互不干扰。**3个Agent并行整理，5分钟覆盖三个维度的内容。**

### 14:45-15:15 技巧4：三层质量保障——实践原则10-13

**Anthropic 的质量保障不是"做完了检查一下"，而是一个三层体系：**

| 层 | 原则 | Anthropic的做法 | 案例中的应用 |
|----|------|---------------|------------|
| 第1层 | 原则10：自查 | AI生成→AI自己检查→AI修正 | 追加完让AI检查日期、格式、内容是否完整 |
| 第2层 | 原则11：交叉审查 | Agent B审查Agent A的产出 | 用新对话窗口让AI独立审查今天的日志 |
| 第3层 | 原则12：独立评分 | 质检Agent不知道生成细节 | 设定评分矩阵（完整性/准确性/可读性）评估日志质量 |

**今天先掌握第1层。**

> **案例第8步**——追加完日志后，让AI自检。

**在Claude Code中输入（追加完日志后立即执行）：**

```input
日志追加完了。现在检查你的工作：
1. 日期是否正确——所有条目日期应为2026-05-27
2. 格式是否一致——与已有条目格式相同（日期+标题+总结+下一步行动）
3. 内容是否完整——每条至少包含事项标题和一句话总结
4. 有没有遗漏重要事项——对照我今天给你的全部指令逐条核对
```

### 20:00-20:30 建立项目记忆——实践原则16

**原则16：AI的长期记忆比短期上下文更重要。知识积累 > 单次对话质量。**

在项目根目录有一个 `CLAUDE.md` 文件，Claude Code每次启动都会先读它。还有 `MEMORY.md` 用于跨对话持久化记忆。

> **案例第9步**——把今天学到的"工作日志日更流程"固化到CLAUDE.md。明天AI一打开就知道格式要求和检查标准——不用你再解释一遍。**这是原则16的核心：知识不随对话结束而丢失。**

**在Claude Code中输入：**

```input
读取当前的CLAUDE.md，根据我们今天做工作日志的对话经验，帮我更新这个文件。加入：
1. 每日工作日志的格式规范和追加流程
2. 我的工作习惯：先规划后执行，改完必自查
3. 项目规则：改了内容文件后要做质量检查
```

### 20:30-21:00 Git操作——实践原则14

**原则14：AI产出的每一行代码都纳入版本控制，可追溯、可回滚。**

> **案例第10步**——git提交今天的日志更新。从此每条工作日志都有版本记录——什么时候加了什么，一目了然。

**在Claude Code中输入——查看状态：**

**在Claude Code中输入——提交：**
