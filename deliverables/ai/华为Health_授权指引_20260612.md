# 华为Health Kit OAuth 授权指引

> 脚本位置：`scripts/huawei_health_cloud.py`
> 脚本状态：✅ v1.0 已完成 | ⚠ OAuth授权未完成

## 为什么需要OAuth

华为Health Kit REST API要求用户通过华为账号授权后才能读取健康数据。授权是一次性操作（后续token自动刷新），但需要**用手机扫码/登录**。

## 授权步骤

### 前提条件

- 一台**华为手机**（登录你的华为账号）
- 电脑浏览器可访问 localhost:8080
- 手机已安装**华为运动健康**App且已同步手表数据

### 执行授权

```bash
cd /Users/cyingfang/claude
python3 scripts/huawei_health_cloud.py --auth
```

执行后脚本会自动：
1. 启动本地回调服务器 (port 8080)  
2. 打开浏览器跳转到华为登录页面
3. 要求你用**手机上的华为账号**扫码登录

### 手机端操作

| 步骤 | 操作 |
|:---:|:-----|
| 1 | 浏览器打开后，**用手机扫码**或直接登录华为账号 |
| 2 | 确认授权范围：步数/心率/睡眠/血氧/体重/活动 |
| 3 | 授权成功后，浏览器显示"OK"页面，回调完成 |

### 验证授权是否成功

```bash
python3 scripts/huawei_health_cloud.py --fetch
```

如果返回健康数据 → ✅ 授权成功
如果报错 → 检查 `~/.huawei_health_token.json` 是否存在

### 授权后可用功能

```bash
# 读取今日数据
python3 scripts/huawei_health_cloud.py --fetch

# 读取最近7天数据
python3 scripts/huawei_health_cloud.py --fetch-all

# 如果需要重新授权
rm ~/.huawei_health_token.json
python3 scripts/huawei_health_cloud.py --auth
```

### 预期数据

成功授权后，每日自动读取以下数据并写入 `deliverables/health/路线图/华为手表数据.md`：

| 数据 | 说明 |
|:----|:----|
| 步数 | 每日总步数 |
| 心率 | 平均/最低/最高心率（bpm） |
| 睡眠 | 总睡眠时长（h） |
| 血氧 | 平均血氧饱和度（%） |

## 如果不做OAuth授权

健康数据的自动追踪只能依赖：
- BLE直连脚本（`huawei_health_ble.py`）— 需要手表在蓝牙范围内
- 手动记录（晨起体重/腰围/血压）

**建议本周完成授权**，让健康数据的基线采集变成自动化流程。
