# 一天学会Claude Code

> 适用对象：已安装Claude Code，想用一天时间从零到能独立用它干活
> 总耗时：约6小时（上午2h + 下午2.5h + 晚上1.5h）

---

## 零、 开始前：一句话理解Claude Code

**Claude Code = 终端里的AI程序员。** 你说话，它读写文件、执行命令、完成任务。

打开终端，输入 `claude`，回车。看到对话界面，就可以开始了。

### 今天你会经历什么

| 今天之前 | 今天之后 |
|---------|---------|
| 修改文件 → 打开编辑器 → 找到位置 → 手动改 → 保存 | 说一句话，改完 |
| 搜索内容 → 逐个文件打开看 | 说一句话，结果列出 |
| 跑命令 → 复制粘贴 → 看报错 → 搜索怎么修 | 说一句话，自动运行+修复 |
| 提交git → 想commit message → git add → git commit | 说一句话，提交完成 |
| 一次只做一件事 | 同时交代多件事，AI并行完成 |

**你今天不是学一个工具，是换一种工作方式。**

## 上午：会说话就会用（2小时）

### 9:00-9:15 第一个对话

```
帮我看看这个项目的目录结构，用一段话告诉我每个文件夹是干什么的
```

Claude Code会自动运行 `ls`、`find` 来了解项目，然后回答你。你不需要知道它用了什么命令。

### 9:15-9:45 三个核心指令

**指令1：帮我读**

```
读取 deliverables/ai/ 目录下所有文件，告诉我每个文件的核心内容，一句话概括
```

**指令2：帮我找**

```
在整个项目中搜索所有包含"昆仑万维"的文件，列出文件名和行号
```

**指令3：帮我写**

```
在deliverables/ai/目录下创建一个新文件 test.md，内容是：
[描述你要的内容]
```

### 9:45-10:15 让AI理解你——说清楚四件事

每次给Claude Code布置任务，说清四个信息：

| 信息 | 例子 |
|------|------|
| 文件在哪 | "在deliverables/investment/持仓分析.md中" |
| 改什么 | "把所有'减持'改成'增持'" |
| 为什么 | "因为最新季报显示机构在加仓" |
| 什么格式 | "保持原有表格格式不变" |

```
❌ "帮我改一下那个投资文件"
✅ "在deliverables/investment/持仓分析.md中，把昆仑万维的持有数量
   从5000改成6000，因为今天追加了1000股。保持表格格式不变。"
```

### 10:15-10:45 第一项实战任务

找一个你本来今天要手动做的事，让Claude Code来做：

- 搜索替换文件内容
- 批量重命名文件
- 生成一个表格/报告
- 检查多个文件的数据一致性

### 10:45-11:00 休息 + 回顾

翻开手机备忘录，写下：**我今天学会的三件事**
1. 
2. 
3.

## 下午：让AI干活而不是你干活（2.5小时）

### 13:00-13:30 技巧1：小步快跑

大任务拆成小步骤，每步验证：

```
第一步："找到deliverables/下所有包含2025的.md文件"
第二步："把这些文件中的2025年3月改成2026年5月"
第三步："检查改动是否正确——列出每个改动过的文件和改动位置"
```

### 13:30-14:00 技巧2：让AI先看再做

```
❌ "帮我修bug"
✅ "先通读deliverables/health/下的所有文件，理解内容结构。
   然后找出所有提到'女性'的地方，改成'男性'。
   改之前先列出你要改的文件和行号，确认后再动手。"
```

### 14:00-14:45 技巧3：Multi-Agent并行——你的AI团队

Claude Code的Multi-Agent能力（2026年核心功能）可以同时启动多个子Agent：

```
同时做三件事：
1. 检查deliverables/下所有.md文件有没有日期错误
2. 统计每个目标目录下有多少个文件，按.md和.pdf分类
3. 检查scripts/下所有.py文件有没有语法错误
```

每个子Agent独立工作，互不干扰。**3个Agent并行5分钟完成的工作，你一个个做要30分钟。** 这就是从"一个助理"到"一个团队"的质变。

### 14:45-15:15 技巧4：让AI自己检查自己

每次改完文件，追加一句：

```
改动完了。现在检查你的改动：
1. 是否正确——有没有改错的地方
2. 是否完整——有没有遗漏的文件
3. 有没有引入新问题——比如破坏了表格格式
```

### 15:15-15:30 休息

## 晚上：把Claude Code变成你的系统（1.5小时）

### 20:00-20:30 建立项目记忆

在项目根目录有一个 `CLAUDE.md` 文件，Claude Code每次启动都会先读它。

让Claude Code帮你更新它：

```
读取当前的CLAUDE.md，然后根据我们今天所有的对话，
帮我更新这个文件。加入：
1. 我的身份和角色
2. 我的工作习惯和偏好
3. 项目的重要规则（比如"改了.md文件后要重建PDF"）
```

### 20:30-21:00 Git操作——把Claude Code当版本管家

```
"给我看看当前git状态，用一句话总结今天做了什么改动"
"把今天所有新增和修改的文件，用一条合适的commit message提交"
```

### 21:00-21:30 明天开始的新工作方式

**每天早上（30秒）**：

```
总结一下我上次离开后项目的状态变化。然后列出我今天最应该做的3件事。
```

**工作中**：有任何想法直接对Claude Code说，不要自己动手找文件、改文件、跑命令。

**每天收尾（1分钟）**：

```
回顾我今天的所有改动，生成一份今日工作总结（200字以内），
然后帮我commit所有改动。
```

**进阶——定时任务**（第3天开始用）：

```
/loop 1h 检查所有.md文件是否正常
/loop 5d 并行推进目标，直到目标达成
```

**你学会了什么**：你不是学了一个工具，是换了一种工作方式。从"操作电脑"到"指挥AI操作电脑"。从"自己做"到"检查AI做得好不好"。

## 速查：5个你一定要记住的斜杠命令

| 命令 | 作用 |
|------|------|
| `/clear` | 话题换了，清空上下文重新开始 |
| `/compact` | 对话太长了，压缩上下文释放token |
| `/memory` | 查看AI记住了你的哪些信息 |
| `/tasks` | 查看后台运行的任务 |
| `/doctor` | 碰到奇怪问题时检查状态 |

---

## 速查：最常见的4个错误

| 你遇到的 | 怎么办 |
|---------|--------|
| AI改错了文件 | "撤销你刚才对XX文件的改动" |
| AI不理解你 | "用你自己的话复述一遍我让你做什么" |
| AI做了一半停了 | 只发一个字："继续" |
| 不知道Claude Code能不能做某件事 | 直接问它："你能帮我做XX吗？" |

---

## 今天结束后的检验标准

你能不查任何资料，做到以下5件事，就说明你学会了：

1. 让Claude Code搜索项目中所有包含某个关键词的文件
2. 让它修改一个文件的特定内容
3. 让它同时做两件不相关的事
4. 让它检查自己的改动是否正确
5. 让它提交一次git commit

---

> **最后的话**：Claude Code不要求你会编程。它要求你会说清楚——你要什么、在哪、什么结果。练一天，你会发现以前需要找程序员帮忙的事，自己就能做了。
>
> 打开终端 → `claude` → 开始说话。就这么简单。
