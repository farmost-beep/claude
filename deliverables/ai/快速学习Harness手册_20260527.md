# 快速学习Harness手册

> 适用对象：非技术背景，需要理解Harness是什么、为什么重要、用在哪儿
> 阅读时间：15分钟 | 配套：《快速学习Claude Code手册》《快速掌握AI手册》

## 零、 一句话定位

**Harness = AI驱动的软件交付自动化平台。代码写完之后的一切——测试、部署、安全、成本——它自动管。**

不用记这个定义。记住一句话就行：**软件开发领域的"自动驾驶"。**

---

## 一、 它解决什么问题（场景先行）

### 场景1：银行APP要更新一个功能

| 步骤 | 没有Harness | 有Harness |
|------|-----------|----------|
| 代码写好了 | 开发在本地电脑 | 同左 |
| 测试 | 测试团队手工跑3天，测了1000个用例，其实只有50个和这次改动有关 | AI自动识别那50个，**10分钟跑完** |
| 部署到测试环境 | 运维手动操作半天，可能搞错环境配置 | **一键自动部署** |
| 部署到生产环境 | 凌晨加班，一群人盯着屏幕，错了回滚要1小时 | 自动金丝雀发布：先给5%用户，没问题→50%→100%。出问题**秒级回滚** |
| 安全检查 | 安全团队最后发现漏洞，已经上线了 | **代码提交时自动扫**，有问题立刻拦下 |
| 云费用 | 下个月账单来了，不知道哪笔是谁花的 | **实时显示**每个团队用了多少钱 |

### 场景2：一个科技公司有50个微服务

- 50个服务要构建、测试、部署
- Harness统一管理，平台团队定义"标准模板"，每个服务团队在模板上填参数即可
- 花旗银行：用Harness后，部署时间从数天 → **7分钟**

## 二、 核心能力（只需记住6个英文缩写）

| 缩写 | 全称 | 做什么 | 一句话 |
|------|------|--------|--------|
| **CI** | Continuous Integration | 自动构建+测试 | "代码一提交，自动跑测试" |

| **FF** | Feature Flags | 功能开关 | "新功能先给1%用户试，没问题全开" |
| **STO** | Security Testing | 安全扫描 | "代码里有漏洞？提交时就告诉你" |
| **CCM** | Cloud Cost Mgmt | 云成本管理 | "谁多花了云的钱，一目了然" |
| **SRM** | Service Reliability | 事故响应 | "出问题了AI先分析，不用人半夜爬起来" |

## 三、 为什么是Harness而不是Jenkins

Jenkins是上一代的CI/CD工具（诞生于2011年），Harness是新一代的。

| 维度 | Jenkins | Harness |
|------|---------|---------|
| AI | 没有 | **AI原生**，自动识别哪些测试该跑、自动回滚 |
| 维护 | 需要专人维护插件和脚本 | 大部分开箱即用 |
| 部署策略 | 手写脚本 | 内置蓝绿/金丝雀/滚动6种策略 |
| 云成本可视 | 无 | **内置**，客户平均降低60%云成本 |
| 适合谁 | 有专职DevOps团队 | 开发团队自服务+平台团队治理 |

> 对陈颖芳的意义：如果你在评估一家科技公司的技术能力，看它用Jenkins还是Harness，是一个判断其工程成熟度的信号。

## 六、 30分钟就能理解的核心概念

### 6.1 Pipeline（流水线）= 一个自动化流程

```
代码提交 → [CI: 构建 → 测试 → 安全扫描] → [CD: 部署到测试 → 部署到生产]
```

就像银行审批流程：客户提交 → 客户经理初审 → 风控复核 → 分行审批 → 放款。每一步有成功/失败判断。

### 6.2 Delegate（代理）= 执行任务的工人

Harness的控制面在云端（SaaS），但执行任务的"工人"（Delegate）部署在你自己的服务器上。Delegate只主动连接控制面，不需要开放防火墙入站端口——**金融行业最关心的安全模型**。

### 6.3 Template（模板）= 可复用的流程标准

平台团队定义"标准上线模板"：必须经过哪几步、每步什么检查。应用团队在这个模板上填写自己服务的参数，不能绕过安全检查——**治理和控制**。

## 七、 如果你需要评估一家公司用Harness的水平

| 问题 | 你问这个能得到什么 |
|------|-----------------|
| "你们用Harness的哪些模块？" | 判断深度——只用了CI还是全模块 |
| "部署频率多少？从提交到上线的平均时间？" | DORA指标，行业benchmark：精英团队每天多次部署，提交到上线<1小时 |
| "有没有自动回滚？回滚需要多久？" | Harness的核心优势，好的团队能做到秒级回滚 |
| "云成本管理用的是什么？" | 如果回答"没有专门的工具"，说明成本治理还有空间 |
| "安全扫描是上线前还是上线后？" | 好的是代码提交时就扫（Shift Left），差的是上线后才发现 |

## 八、 免费开始（如果你想自己试试）

```bash
# 一台有Docker的电脑，一条命令
docker run -d -p 3000:3000 \
  -v /var/run/docker.sock:/var/run/docker.sock \
  --name harness harness/harness
```

打开浏览器 `http://localhost:3000`，账号 `admin`，密码 `changeit`。5分钟就能看到界面。

免费计划包含每月2000次构建额度，足够个人学习。
