# AI指令层规范

> 定义"怎么给AI下指令"。不是话术技巧，是任务组织原则。指令质量决定产出质量的上限。

## 五条核心原则

| # | 原则 | 含义 | 反例 |
|:---:|:---|:---|:---|
| 1 | **给目标不给步骤** | 说"完成X"，不说"先A再B再C" | "帮我找3篇研报，然后读第1篇，然后提取数据……" → 错误：你在替AI规划 |
| 2 | **复杂任务拆解→分发→整合** | 一个复杂任务 = N个独立子任务，分给N个Agent并行执行 | 把投资分析/事业规划/健康扫描揉成一团交给一个Agent |
| 3 | **Agent角色分工** | 每个Agent有明确角色定义和产出格式，不混用 | 让同一个Agent既做研究又做写作又做检查 |
| 4 | **规划与执行解耦** | 先出计划→审阅批准→再执行。规划阶段不改代码，执行阶段不改计划 | "直接做吧"——跳过了审阅环节 |
| 5 | **给完整上下文** | 每次指令附带相关规范文件、数据文件、约束条件 | 让AI"猜"你的偏好或数据在哪里 |

---

## 原则详解

### 原则1：给目标不给步骤

**为什么**：AI比你更擅长找执行路径。你定义"什么是完成"，AI定义"怎么完成"。你替AI规划步骤=放弃AI的核心优势。

**正确做法**：
```
完成 [目标描述]。
约束条件：[时间/资源/格式限制]。
完成标准：[可验证的验收条件]。
产出位置：[文件路径]。
```

**错误做法**：
```
先打开A文件，提取第3-5段，计算平均值，然后写入B文件……
```

**判断**：
- 好：指令中有"完成X"和"怎么才算完成"，没有逐步骤操作
- 需关注：指令中混入了操作步骤（"先……再……然后……"）
- 有问题：AI做了某步骤后等你确认才继续——你变成了AI的执行监督员

---

### 原则2：复杂任务拆解→分发→整合

**为什么**：一个Agent的注意力是有限的。两个独立Agent并行产出 > 一个Agent串行产出。拆解质量 = 并行效率的上限。

**拆解规则**：
1. 子任务间无相互依赖 → 可并行
2. 子任务有明确产出物 → 可独立验收
3. 每个子任务 ≤ 一个Agent可独立完成的范围

**标准模式**：
```
任务：[复杂任务描述]
拆解为3个子任务：
- Agent A（角色）：[子任务1] → 产出 [文件/格式]
- Agent B（角色）：[子任务2] → 产出 [文件/格式]
- Agent C（角色）：[子任务3] → 产出 [文件/格式]
三个Agent并行启动。全部完成后由我整合。
```

**判断**：
- 好：每周至少1次多Agent并行任务
- 需关注：连续2周全是单Agent串行——编排力闲置
- 有问题：把"能并行"的任务串行做了——浪费了AI能力

---

### 原则3：Agent角色分工

**为什么**：Agent像人一样需要角色定义。模糊角色=模糊产出。一个"万能Agent"产出的东西你需要自己再筛选。

**角色定义模板**：
```
你是 [角色名称]。
你的任务：[一句话核心职责]。
你的产出格式：[明确的输出结构]。
你的约束：[什么不能做、什么数据不能用]。
```

**常用角色类型**：

| 角色类型 | 示例角色 | 适用场景 |
|:---|:---|:---|
| 研究者 | Explore Agent | 搜索文件、收集信息、发现问题 |
| 执行者 | Write/Edit | 写文件、改代码、生成内容 |
| 审查者 | Plan Agent / 独立质检 | 检查产出、发现问题、评分 |
| 整合者 | 汇总Agent | 合并多个Agent产出为统一报告 |

**判断**：
- 好：每个Agent启动时都有角色定义+产出格式要求
- 需关注：用"你帮我看看这个"启动Agent——角色模糊
- 有问题：同一个Agent同时扮演"研究者+执行者+审查者"——既当运动员又当裁判

---

### 原则4：规划与执行解耦

**为什么**：规划和执行是两种不同的认知活动。规划阶段需要"广度"（考虑多种方案），执行阶段需要"深度"（做好当前方案）。混在一起=计划边做边改=质量不可控。

**标准流程**：
```
第一步：出规划
  - 输出：[计划文档]，包含步骤、时间估计、风险点、备选方案
  - 此时不改任何代码/文件

第二步：审阅
  - 你审阅计划，修改、确认或否决
  - 确认后进入执行

第三步：执行
  - 严格按照批准的计划执行
  - 执行中遇到计划外问题 → 暂停，更新计划，重新审阅
```

**判断**：
- 好：重要任务（影响≥2个文件或涉及投资决策）都先出计划再执行
- 需关注：规划被跳过，"直接做吧"
- 有问题：边执行边改计划，最终产出和原计划完全不一样

---

### 原则5：给完整上下文

**为什么**：AI每次对话从零开始。你给的上下文 = AI的知识边界。上下文缺失 → AI用"通用知识"填补 → 产出和你无关。

**每次任务必须提供**：
1. 本次任务的目标和约束
2. 相关的规范文件（从`记忆规范/`目录加载）
3. 数据来源（文件路径、数据文件）
4. 上次相关对话的关键结论（如适用）

**判断**：
- 好：每次协作都引用至少1份规范文件
- 需关注：连续3次协作没提供规范文件 → 在退化到"通用AI"
- 有问题：AI反复问"文件在哪里""数据是什么格式"——你上下文没给够

## 指令质量自检清单（每次下指令前30秒）

- [ ] 我说的是"目标"还是"步骤"？（如果是步骤 → 改成目标）
- [ ] 这个任务可以拆成2-3个并行子任务吗？（如果可以 → 拆开）
- [ ] 每个Agent有明确的角色和产出格式吗？
- [ ] 重要任务先出计划了吗？（≥2文件影响 → 必须先出计划）
- [ ] 相关规范文件加载了吗？（至少关联1份规范）
