# 知识库建设规范

> v2.0 重新定位：知识库不是第六个目标，是其他五个目标的第二大脑。不衡量"存了多少"，衡量"用了几次"。

## 定位：第二大脑，不是数字仓库

知识库是你认知能力的延伸——像第二大脑一样，承担两个功能：

1. **自动调用**（Layer 1）：AI在协作时自动读取相关知识，你不需要记得"我写过什么"
2. **被动探索**（Layer 2）：你主动浏览时，3秒判断是否深入，帮你在需要时快速找到

**衡量知识库好坏的唯一标准：它有没有让你做出更好的决策。** 存了1000张卡片但从没被调用过=数字垃圾。只有20张卡片但每张都支撑过实际决策=好的知识库。

---

## 双层架构

```
Layer 1: AI记忆层（规范文件）
  ├── 自动调用，不需要人分配注意力
  ├── 每次AI协作自动加载对应规范
  ├── 内容：规则、标准、边界、检查清单
  └── 质量指标：规范被调用的频率 × 调用后产出的一致性

Layer 2: Obsidian探索层（MOC + 知识卡片）
  ├── 人主动浏览，3秒判断是否深入
  ├── 内容：概念卡、框架卡、案例卡、跨域连接
  └── 质量指标：卡片被实际决策引用过的比例
```

**两层的关系**：Layer 2是原料库→经过验证有效的知识→固化为Layer 1的规则。Layer 1是生产线→自动化执行→暴露出规则盲区→回到Layer 2探索补充。

### 什么进Layer 1（AI记忆层）

- 被3次以上实际决策验证过的规则/标准
- "如果...就..."型判断逻辑（AI可以直接执行的）
- 模板和检查清单（AI可以套用的）
- 风险信号（AI可以自动扫描的）

### 什么进Layer 2（Obsidian探索层）

- 新概念/新框架（尚未验证）
- 具体案例（用于参考，不是规则）
- 跨域连接（启发思考，不是执行）
- 还在迭代的想法

### 什么不进知识库

- 能Google到的常识
- 不用卡片也能记住的浅概念
- 和自己实际场景无关的理论
- 和已有知识重复超过70%的内容

## Layer 1 标准：规范文件

**格式要求**（每份规范必须包含5要素）：
1. 核心指标/维度（5-8个）
2. 判断标准（好/需要关注/有问题，各含具体量化阈值）
3. 风险信号（至少5个"如果...就..."规则）
4. 输出/回顾模板
5. 必须回答的问题（月度/季度）

**质量验证**（每份规范每月抽检一次）：
- 规范中的"如果...就..."最近一个月触发过几次？
- 有没有你实际遇到但规范没覆盖的情况？（覆盖盲区）
- 有没有规范中的判断标准你已经不再认同了？（记忆腐烂）

## Layer 2 标准：知识卡片

**每张卡片必须包含4要素**：
1. 一句话定义（20字以内）
2. 在我的场景中的应用（不是教科书定义，是我的用法）
3. 适用边界（什么时候不适用）
4. 关联卡片（至少链接1-2个相关概念）

**不做的卡片**：
- 概念太浅的（不需要卡片也能记住）
- 只有定义没有"我的用法"的
- 和已有知识重复超过70%的

**卡片升级规则**：一张卡片被3次以上实际决策引用→评估是否可以将其中内容固化为Layer 1的规则。

---

## 判断标准

**好**：
- Layer 1：规范文件每月至少1次实战驱动的更新（不是"我觉得该更新了"而是"这次用的时候发现这里不对"）
- Layer 2：新学到的概念/框架，能在当周内用自己的话写成卡片（含"我的应用"）
- 连接：跨域卡片之间有双向链接，MOC能导航到具体卡片
- 调用：AI协作中规范被自动调用，不需要人每次额外提供上下文

**需要关注**：
- 某个规范超过2个月未更新 → 要么完美（不可能），要么腐烂
- 连续2周未新增任何卡片 → 输入管道断裂（没在学习）或输出管道断裂（学了没转化）
- 出现了3次以上"这个我以前整理过但找不到了" → 索引失效
- Layer 2的卡片从没有被引用过 → 可能是数字垃圾

**有问题**：
- 某个规范中的判断标准你已经不再认同了 → 记忆腐烂，需要立即修正
- 知识库变成了"存了就忘"的数字仓库——MOC 1个月未更新、卡片搁置>3个月
- AI协作时你发现自己反复解释同一个概念 → 这个概念应该进Layer 1但还没进
- Obsidian Vault与deliverable出现矛盾信息 → 两边有一边是错的

---

## 风险信号

1. 你分享/讲解一个概念超过3次，但它还没进知识库 → 知识在流失（只在脑子里，没进系统）
2. 规范中的某个判断标准你跳过/忽略超过3次 → 标准不适用了但没更新
3. AI协作时需要额外解释上下文超过3分钟 → Layer 1索引没做好
4. 发现自己在搜索引擎中找"我记得有个..." → 知识库的检索通路断了
5. 有人问你一个问题，你不能在30秒内从知识库中找到相关的卡片/规范 → 知识库不可用

## 与五个目标的关系

知识库不是独立目标，是五个目标的基础设施：

| 目标 | Layer 1（规范文件） | Layer 2（知识卡片） |
|:---|:---|:---|
| 投资成功 | 投资行为规范、Agent组配置 | 公司分析卡、行业框架卡、估值方法卡 |
| 事业成功 | 事业决策规范 | 银行科技金融卡、政策趋势卡、写作框架卡 |
| 家庭支持 | 家庭管理规范 | 育儿方法卡、关系沟通卡 |
| 百岁健康 | 健康追踪规范 | 运动方法卡、营养知识卡、体检指标卡 |
| AI能力 | AI能力发展规范、雷达周报模板 | AI工具卡、方法论卡、产品评价卡 |

---

## 我必须回答的问题（每月）

1. Layer 1：哪份规范这个月被实战检验后更新了？哪份可能腐烂了？
2. Layer 2：这个月新增了几张"有我的用法"的卡片？有没有旧的卡片被升级到Layer 1？
3. 调用：这个月AI协作中规范被自动调用的频率如何？有没有需要额外解释的情况？
4. 腐烂：有没有发现自己不再认同的判断标准？有没有"存了但从没看过"的知识？
5. 盲区：这个月有没有遇到"我应该有相关知识但知识库里没有"的情况？最缺什么？
