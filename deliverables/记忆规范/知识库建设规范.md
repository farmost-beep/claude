# 知识库建设规范

## 核心维度

| 维度 | 目标 | 当前状态 | 下一里程碑 |
|:---|:---|:---:|:---|
| 知识卡片数量 | ≥150张 | ~50张 | 100张 |
| 跨域连接 | ≥50条双向链接 | ~20条 | 30条 |
| 记忆文件质量 | 六大目标各一份成熟规范 | 建设中 | 全部v1.0完成 |
| Obsidian Vault覆盖率 | 六大目标MOC全部有摘要 | 基础覆盖 | 全部目标有完整导航 |
| AI可调用性 | AI可自动调用所有知识卡片 | 部分可调用 | 记忆文件全部路径化 |

## 知识卡片标准

每张卡片必须包含：
1. **一句话定义**（20字以内）
2. **在我的场景中的应用**（不是教科书定义，是我的用法）
3. **适用边界**（什么时候不适用）
4. **关联卡片**（至少链接1-2个相关概念）

不做的卡片：
- 概念太浅的（不需要卡片也能记住的）
- 和已有知识重复超过70%的
- 只有定义没有"我的用法"的

## 记忆文件标准

每个目标的记忆文件必须包含：
1. 核心指标/维度（5-8个）
2. 判断标准（好/需要关注/有问题）
3. 风险信号（至少5个"如果...就..."规则）
4. 输出/回顾模板
5. 必须回答的问题

## 判断标准

**好**：
- 每周新增至少2张知识卡片
- 记忆文件每月至少1次更新
- Obsidian Vault 与 deliverable 保持同步
- AI在协作中能自动调用相关知识卡片

**需要关注**：
- 连续2周无新卡片
- 某个目标的记忆文件超过2个月未更新
- 阅读了但没有提取卡片（知识在流失）
- Obsidian Vault 中出现了"幽灵链接"（链接到不存在或过时的文件）

**有问题**：
- 知识卡片数量停滞>1个月
- 某个目标的记忆文件缺失或严重过时
- 知识库变成了"数字垃圾场"——存了很多但不回顾不调用
- Obsidian与deliverable严重不同步（两边信息矛盾）

## 风险信号

1. 连续3本书/文章读完但没有产出卡片 → 知识管理流程断裂
2. 记忆文件中出现了自己不再认同的判断标准 → 记忆腐烂，需要立即更新
3. AI协作时你发现自己在重复解释同一个概念 → 这个概念应该做成卡片但还没做
4. Obsidian Vault 中某目标的MOC超过1个月未更新 → 导航层滞后

## 我必须回答的问题（每月）

1. 这个月新增了多少张卡片？质量如何（有没有"我的用法"和"边界"）？
2. 哪些目标的记忆文件需要更新？（对照最近的实际经验）
3. AI在协作中调用知识卡片的效果如何？有没有调用错误的？
4. 有没有应该提取但一直拖延的知识？（标记出来，下个月优先）
5. Obsidian Vault 和 deliverables 是否一致？（抽检3个文件）
