# AI质量层规范

> 定义"怎么确保AI产出靠谱"。不用每件事都亲自检查——用系统代替人肉检查。

## 五层质量网

```
第一层：自查（生成后立即自检）
   ↓ 通过
第二层：交叉审查（另一个Agent独立审查）
   ↓ 通过
第三层：独立评分（质检Agent按评分矩阵打分）
   ↓ 通过
第四层：自动修复（发现问题→自动修→再验证）
   ↓ 通过
第五层：版本追溯（Git记录一切，可回滚）
```

---

## 第一层：生成后必自查

**规则**：每次产出后，AI必须立即检查自己的产出，不等你提醒。

**自查内容**：

| 检查项 | 针对的产出类型 | 检查方法 |
|:---|:---|:---|
| 文件是否存在 | Write/Edit操作 | 确认文件路径正确、文件可被系统识别 |
| 内容完整性 | 任何写入 | 关键信息逐项核对——不是"看一遍"，是对照原始要求逐条勾 |
| 格式正确性 | Markdown/JSON/代码 | 检查语法、闭合标签、引号匹配 |
| 数据一致性 | 含数据的产出 | 数字前后一致、单位正确、总计=各项之和 |
| 链接有效性 | 含链接的产出 | wikilink指向的文件存在、外部URL格式正确 |

**判断**：
- 好：AI主动自查，输出中包含"已验证：X项通过"的确认
- 需关注：AI产出后你发现明显错误——自查环节失效
- 有问题：连续3次产出有同类错误——自查模式未内化

---

## 第二层：交叉审查

**规则**：关键产出必须经过第二个独立Agent审查。同一个Agent不能审查自己的产出。

**触发条件**（满足任一条即触发）：
- 产出影响投资决策（持仓分析、风险判断、交易建议）
- 产出将被对外发表（公众号文章、民建建言、财新投稿）
- 产出包含定量数据（计算、统计、预测）
- 产出修改了≥3个文件

**审查流程**：
```
Step 1: 生成Agent产出 → 保存到文件
Step 2: 启动独立审查Agent
        - 角色：独立审查员
        - 给审查Agent生成Agent的原始要求和产出文件
        - 不给审查Agent生成Agent的思考过程（防止偏见）
Step 3: 审查Agent输出审查报告：
        - 事实性错误
        - 逻辑矛盾
        - 数据不一致
        - 遗漏的关键点
        - 总体评价：通过 / 修改后通过 / 重做
Step 4: 如为"修改后通过"，生成Agent修改后重新提交审查
```

**判断**：
- 好：每周至少1次交叉审查
- 需关注：连续2周关键产出无交叉审查
- 有问题：对外发表的内容未经交叉审查即发布

---

## 第三层：独立评分

**规则**：重要决策前，让AI用你定义的评分矩阵评估选项。评分Agent不知道你的偏好。

**触发条件**：
- 投资决策（买入/卖出/加减仓）
- 方案选择（A方案 vs B方案）
- 优先级排序（多项任务的资源分配）

**评分矩阵模板**：
```
你是独立评估员。按以下矩阵对[对象]评分（1-10分）：

| 维度 | 权重 | 评分标准 |
|:---|:---:|:---|
| [维度1] | [%] | [10分=什么表现 / 1分=什么表现] |
| [维度2] | [%] | [评分标准] |
| ... | | |

约束：
- 每个分数必须附理由（≤50字）
- 不能使用"可能""大概"等模糊词——每个判断需要具体数据或证据
- 总分计算后标注置信度（高/中/低）
```

**判断**：
- 好：投资相关决策都经过评分矩阵评估
- 需关注：评分矩阵用了但每次都得出你想要的结论——矩阵可能被你操控了
- 有问题：评分结果与你直觉强烈冲突但你忽略了——要么矩阵有问题，要么直觉有问题

## 第四层：失败自动修复

**规则**：AI产出有问题→AI自己修→AI验证修好了。人只在"修不好"时介入。

**修复流程**：
```
发现问题（自查/交叉审查/独立评分发现）
   → AI诊断根因（不是修表面，是找到为什么错）
   → AI提出修复方案（一句话）
   → AI执行修复
   → AI验证修复效果（跑测试/检查文件/重新评分）
   → 如修复失败：记录失败原因，人介入
   → 如修复成功：记录修复历史，继续
```

**人不介入的条件**：
- 修复范围 ≤ 原任务范围
- 不涉及资金操作、对外发布、不可逆操作
- 修复尝试 ≤ 2次（2次后强制人工介入）

**判断**：
- 好：小错误AI自动修，你只看到"已修复"的结果
- 需关注：同样的错误出现≥3次——根因诊断失效
- 有问题：AI"修复"引入了新错误而未被发现——验证环节缺失

---

## 第五层：版本追溯

**规则**：AI产出的每一次写入都有Git记录。可追溯、可对比、可回滚。

**追溯要求**：
- 每次AI产出写入文件后，纳入Git版本控制
- Commit信息标明：产出Agent、任务描述、变更摘要
- 关键决策节点打tag，方便回溯

**判断**：
- 好：任何时候都能回答"这个文件什么时候、被谁、因为什么原因改的"
- 需关注：手动绕过Git直接改文件
- 有问题：Git历史混乱，无法追溯某个改动的原因

## 质量层风险信号

| 风险信号 | 严重程度 | 含义 |
|:---|:---:|:---|
| 连续3次AI产出被你发现明显错误 | 🔴 | 自查+交叉审查两层均失效 |
| AI产出直接对外发布，无任何审查 | 🔴 | 质量层全部跳过 |
| 同一类错误重复出现≥3次 | 🟡 | 修复只修了表面，未诊断根因 |
| 开了交叉审查但审查Agent总是"通过" | 🟡 | 审查Agent的独立性存疑，可能有偏见 |
| Git历史超过2天无commit | 🟢 | 轻微——但说明AI产出量下降了 |

---

## 质量层应用分级

不是所有产出都需要跑满5层。按产出重要性分级：

| 产出等级 | 最少质量层 | 示例 |
|:---:|:---|:---|
| **S级**（对外发布/投资决策） | 5层全跑 | 公众号文章、持仓分析、交易指令 |
| **A级**（内部分析/方案设计） | 1+2+5 | 行业研究、备考计划、健康周报 |
| **B级**（日常记录/草稿） | 1+5 | 每日简报、AI简报、个人画像 |
| **C级**（临时探索/对话笔记） | 1 | 探索性搜索、随手记、方案草图 |

---

## 质量自检清单（每周）

- [ ] 本周有S级产出吗？→ 是否跑了5层质量网？
- [ ] 本周有A级产出吗？→ 是否跑了1+2+5？
- [ ] 本周发现了几个AI产出错误？→ 是自查发现的还是交叉审查发现的？
- [ ] 本周AI修复了几个自己的错误？→ 修复成功率？
- [ ] Git log中本周的commit信息是否可追溯？
