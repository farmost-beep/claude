# AI能力发展规范

> v2.0 重新定义五维度——从"功能堆叠"升级为"价值链覆盖"：输入（感知力）→ 转化（规范力）→ 执行（编排力）→ 检查（验证力）→ 突破（扩展力）。

## 核心能力维度

| 维度 | 定义 | 当前等级 | 目标等级 | 一句话 |
|:---|:---|:---:|:---:|:---|
| 规范力 | 把隐性知识写成AI可执行规则 | L1.5 | L2.5 | 你说得清，AI才做得对 |
| 编排力 | 把复杂任务拆成多Agent并行协作 | L1.0 | L2.0 | 一个人管多个AI，不是被AI带着走 |
| 验证力 | 系统性检查AI产出、发现异常 | L0.5 | L1.5 | AI错了你能发现，不等出事了才知道 |
| 扩展力 | 用代码/工具突破AI对话边界 | L0.5 | L1.5 | AI做不到的，你能写脚本让它做到 |
| 感知力 | 系统追踪AI演进、判断什么值得跟 | L0.5 | L1.5 | 知道风往哪吹，但不被风吹着跑 |

### 等级定义

| 等级 | 标准 |
|:---:|:---|
| L0 | 不知道这个概念 |
| L0.5 | 知道但未实践，或偶尔无系统 |
| L1.0 | 有系统地在做，但未形成可复用资产 |
| L1.5 | 有可复用资产（规范/Agent组/验证规则/脚本/雷达），经1-2次实战验证 |
| L2.0 | 资产经3次以上实战迭代，形成习惯，能指导他人 |
| L2.5 | 多场景覆盖，资产自进化（不依赖手动更新） |

## 判断标准

**好**：
- 规范力：每次AI协作自动调用对应规范，规范至少每月迭代1次
- 编排力：每周至少1次多Agent并行任务，Agent配置随实战持续优化
- 验证力：每次AI关键产出有交叉验证步骤，异常AI行为被记录和复盘
- 扩展力：能独立写50行以内的工具脚本，读得懂AI生成的代码逻辑
- 感知力：每周15分钟雷达更新完成，重要的新能力在发布2周内了解

**需要关注**：
- 规范力：连续2周用AI时未提供记忆文件 → 在退化到"通用AI"
- 编排力：连续2周无Agent并行使用 → 编排能力停滞
- 验证力：AI产出直接采用无检查 → 验证意识丧失
- 扩展力：连续2周未写任何代码（含AI辅助） → 扩展力闲置
- 感知力：连续2周未更新雷达 → 信息源断开

**有问题**：
- 规范力：规范超过2个月未更新 → 记忆腐烂
- 编排力：退回纯问答模式1个月以上 → 管理模式退化
- 验证力：AI产出导致实际损失 → 验证系统失效
- 扩展力：连续1个月未碰代码/脚本 → 能力萎缩
- 感知力：重要AI产品发布1个月后才知道 → 雷达失效

## 风险信号

1. 连续3次AI协作没有提供规范文件 → 规范力退化，在用"平均AI"而非"你的AI"
2. 在Chat界面的时间多于Agent/多Agent模式 → 编排力闲置
3. AI产出被直接采用，无交叉验证步骤 → 验证力缺失，管理模式未闭环
4. Claude Code质量变化（如思考深度下降）未被自己发现 → 验证力空白
5. 新的重要AI产品/模型发布1个月后才知道 → 雷达失效
6. 需要重复性操作时想不到"这个可以写脚本" → 扩展力未形成思维习惯

## 我必须回答的问题（每月）

1. 这个月规范力有没有进步？哪份规范被实战检验后更新了？
2. 这个月编排力有没有进步？Agent并行跑了几次？有什么改进？
3. 这个月验证力有没有进步？有没有发现并记录了AI的错误/异常？
4. 这个月扩展力有没有进步？写了什么脚本/代码？
5. 这个月感知力有没有进步？最重要的AI新发现是什么？对我有什么用？
6. 有没有在"为了用AI而用AI"？还是在真正解决实际问题？
