# 上下文加载清单

> 按手册原则2（给AI完整上下文）+ 上下文适度原则：每次任务加载1-3份最相关的规范文件
>
> 版本 v1.0 | 2026-05-30 | 当新增规范或任务类型变化时更新本清单

---

## 使用方法

1. 确定本次任务类型（见下表）
2. 加载"必须"列中的规范文件（1-3份）
3. "可选"列仅在特定条件下加载（如产出等级为S/A级时加载质量层规范）
4. 不确定时，宁可少加载（1份 > 0份），下一次根据AI表现调整

## 加载规则

```
每次AI协作开始时：
1. 识别任务类型（对照上表）
2. 非代码类任务 → 先加载 claude/CLAUDE.md + ~/.claude/CLAUDE.md（个人偏好+项目规则）
3. 根据任务类型 → 加载1-3份领域规范（必须列）
4. 判断产出等级 → S/A级加AI质量层规范
5. 不确定该加载什么 → 加载最相关的1份 > 不加载
```

## 反模式

| 反模式 | 后果 | 正确做法 |
|:---|:---|:---|
| 每次协作加载全部14个规范 | 上下文过载，AI注意力分散，产出质量下降 | 按任务类型选1-3份 |
| 什么都不加载 | AI用通用知识填补，产出和你无关 | 至少加载1份 |
| 连续3次同类型任务加载不同规范 | AI表现不稳定，每次从不同基线出发 | 同类型任务固定加载组合 |
| 用旧版规范（如投资分析v1.0而非v2.0） | 规范冲突，AI执行已废弃的规则 | 检查文件版本和日期 |
