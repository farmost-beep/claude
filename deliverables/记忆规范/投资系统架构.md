# 投资系统架构

> 投资成功是幸福人生的第一目标。这份文件是所有投资能力的总装图——行为规范、分析管道、组合管理、系统进化四组件如何协同工作。

---

## 定位：投资不是"做分析"，是"运行一个系统"

大多数投资者把时间花在"研究公司"上。职业投资者把时间花在"运行一个研究→决策→执行→进化的系统"上。

这个系统有四层：

```
决策质量层  →  行为纪律层  →  资本配置层
（分析管道）    （投资行为规范）  （组合管理）
    ↓               ↓               ↓
        系统进化层（Auto-PR / Agent E / 知识升级）
```

---

## 第一层：决策质量 — 分析管道

**回答的问题**：这家公司好不好？值多少钱？风险在哪？

### 1.1 5-Agent分析管道

```
原始数据（财报/研报/市场数据）
    ↓
Agent A：指标提取 → 100分加权评分 + 8季度趋势
Agent B：行业对标 → 与行业75分位数对比 + 竞争位势
Agent C：风险扫描 → 7条风险信号逐一检测 + 额外关注项
    ↓  并行处理
Agent D：一句话总结 → 200字以内投资结论
    ↓
Agent E：汇总与异常排序 → 交叉验证A/B/C/D数据一致性
    → 异常程度排序（🔴高/🟡中/🟢低）
    → Top N需要人判断的问题
    → 可跳过公司识别
```

**关键设计原则**：
- Agent A/B/C并行运行，互不污染——B不知道A的结论，C不知道B的判断
- Agent E的核心价值不是汇总，是**交叉验证**——找A/B/C之间的矛盾
- 每个Agent输出必须标注置信度（数据充足/数据不足/估计值）

### 1.2 数据质量阶梯

| 等级 | 数据来源 | 可信度 | 使用场景 |
|:---|:---|:---:|:---|
| **一级** | 年报/季报完整PDF原文 | 最高 | Agent A的输入基础 |
| **二级** | Wind/同花顺行业数据 | 高 | Agent B行业对标的第一选择 |
| **三级** | 券商研报（交叉验证3份以上） | 中 | Agent A的补充 + Agent B的参考 |
| **四级** | Agent自身训练知识（标注"[估计]"） | 低 | 仅在无客观数据时使用，Agent B已标注为改进项 |

**当前缺陷**（来自Agent E首次运行的自我验证）：
- Agent A大量标注"[数据不足]"，缺少完整财报原文 → 下次财报季将PDF作为输入
- Agent B的"行业估计值"依赖自身训练知识，未使用Wind等客观数据 → 改进中
- 数据质量直接影响分析质量——这是系统的阿基里斯之踵

### 1.3 决策流程

```
财报发布
    ↓
Agent A/B/C/D/E 管道运行（约30分钟，3-5家公司并行）
    ↓
Agent E 输出异常排序
    ↓
人只看两样：
  1. 🔴高异常公司 → 必看Agent C风险信号
  2. 🟡中异常公司 → 必看Agent E交叉验证发现的矛盾
    ↓
人做判断（Agent E已列出Top N需要人判断的问题）
    ↓
决策记录（写入投资日志：为什么买/卖/持有，<=200字）
```

**人做什么、AI做什么**：
- AI做：数据提取、趋势计算、风险信号检测、数据一致性检查、异常排序
- 人做：判断"数据不足"要不要去补、判断Agent E发现矛盾后信谁、最终买卖决策

## 第二层：行为纪律 — 投资行为规范

**回答的问题**：知道该怎么做的时候，我按规矩做了吗？

由 **《投资行为规范》v2.0** 承载，不在此重复。核心接口：

| 模块 | 对应规范章节 | 关键机制 |
|:---|:---|:---|
| 买入前检查 | 第二章 | 8项必完成动作 + 24小时冷却期 |
| 分析框架 | 第三章 | 9指标100分加权评分（Agent A已实现） |
| 仓位上限 | 第四章 | 核心15%/一般10%/观察2%/圈外0% |
| 卖出规则 | 第五章 | 4项强制卖出 + 4项禁止卖出 |
| 行为禁区 | 第六章 | 10条禁令 |
| 节奏回顾 | 第七章 | 日0min/周15min/月30min/季半天/年半天 |

**架构关系**：分析管道输出"这家公司打几分"，行为规范决定"打几分的时候能买多少、什么时候必须卖"。

## 第三层：资本配置 — 组合管理

**回答的问题**：投多少？现金留多少？回撤到哪该停？

### 3.1 组合仪表盘

| 指标 | 规则 | 当前状态 | 检查频率 |
|:---|:---|:---:|:---:|
| 单只股票仓位 | ≤15%（核心）/ ≤10%（一般） / ≤2%（观察） | 自动检查 | 每周 |
| 单行业仓位 | ≤40% | 自动检查 | 每周 |
| AI产业链总仓位 | ≤60% | 自动检查 | 每周 |
| 现金仓位 | ≥10% | 自动检查 | 每周 |
| 海外资产仓位 | ≤30% | 自动检查 | 每周 |
| 组合总回撤 | ≤15%（暂停买入）/ ≤25%（强制复盘） | 自动检查 | 每周 |

### 3.2 资本来源与分流

```
事业收入 ──→ 月储蓄（目标≥60%储蓄率）
                │
                ├──→ [还本专户]（还贷本金，不可投资）
                │
                └──→ [投资账户]
                      │
                      ├──→ 股票仓位（≤90%投资账户）
                      ├──→ 现金仓位（≥10%投资账户）
                      └──→ 教育金专户（独立，不纳入投资仓位）
                           赡养费预算（独立，不纳入投资仓位）
                           医疗应急金（独立，货币基金，不纳入投资仓位）
```

**关键原则**：教育金、赡养费、医疗应急金是"防火墙"——与投资账户物理隔离。不管投资账户涨跌，这三笔钱不能动。

### 3.3 组合追踪器

由 `auto_pr.py` 投资组合模块承载：
- 每日自动扫描持仓盈亏
- 触发强制复盘信号（单只跌幅>20%）
- 仓位超标预警
- 与投资行为规范§4.3联动

## 第四层：系统进化 — 投资飞轮

**回答的问题**：这个投资系统本身是不是在变聪明？

### 4.1 进化飞轮

```
分析管道运行 → Agent E发现矛盾 → 记录为AI异常日志
      ↓                                      ↓
  投资决策      ←    知识卡片升级    ←  auto_fix草案
      ↓
  决策结果（赚/亏/持有）
      ↓
  月度复盘：决策质量 vs 决策结果
      ↓
  改进系统：更新规范 → 更新Agent配置 → 更新知识卡片
      ↓
  下一轮分析管道运行（比上一轮强一点点）
```

### 4.2 进化机制

| 机制 | 触发 | 产出 | 频率 |
|:---|:---|:---|:---|
| **Auto-PR** | 每日自动扫描4个代码库 | 异常修复草案 → `auto_fixes/` | 每日 |
| **Agent E交叉验证** | 每次财报季3+公司并行分析 | 数据一致性矛盾 → AI异常日志 | 每季 |
| **知识库飞轮** | 每周扫描幽灵链接/孤儿卡片/升级候选 | 知识卡片→规范升级 | 每周 |
| **月度自审** | 投资行为规范§7.2 | 本月最大错误+改进方向 | 每月 |
| **半年能力圈review** | 投资行为规范§1.3 | 能力圈边界调整 | 每半年 |
| **年度复盘** | 投资行为规范§7.3 | 收益率vs基准+系统级改进 | 每年 |

### 4.3 AI异常日志

> 当前为proto阶段——Agent E首次运行已产生3条记录。

每次Agent E发现以下情况时记录：
- A/B/C对同一事实的表述矛盾
- 某个Agent的数据置信度标注为"[数据不足]"且影响结论
- Agent的"估计值"被后续客观数据证伪

所有异常日志积累为季度改进输入：改进Agent配置、升级数据源、更新评分权重。

## 系统健康检查（每季度）

1. **管道健康**：Agent E最近一次运行发现了几个矛盾？是新矛盾还是老问题？
2. **纪律健康**：最近3个月有没有违反投资行为规范？哪一条最危险？
3. **配置健康**：仓位有没有超标？现金够不够？回撤到哪了？
4. **进化健康**：Auto-PR最近生成了几个有效修复？Agent配置有没有比上季度更好？
5. **知识健康**：知识库飞轮里，有没有投资相关的卡片该升级到规范了？

## 与其他五个目标的关系

投资成功是总目标。其他五个目标都是投资系统的上游输入或下游保障：

| 目标 | 对投资系统的作用 | 接口 |
|:---|:---|:---|
| **事业** | 提供现金流（可投资本金）+ 信息优势（科技金融行业Alpha） | 资本转化率监控 |
| **AI能力** | 加速分析管道（规范力→Agent配置、编排力→并行分析、验证力→交叉检查） | 五维度飞轮 |
| **知识库** | 存储投资知识卡片（Layer 2→Layer 1升级） | 知识库飞轮 |
| **健康** | 确保长期复利（活到100岁才能享受复利） | 生命体征监控 |
| **家庭** | 防火墙：教育金/赡养费/应急金不侵蚀投资本金 | 家庭财务管理 |
