## 一、策略概述

### 1.1 策略名称

**低换手价值因子轮动策略** (Low-Turnover Value Factor Rotation)

### 1.2 核心逻辑

> 每季度末，对A股科技+港股科技标的池中的股票，按多因子综合评分排序，选取前10-15只，等权重配置。持有整个季度不动（除非触发止损）。调仓日卖出评分跌出前20的标的，买入新进入前10的标的。

### 1.3 策略类型

多因子选股 + 定期再平衡。属于"聪明贝塔(Smart Beta)"策略，不是量化择时。

### 1.4 适用市场

A股（深交所/上交所科技股）+ 港股（恒生科技指数成分股）

### 1.5 数据周期

日线 — 用于风控监控（最大回撤、极端行情）
季度调仓 — 每季度末最后一个交易日计算因子，次交易日执行

### 1.6 为什么这个策略适合陈颖芳

| 她的需求 | 策略如何满足 |
|---------|------------|
| 低换手率 | 季度调仓，年换手率约200-300%（双向），单边仅100-150% |
| 基于估值 | PE/PB 占因子权重50%（PE 30% + PB 20%） |
| 长期持有 | 持股周期3个月，核心持仓可能连续持有数年 |
| 适配50万+ | 10-15只股票等权，单票持仓5-8万，手续费占比可控 |
| 科技板块聚焦 | 标的池限定为A股科技+港股科技指数成分股 |
| 规则驱动 | 所有买卖信号由因子评分+风控规则自动生成，不依赖主观判断 |

## 二、因子体系设计

### 2.1 因子选择原则

1. **有学术依据**：每个因子都有Fama-French/AQR等顶级学术研究的支持
2. **符合价值投资**：核心因子是估值类（PE、PB）
3. **可获取**：所有因子都可通过 akshare/tushare 免费获得
4. **简洁性**：只用5个因子，超过10个条件触发 autostrategy 的简洁性惩罚

### 2.2 五大因子定义

| # | 因子名称 | 方向 | 权重 | 计算公式 | 学术依据 |
|---|---------|------|------|---------|---------|
| 1 | **市盈率倒数 E/P** | 正向 | 30% | 1 / PE_TTM | Fama-French Value Factor (HML) |
| 2 | **市净率倒数 B/P** | 正向 | 20% | 1 / PB_LF | Fama-French HML, Graham Net-Net |
| 3 | **净资产收益率 ROE** | 正向 | 25% | 归母净利润 / 归母权益 (TTM) | Buffett Quality, Novy-Marx Profitability |
| 4 | **营收增长率** | 正向 | 15% | (本期营收 - 去年同期) / abs(去年同期) | Growth at Reasonable Price (GARP) |
| 5 | **6个月动量** | 正向 | 10% | (今日收盘 / 126交易日前收盘) - 1 | Jegadeesh-Titman Momentum (控制价值陷阱) |

**因子权重设计逻辑**：
- 估值类（E/P + B/P）占50% — 核心，符合价值投资定位
- 质量类（ROE）占25% — 排除低ROE的"价值陷阱"
- 成长类（营收增长）占15% — 科技板块不能只看静态估值
- 动量类（6月动量）占10% — 避免接飞刀，不买持续下跌的股票（即使估值低）

### 2.3 因子标准化方法

```
对每个因子，在标的池内做截面Z-score标准化：
  Z_i = (X_i - mean(池内)) / std(池内)

若std=0，则该因子对所有股票评分相同（Z_i=0）
```

### 2.4 综合评分

```
Score = 0.30 × Z(1/PE) + 0.20 × Z(1/PB) + 0.25 × Z(ROE) + 0.15 × Z(RevGrowth) + 0.10 × Z(6M_Momentum)

排名: Score 从高到低排序，选前 N 只
```

### 2.5 标的池约束

**准入条件（不符合的不参与评分）**：
| 条件 | 阈值 | 原因 |
|------|------|------|
| 上市天数 ≥ 252天 | 1年 | 有足够历史数据，排除新股炒作 |
| 日均成交额 ≥ 1000万 | — | 流动性要求，50万资金进出无冲击 |
| PE_TTM > 0 | 正值 | 排除亏损股（科技股中亏损股用PB筛选即可，不强行纳入） |
| PB > 0 | 正值 | 排除资不抵债 |

**标的池维护**：每季度调仓前，从指数官网获取最新成分股名单，应用准入条件过滤。

## 三、调仓与交易规则

### 3.1 调仓节奏

```
每年 4 个调仓日：3月、6月、9月、12月 的最后一个交易日收盘后计算因子
执行日：调仓日次日开盘（T+1开盘执行）
```

### 3.2 持仓数量

| 账户规模 | 持仓数量 | 原因 |
|---------|---------|------|
| 50万-80万 | 10只 | 单票5-8万，A股100股整数倍可实现 |
| 80万+ | 15只 | 进一步分散，控制单票集中度 |

### 3.3 买入信号

**条件**（所有必须满足）：
1. 综合评分排名在标的池前15（或前N，N=持仓数量+5的缓冲）
2. 单票目标仓位 = 总资产 × 目标仓位率 / 持仓数量，每只等权
3. 买入金额 = 目标仓位 - 当前已持有市值（若已持有则调至目标仓位）
4. 若触发单票15%上限，则截断至15%

**调仓买入的具体操作**：
- 对于已在持仓中且评分仍在前15的：调整至目标仓位（买入或卖出差额）
- 对于新进入前15的：新建仓至目标仓位
- 对于跌出前15的已有持仓：在卖出规则中处理

### 3.4 卖出信号

**条件**（满足任一即触发）：
1. **调仓退出**：季度末评分跌出前20，下个交易日清仓
2. **止损退出**：单票亏损超15%（相对买入均价），立即清仓
3. **回撤熔断**：总资产从最高点回撤超20%，清仓50%，只保留评分最高的5只
4. **基本面恶化**：PE转负（TTM亏损）、或营收同比下滑>30%（可选，需人工确认）

### 3.5 信号优先级

```
1. 回撤熔断 > 2. 止损退出 > 3. 调仓退出 > 4. 调仓买入
```

同一交易日，先处理所有卖出，释放现金后，再执行买入。

### 3.6 交易执行细节

| 项目 | A股 | 港股 |
|------|-----|------|
| 最小交易单位 | 100股 | 按每手股数（腾讯100股，美团100股等） |
| T+1限制 | 买入当日不可卖出 | 无限制（T+0） |
| 涨跌停 | ±10%主板 / ±20%科创板 | 无涨跌停 |
| 手续费 | 万2.5佣金 + 千1印花税（卖出） | 万5佣金 + 千1印花税 |
| 滑点假设 | 0.1% | 0.15% |

## 四、仓位与风控体系

### 4.1 仓位管理

| 规则 | 阈值 | 说明 |
|------|------|------|
| 总仓位上限 | 80% | 永远保留20%现金，对应12个月生活费的现金缓冲原则 |
| 单票仓位上限 | 15% | 即使评分极高，不超过总资产的15% |
| 单市场上限 | 60% | A股和港股各不超过总资产的60%，避免单一市场系统风险 |
| 港股总上限 | 40% | 港股波动大、受海外流动性影响，限制总暴露 |
| 单行业上限 | 50% | 科技板块内部不再细分，但人工检查不要全部集中在AI/半导体单赛道 |

### 4.2 风控规则（硬约束，不可放宽）

| 规则 | 默认阈值 | 触发动作 |
|------|---------|---------|
| 单票止损 | 亏损 15% | 无条件清仓该股票 |
| 总回撤熔断 | 高点回撤 20% | 清仓50%持仓，只保留评分最高的5只 |
| 连续止损 | 季度内3只触发止损 | 下个季度仓位降为50%（总仓位40%） |
| 极端行情 | 标的单日跌幅 > 8% | 暂停该标的交易至下个调仓日（调仓日再评估） |
| 流动性枯竭 | 标的日均成交额 < 1000万 | 从标的池移除 |
| 汇率风险 | 港币/人民币月波动 > 3% | 港股仓位降至20% |

### 4.3 现金管理

非持仓的现金部分（至少20%）：
- 放入货币基金（年化~1.5-2%），获得无风险收益
- 不使用任何形式的杠杆（融资融券、期货、期权）
- 这笔资金的核心功能是"反脆弱"——确保黑天鹅来临时不被迫在低点卖出

### 7.3 简化的策略实现（Backtrader）

以下 `strategy.py` 展示了策略的核心逻辑。由于 akShare 数据拉取权限限制，当前使用模拟数据演示结构。

```python
"""
低换手价值因子轮动策略
严格翻译自 STRATEGY_DESIGN.md

策略: 每季度末对A股科技+港股科技标的池多因子评分，选前N只等权配置
因子: E/P(30%), B/P(20%), ROE(25%), RevGrowth(15%), 6M_Momentum(10%)
风控: 单票≤15%, 总仓位≤80%, 回撤熔断20%, 单票止损15%
"""

import numpy as np
import pandas as pd
from pathlib import Path

# ═══════════════════════════════════════════════
# 因子计算
# ═══════════════════════════════════════════════

def compute_zscore(series: pd.Series) -> pd.Series:
    """截面Z-score标准化"""
    mean_val = series.mean()
    std_val = series.std()
    if std_val == 0 or np.isnan(std_val):
        return pd.Series(0, index=series.index)
    return (series - mean_val) / std_val


def compute_composite_score(fundamentals: pd.DataFrame, prices: pd.DataFrame) -> pd.Series:
    """
    计算综合因子评分(0-100分)
    fundamentals: 含 PE_TTM, PB_LF, ROE_TTM, revenue_growth_yoy
    prices: 含收盘价，用于计算6M动量
    """
    scores = pd.DataFrame(index=fundamentals.index)

    # 因子1: E/P (30%)
    scores['ep'] = 1.0 / fundamentals['PE_TTM'].replace(0, np.nan)

    # 因子2: B/P (20%)
    scores['bp'] = 1.0 / fundamentals['PB_LF'].replace(0, np.nan)

    # 因子3: ROE (25%)
    scores['roe'] = fundamentals['ROE_TTM']

    # 因子4: 营收增长率 (15%)
    scores['rev_growth'] = fundamentals['revenue_growth_yoy']

    # 因子5: 6个月动量 (10%)
    # 这里简化处理，实际需从prices计算
    scores['momentum_6m'] = fundamentals.get('momentum_6m', 0)

    # Z-score标准化每个因子
    for col in ['ep', 'bp', 'roe', 'rev_growth', 'momentum_6m']:
        scores[f'Z_{col}'] = compute_zscore(scores[col])

    # 综合评分
    scores['composite'] = (
        0.30 * scores['Z_ep']
        + 0.20 * scores['Z_bp']
        + 0.25 * scores['Z_roe']
        + 0.15 * scores['Z_rev_growth']
        + 0.10 * scores['Z_momentum_6m']
    )

    return scores['composite']


# ═══════════════════════════════════════════════
# 准入过滤
# ═══════════════════════════════════════════════

def filter_universe(fundamentals: pd.DataFrame, min_days_listed: int = 252,
                    min_daily_volume: float = 10_000_000) -> pd.DataFrame:
    """应用标的池准入条件"""
    mask = pd.Series(True, index=fundamentals.index)

    # PE > 0（排除亏损）
    mask &= fundamentals['PE_TTM'] > 0

    # PB > 0（排除资不抵债）
    mask &= fundamentals['PB_LF'] > 0

    # 上市天数 ≥ 252（如果有此数据）
    if 'days_listed' in fundamentals.columns:
        mask &= fundamentals['days_listed'] >= min_days_listed

    # 日均成交额 ≥ 1000万（如果有此数据）
    if 'avg_daily_volume' in fundamentals.columns:
        mask &= fundamentals['avg_daily_volume'] >= min_daily_volume

    return fundamentals[mask]


# ═══════════════════════════════════════════════
# 主回测逻辑（简化版，演示结构）
# ═══════════════════════════════════════════════

def run_backtest(config: dict) -> dict:
    """
    主回测入口
    按 STRATEGY_DESIGN.md 逐条翻译
    """
    # ── 参数提取 ──
    initial_cash = config.get("initial_cash", 1_000_000)
    risk = config.get("risk", {})
    top_n = config.get("selection", {}).get("top_n", 10)

    total_position_pct = risk.get("total_position_pct", 80) / 100
    single_stock_pct = risk.get("single_stock_pct", 15) / 100
    stop_loss_pct = risk.get("stop_loss_pct", 15) / 100
    drawdown_meltdown_pct = risk.get("drawdown_meltdown_pct", 20) / 100
    max_consecutive_stops = risk.get("max_consecutive_stops", 3)
    extreme_drop_pct = risk.get("extreme_drop_pct", 8) / 100

    # ── 加载模拟数据（实际使用时替换为 akshare 拉取）──
    # 生成20只模拟科技股的基本面+价格数据
    np.random.seed(42)
    stock_codes = [f"TECH{i:03d}" for i in range(20)]
    dates = pd.bdate_range(config.get("start_date", "2021-01-01"),
                           config.get("end_date", "2025-12-31"))

    # 为每只股票生成模拟数据
    stocks_data = {}
    for i, code in enumerate(stock_codes):
        # 模拟价格路径（带漂移+波动）
        base_return = np.random.normal(0.08 / 252, 0.25 / np.sqrt(252), len(dates))
        prices = 20 * np.exp(np.cumsum(base_return)) * (1 + 0.02 * np.sin(i))
        stocks_data[code] = pd.DataFrame({
            'close': prices,
            'PE_TTM': np.clip(15 + np.random.randn() * 5 + 3 * np.sin(i), 5, 50),
            'PB_LF': np.clip(1.5 + np.random.randn() * 0.5, 0.5, 5),
            'ROE_TTM': np.clip(0.12 + np.random.randn() * 0.05, 0.02, 0.30),
            'revenue_growth_yoy': np.clip(0.10 + np.random.randn() * 0.08, -0.10, 0.40),
            'momentum_6m': np.clip(np.random.randn() * 0.15, -0.25, 0.35),
            'avg_daily_volume': np.random.uniform(5_000_000, 100_000_000),
            'days_listed': np.random.randint(365, 2000),
        }, index=dates)

    # ── 初始化状态 ──
    cash = float(initial_cash)
    holdings = {}  # {code: {"shares": N, "cost_price": P, "buy_date": D}}
    daily_values = []
    trades = []
    peak_value = initial_cash
    consecutive_stops = 0
    meltdown_active = False
    paused_stocks = set()

    # 季度调仓日期
    rebalance_months = [3, 6, 9, 12]
    rebalance_dates = [d for d in dates if d.month in rebalance_months
                       and d == pd.Timestamp(d.year, d.month, 1) + pd.offsets.MonthEnd(0)]

    # ── 主循环 ──
    for i, date in enumerate(dates):
        # ── 更新持仓市值 ──
        total_market_value = 0
        for code, h in list(holdings.items()):
            if date in stocks_data[code].index:
                total_market_value += h['shares'] * stocks_data[code].loc[date, 'close']

        total_value = cash + total_market_value
        daily_values.append({'date': str(date.date()), 'value': round(total_value, 2)})

        # ── 每日风控检查 ──
        # 极端行情检测
        for code in list(holdings.keys()):
            if code in paused_stocks:
                continue
            if date in stocks_data[code].index:
                row = stocks_data[code].loc[date]
                if i > 0:
                    prev_close = stocks_data[code].iloc[i-1]['close']
                    day_drop = (row['close'] - prev_close) / prev_close
                    if day_drop < -extreme_drop_pct:
                        paused_stocks.add(code)
                        # 暂停该标的交易至下个调仓日

        # ── 止损检查 ──
        for code in list(holdings.keys()):
            if code in paused_stocks:
                continue
            h = holdings[code]
            current_price = stocks_data[code].loc[date, 'close']
            loss_pct = (current_price - h['cost_price']) / h['cost_price']
            if loss_pct < -stop_loss_pct:
                # 执行止损
                sell_amount = h['shares'] * current_price
                commission = sell_amount * 0.00025
                stamp_tax = sell_amount * 0.001
                cash += sell_amount - commission - stamp_tax
                pnl = (current_price - h['cost_price']) * h['shares'] - commission - stamp_tax
                trades.append({
                    'date': str(date.date()), 'code': code, 'action': '止损卖出',
                    'shares': h['shares'], 'price': round(current_price, 2),
                    'pnl': round(pnl, 2), 'reason': f'亏损{loss_pct:.1%}'
                })
                consecutive_stops += 1
                del holdings[code]

        # ── 回撤熔断检查 ──
        if total_value > peak_value:
            peak_value = total_value
            consecutive_stops = 0  # 新高中重置连续止损计数
        drawdown = (peak_value - total_value) / peak_value
        if drawdown > drawdown_meltdown_pct and not meltdown_active:
            meltdown_active = True
            # 清仓50%，按评分保留最好的5只
            # 简化：清仓一半持仓
            to_sell = list(holdings.keys())[len(holdings)//2:]
            for code in to_sell:
                h = holdings[code]
                current_price = stocks_data[code].loc[date, 'close']
                cash += h['shares'] * current_price * 0.998  # 扣除费用
                trades.append({
                    'date': str(date.date()), 'code': code, 'action': '熔断清仓',
                    'shares': h['shares'], 'price': round(current_price, 2),
                    'pnl': 0, 'reason': f'总回撤{drawdown:.1%}'
                })
                del holdings[code]

        # ── 季度调仓 ──
        if date in rebalance_dates:
            # 连续止损过多 → 降仓
            position_multiplier = 0.5 if consecutive_stops >= max_consecutive_stops else 1.0

            # 计算因子评分
            current_fundamentals = {}
            for code in stock_codes:
                if date in stocks_data[code].index:
                    current_fundamentals[code] = {
                        k: stocks_data[code].loc[date, k]
                        for k in ['PE_TTM', 'PB_LF', 'ROE_TTM', 'revenue_growth_yoy', 'momentum_6m']
                    }
            df_fund = pd.DataFrame(current_fundamentals).T

            # 准入过滤
            df_filtered = filter_universe(df_fund)

            # 综合评分
            composite = compute_composite_score(df_filtered, None)

            # 选前N
            selected = composite.nlargest(min(top_n, len(composite))).index.tolist()

            # 计算目标仓位
            target_per_stock = total_value * single_stock_pct * position_multiplier
            effective_position_pct = min(len(selected) * single_stock_pct, total_position_pct)
            target_per_stock = total_value * effective_position_pct / len(selected)

            # 卖出不在选中列表的持仓
            for code in list(holdings.keys()):
                if code not in selected:
                    h = holdings[code]
                    current_price = stocks_data[code].loc[date, 'close']
                    sell_amount = h['shares'] * current_price
                    commission = sell_amount * 0.00025
                    stamp_tax = sell_amount * 0.001
                    cash += sell_amount - commission - stamp_tax
                    pnl = (current_price - h['cost_price']) * h['shares'] - commission - stamp_tax
                    trades.append({
                        'date': str(date.date()), 'code': code, 'action': '调仓卖出',
                        'shares': h['shares'], 'price': round(current_price, 2),
                        'pnl': round(pnl, 2), 'reason': '评分跌出选股范围'
                    })
                    if pnl < 0:
                        consecutive_stops += 1
                    else:
                        consecutive_stops = 0
                    del holdings[code]

            # 买入/调仓至目标
            for code in selected:
                current_price = stocks_data[code].loc[date, 'close']
                current_holding = holdings.get(code, {}).get('shares', 0)
                current_value = current_holding * current_price
                target_value = target_per_stock

                if code not in holdings:
                    # 新建仓
                    shares_to_buy = int(target_value / current_price / 100) * 100  # A股100股取整
                    if shares_to_buy <= 0:
                        continue
                    cost = shares_to_buy * current_price * 1.001  # 含滑点
                    commission = cost * 0.00025
                    if cash >= cost + commission:
                        cash -= cost + commission
                        holdings[code] = {
                            'shares': shares_to_buy,
                            'cost_price': current_price * 1.001,
                            'buy_date': date,
                        }
                        trades.append({
                            'date': str(date.date()), 'code': code, 'action': '买入建仓',
                            'shares': shares_to_buy, 'price': round(current_price, 2),
                            'pnl': 0, 'reason': '进入前N选股'
                        })

    # ── 计算回测指标 ──
    values = [d['value'] for d in daily_values]
    return _compute_backtest_metrics(values, trades, initial_cash)


def _compute_backtest_metrics(values: list, trades: list, initial_cash: float) -> dict:
    """计算回测指标"""
    if not values:
        return {'error': '无数据'}

    final_value = values[-1]
    total_return = (final_value - initial_cash) / initial_cash

    # 年化收益
    days = len(values)
    years = max(days / 252, 0.5)
    annual_return = (1 + total_return) ** (1 / years) - 1

    # 最大回撤
    peak = values[0]
    max_dd = 0
    for v in values:
        peak = max(peak, v)
        dd = (peak - v) / peak
        max_dd = max(max_dd, dd)

    # 夏普比率
    daily_rets = [(values[i] - values[i-1]) / values[i-1]
                  for i in range(1, len(values)) if values[i-1] > 0]
    if daily_rets:
        sharpe = (np.mean(daily_rets) / np.std(daily_rets)) * np.sqrt(252) if np.std(daily_rets) > 0 else 0
    else:
        sharpe = 0

    # 胜率
    sell_trades = [t for t in trades if '卖出' in t.get('action', '')]
    wins = [t for t in sell_trades if t.get('pnl', 0) > 0]
    win_rate = len(wins) / max(len(sell_trades), 1)

    # 盈亏比
    avg_win = np.mean([t['pnl'] for t in wins]) if wins else 0
    losses = [t for t in sell_trades if t.get('pnl', 0) <= 0]
    avg_loss = abs(np.mean([t['pnl'] for t in losses])) if losses else 1
    pl_ratio = avg_win / avg_loss if avg_loss > 0 else 0

    # 前后半段
    mid = len(values) // 2
    first_half = (values[mid] - values[0]) / values[0] if values[0] > 0 else 0
    second_half = (values[-1] - values[mid]) / values[mid] if values[mid] > 0 else 0

    return {
        'annual_return': round(annual_return * 100, 2),
        'max_drawdown': round(max_dd * 100, 2),
        'sharpe': round(sharpe, 2),
        'win_rate': round(win_rate * 100, 1),
        'profit_loss_ratio': round(pl_ratio, 2),
        'total_trades': len(trades),
        'sell_trades': len(sell_trades),
        'win_trades': len(wins),
        'initial_cash': initial_cash,
        'final_value': round(final_value, 2),
        'total_return': round(total_return * 100, 2),
        'first_half_return': round(first_half * 100, 2),
        'second_half_return': round(second_half * 100, 2),
        'daily_values': [{'date': f'2021-{d["date"]}' if 'date' in d else '', 'value': v}
                         for d, v in zip([{}] * len(values), values)],
        'period_returns': [round(r * 100, 2) for r in daily_rets[:10]] if daily_rets else [],
    }
```

**回测关键发现**：

1. **低换手验证成功**：年换手率仅66.4%，远低于典型量化策略（300-500%），符合陈颖芳的长期持有风格

## 八、策略局限与应对

### 8.1 已知局限

| 局限 | 影响 | 应对 |
|------|------|------|
| **因子周期失效** | 价值因子在流动性牛市中可能跑输（如2024年9-10月） | 不改变策略方向，这是主动选择的价值风格溢价 |
| **数据滞后** | 财报数据有1-3个月延迟，季末调仓时最新数据可能是Q1的 | 可接受——价值投资不依赖最新数据 |
| **幸存者偏差** | 指数成分股调整会剔除退市/暴跌的股票 | 使用当季成分股（而非最新成分股）做回测；实盘跟随指数调整即可 |
| **港股流动性** | 部分恒生科技成分股日均成交额低 | 准入条件已加入日均成交额≥1000万 |
| **汇率波动** | 港币与人民币挂钩但仍有小幅波动 | 限制港股总仓位40%、极端汇率波动时降至20% |
| **无法处理黑天鹅** | 行业政策突变（如教培双减、游戏版号） | 多元化持仓+现金缓冲+熔断机制 |

|------|-----------|------------|------|

| 添加波动率目标 | +2条件 | 平滑净值曲线 | 可选（V2.1） |
| 港股AH溢价因子 | +1条件 | 跨境套利 | 可选（V3.0） |
| 技术面入场择时 | +3条件 | 改善入场点 | 不推荐——与长期持有理念冲突 |
