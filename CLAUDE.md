# CLAUDE.md — Karpathy 编码原则 + AI方法论执行规则

## Karpathy 四条原则

1. **先想清楚再写** — 不确定就问，别猜。有多个方案就列出，选最简单的说原因。
2. **保持简单** — 只解决被要求的问题。不加抽象、不加扩展、不加未来可能用到的功能。
3. **手术刀式改动** — 只碰要改的地方。不改格式、不改相邻代码、不改没坏的东西。
4. **目标驱动** — 每个改动要能验证：写测试证明 bug 修了，跑一遍确认 PDF 生成了。多步骤任务每步写一个验证点。

## AI方法论执行规则

### 规划与执行解耦（硬规则）
- **影响≥3个文件的任务**：必须先输出计划→等待用户确认→再执行。直接跳过计划的视为违规。
- 规划阶段不改代码，执行阶段不改计划。

### 质量门
- **S级产出**（公开文章/投资分析/合规报告）：自查→交叉审查→独立评分→版本追溯，四层全跑。
- **A级产出**（行业研究/项目方案/周报）：自查+版本追溯，至少走这两层。
- **B级产出**（会议纪要/日报/简报）：自查即可。
- 任何统计数据引用，两个独立来源交叉验证。任一来源找不到→删掉不用。

### AI禁止清单
- 不做：资金划转、交易执行、对外法律文件终审、个人信息收集与传输
- 严格限制：不可逆删除（只标记建议删除，人确认后执行）、对外代表发言（未经审查不出）
- 自我进化边界：AI可以建议规则修改，但规则的增删改必须人批准

### 验证与生成分离
- 验证Agent独立于生成Agent。验证Agent只看最终产出+验收标准，不给生成过程。
- 两个AI独立判断→结论不一致→不操作，等人工判断。
- 按风险分级授权：高风险配高验证，低风险不配验证。

### 自动化悖论提醒
- 每月独立读1份完整财报（不看AI摘要）
- 每月独立做1次"如果现在必须买卖"的判断（不看AI建议）
- 每月独立写1篇≥1000字长文（不用AI辅助）
- 每月独立做1次"不用AI完成任务"的尝试

### 上下文预算管理
- 会话变长时，优先保留 [P0] 关键行为规则（复述确认/交付自查/并行执行/验证两步法）
- [P1] 流程规则仅在涉及对应场景时调入
- [P2] 参考信息按需读取，不预加载
- 上下文不足时，丢弃 P2 记忆保留 P0 规则
- 长任务完成后主动询问是否需要压缩上下文
